# 任务完成检查清单

## 代码变更前

- [ ] 已阅读相关规范文档（CLAUDE.md、需求文档、技术方案）
- [ ] 理解变更的影响范围
- [ ] 构思方案并获得用户确认

## 代码变更后

- [ ] 运行 `pnpm typecheck` 确保类型正确
- [ ] 运行 `pnpm test` 确保测试通过
- [ ] 如有新功能，补充对应测试
- [ ] 代码变更与规范定义一致

## 提交前

- [ ] **必须** 在 `CHANGELOG.md` 添加变更记录
  - 格式：`- vX.Y.Z YYYY-MM-DD 作者: 一句话摘要`
  - 用户可见变更追加 `(user-visible)`
- [ ] 如涉及规范变更，同步更新 `CLAUDE.md`
- [ ] Git commit message 清晰描述变更

## 变更记录格式示例

```
- v0.5.37 2026-02-26 Leo: fix: Planning-with-Files P1-2 PostToolUse 进度同步提醒 (user-visible)
- v0.5.36 2026-02-26 Leo: feat: 新增 xxx 功能
```

## 测试要求

- 单元测试：覆盖核心逻辑
- 集成测试：覆盖模块间交互
- E2E 测试：覆盖用户场景

## 文档要求

- 新增功能需更新使用手册
- API 变更需更新命令参考
- 架构变更需更新技术方案
