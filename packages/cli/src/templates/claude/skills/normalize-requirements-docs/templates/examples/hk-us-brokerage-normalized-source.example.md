# 港美股券商产品需求文档

## 一、产品范围

### 1.1 市场范围
- 产品支持港股与美股市场交易。
- 账户类型至少包含现金账户与保证金账户。
- 用户需先完成开户审核，审核通过后方可进行入金与交易。

### 1.2 用户角色
- 终端角色包括个人投资者与内部审核人员。

## 二、开户流程

### 2.1 开户入口
- 源锚点：`broker-prd/02-onboarding.pdf | p.3 | Account Opening Entry`
- 开户入口展示市场选择、账户类型、身份信息收集与风险披露确认。

#### 开户表单页面

**图片说明：开户表单标注图**
- 源锚点：`broker-prd/images/account-opening-annotated.png | image | Account Opening Form`
- 可见文字：开户市场、账户类型、税务身份、居住地址、职业信息、上传身份证明、上传住址证明
- 备注：
  - 页面为开户表单，包含市场选择、账户类型、税务身份、地址、职业信息和上传区域
  - 红线标注强调 W-8BEN 仅在涉及美股税务声明时出现
  - 图中有“上传身份证明”“上传住址证明”两个独立上传区
  - 标注文字包含“开发画图无需标黄”

**核心需求点：**
- 开户入口需要同时支持香港市场和美国市场选择。
- 账户类型至少支持现金账户和保证金账户。
- 涉及美股税务声明时展示 W-8BEN 表单入口。
- 身份证明和住址证明为两个独立上传区域。

**设计要求：**
- 市场和账户类型使用并列单选样式展示。
- 上传身份证明与上传住址证明需在同一页面清晰分区展示。
- 红线标注中的特殊实现说明需要保留在产出中。

**开发备注：**
- “开发画图无需标黄”属于图片中的明确标注，需原样保留。

#### 开户字段规则

| 字段 | 必填 | 规则 | 展示 / 交互 | 备注 |
|------|------|------|-------------|------|
| 开户市场 | 是 | 仅可单选香港市场或美国市场 | 默认不选，需用户主动选择 | 市场选择影响后续税务身份字段 |
| 账户类型 | 是 | 仅可单选现金账户或保证金账户 | 与开户市场同层展示 | 保证金账户需展示额外风险披露 |
| 税务身份 | 条件必填 | 涉及美股税务声明时显示 W-8BEN | 非美股场景不展示 | 保留英文术语 W-8BEN |
| 居住地址 | 是 | 首尾去空格，详细地址保留原始中英文字符 | 多字段组合输入 | 用于住址证明核验 |
| 上传身份证明 | 是 | 支持上传单个身份证明文件 | 上传区独立展示 | 与住址证明分开 |
| 上传住址证明 | 是 | 支持上传单个住址证明文件 | 上传区独立展示 | 与身份证明分开 |

### 2.2 身份认证与合规采集
- 源锚点：`broker-prd/02-onboarding.pdf | p.5-p.9 | Identity Verification`
- 开户流程要求提交身份证明、住址证明与职业信息。
- 对美股交易相关账户，文档明确提到税务声明表单 W-8BEN。
- 文档要求 AML/KYC 校验。

### 2.3 风险披露确认
- 源锚点：`broker-prd/02-onboarding.pdf | p.10-p.12 | Risk Disclosure`
- 用户在提交开户申请前，需阅读并确认风险披露。
- 保证金账户另有杠杆与强平风险提示。

## 三、交易链路

### 3.1 下单与校验
- 源锚点：`broker-prd/03-trading-flow.docx | Order Ticket | Order Entry`
- 下单页包含证券代码、买卖方向、订单类型、数量、价格与可用购买力信息。
- 提交前会进行余额/购买力校验与交易时段校验。

#### 下单页面

**图片说明：下单页面原型**
- 源锚点：`broker-prd/images/order-ticket.png | image | Order Ticket Screen`
- 可见文字：证券代码、买卖方向、订单类型、数量、价格、可用购买力、提交订单
- 备注：
  - 页面展示标准下单表单
  - 页面包含实时购买力显示区域
  - 页面底部有风险提示信息和提交按钮

```text
┌────────────────────────────────────────────────────────────┐
│ 证券代码 [__________]    名称 / 市场 [自动回显]            │
├────────────────────────────────────────────────────────────┤
│ 买卖方向  [买入 ▼]                                         │
│ 订单类型  [限价单 ▼]                                       │
│ 数量      [__________]                                     │
│ 价格      [__________]                                     │
├────────────────────────────────────────────────────────────┤
│ 可用购买力：HKD 100,000.00 / USD 50,000.00                 │
│ 风险提示：非交易时段仅支持预约下单                          │
├────────────────────────────────────────────────────────────┤
│                        [提交订单]                           │
└────────────────────────────────────────────────────────────┘
```

**核心需求点：**
- 下单页面需展示证券代码、买卖方向、订单类型、数量、价格和可用购买力。
- 提交前需要进行余额 / 购买力校验和交易时段校验。
- 非交易时段需要明确提示预约下单或不可交易状态。

**设计要求：**
- 表单字段按下单顺序垂直排列。
- 购买力信息需在提交按钮前可见。
- 风险提示需和提交动作在同一视区内出现。

**开发备注：**
- 风险提示文案属于页面显式信息，不能在转写时省略。

#### 下单字段与校验规则

| 字段 | 必填 | 规则 | 展示 / 交互 | 备注 |
|------|------|------|-------------|------|
| 证券代码 | 是 | 支持证券代码输入与名称回显 | 输入后自动回显证券名称 / 市场 | 回显信息属于页面可见内容 |
| 买卖方向 | 是 | 仅可选择买入或卖出 | 下拉或单选展示 | 保留原始术语 |
| 订单类型 | 是 | 至少支持限价单 | 与买卖方向同区展示 | 不补充源文档未写明的订单类型 |
| 数量 | 是 | 输入后参与购买力校验 | 与价格联动 | 示例值需原样保留 |
| 价格 | 条件必填 | 限价单场景下输入价格 | 输入框展示 | 非交易时段规则另行提示 |
| 可用购买力 | 否 | 实时展示计算结果 | 在提交按钮前展示 | 属于只读信息 |

### 3.2 交易时段约束
- 源锚点：`broker-prd/03-trading-flow.docx | Session Rules | Trading Constraints`
- 港股与美股交易时段规则不同。
- 文档要求在非交易时段提示"可预约下单"或"当前不可交易"。

### 3.3 结算与资金可用性
- 源锚点：`broker-prd/03-trading-flow.docx | Settlement | Funds Availability`
- 成交后展示预计结算信息。
- 入金到账后才可释放对应购买力。

#### 入金方式

**图片说明：入金页面**
- 源锚点：`broker-prd/images/funding.png | image | Funding Screen`
- 可见文字：入金方式、到账时间、手续费
- 备注：
  - 页面展示多种入金渠道
  - 每种方式显示预计到账时间
  - 标注是否收取手续费

```text
┌────────────────────────────────────────────────────────────┐
│ 入金方式                                                   │
├────────────────────────────────────────────────────────────┤
│ [银行转账]   到账时间：1-2 个工作日   手续费：按银行收取   │
│ [FPS 转数快] 到账时间：实时/当日      手续费：0            │
│ [支票入金]   到账时间：2-3 个工作日   手续费：0            │
├────────────────────────────────────────────────────────────┤
│ 红色提示：到账时间以银行处理时效为准                        │
└────────────────────────────────────────────────────────────┘
```

**核心需求点：**
- 入金页面需要列出不同入金方式、到账时间和手续费信息。
- 红色提示语属于页面显式内容，需原样保留。

**设计要求：**
- 入金方式列表按渠道分行展示。
- 到账时间和手续费与渠道信息同行显示，便于比较。

**开发备注：**
- 红色提示文字属于必须保留的显式视觉信息。
