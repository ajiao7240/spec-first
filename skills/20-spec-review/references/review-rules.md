# 审查规则

> Spec-Review Skill 的质量审查规则与评分标准

---

## 审查清单来源

**主清单**: `skills/03-spec/references/spec-review-checklist.md`

**术语表**: `skills/03-spec/references/test-level-glossary.md`

---

## 审查维度

### 1. 完整性

**检查项**:
- 所有 FR 都有明确定义
- 所有 NFR 都有量化指标
- 边界条件已说明
- 异常场景已覆盖

---

### 2. 清晰性

**检查项**:
- 无歧义词（"可能"、"大概"、"尽量"）
- 术语定义一致
- 描述具体可测
- 无冗余重复

---

### 3. 可测性

**检查项**:
- 每个 FR 可验证
- 验收标准明确
- 测试场景可枚举

---

### 4. 一致性

**检查项**:
- 与 constitution.md 无冲突
- FR 间无矛盾
- 追溯矩阵完整

---

## C10 评分标准

**计算**: `C10 = (通过项 / 总项) * 100%`

**阈值**:
- ≥ 80%: 通过
- < 80%: 阻断风险

---

## 审查结果格式

```markdown
# Spec 质量审查

> 生成时间: {timestamp}
> Feature: {featureId}

## 审查结果

- [x] 完整性: 所有 FR 已定义
- [ ] 清晰性: 存在歧义词
- [x] 可测性: 验收标准明确
- [x] 一致性: 无冲突

## C10 评分

通过: 3 / 4 = 75%

⚠️ 低于 80% 阈值，存在阻断风险

## 修订建议

1. 清除歧义词: spec.md:45 "系统应该尽量提高性能"
```
