# Spec-First Skill Boundary Map

- `spec-brainstorm`: clarify what should be built and why.
- `spec-plan`: produce a technical plan.
- `spec-write-tasks`: compile an approved plan into a derived task pack.
- `spec-work`: execute settled work.
- `spec-debug`: investigate and fix failures.
- `spec-code-review`: review code changes.
- `spec-doc-review`: review requirements, plans, and task packs.
- `spec-mcp-setup`: install and verify harness runtime.
- `spec-graph-bootstrap`: compile graph readiness facts after setup.
- `spec-update`: check and repair spec-first installation freshness.

This workflow reviews skill quality. It does not mine external methods or apply source fixes by default.
