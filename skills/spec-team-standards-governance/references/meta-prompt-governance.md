# Meta-Prompt Governance

The meta-prompt layer explains and orchestrates standards use. It is not a standards source and not an approver.

Responsibilities:

- Extract scope from workflow, changed paths, artifact type, surface, layer and capability.
- Load the contract, then the index, then only matched rule files.
- Explain confirmed rules in plan/work/review/debug/doc-review handoffs.
- Convert repeated workflow feedback into `suggested`, `observed`, conflict or audit outputs.
- Surface owner gaps, conflicts, stale index and overbroad scope.

Forbidden:

- Auto-confirming standards.
- Editing role contract, public route maps, generated runtime mirrors or host instructions.
- Treating graph/code evidence, history docs or LLM summaries as confirmed source.
- Packaging generic best practice as project-standards finding.
