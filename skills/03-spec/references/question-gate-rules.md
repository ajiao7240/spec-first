# Question Gate 规则

Step 3 Question Gate 的提问策略与分类规则。

---

## 核心原则

**先判断可推导再提问，避免无效提问。**

---

## 问题分类

### Blocking（阻断级）

**定义**：必须回答，否则无法继续。

**优先级**：范围 > 安全 > 合规

**示例**：
- 功能边界不清（"支持哪些支付方式？"）
- 安全要求缺失（"是否需要加密存储？"）
- 合规约束不明（"是否需要 GDPR 合规？"）

### Preference（偏好级）

**定义**：可选回答，影响体验但不阻断。

**优先级**：UX > 技术偏好

**示例**：
- UX 细节（"加载动画用哪种风格？"）
- 技术选型（"用 Redis 还是 Memcached？"）

---

## 提问约束

### 数量限制

- **每轮最多 3 个问题**
- **最多 5 轮澄清**
- **超过 5 轮仍有阻断级歧义 → 标记 [BLOCKED]**

### 格式要求

```markdown
[NEEDS CLARIFICATION][<TYPE>] <ID>: <问题>？候选范围 A/B/C
```

**TYPE 枚举**：
- `BOUNDARY` - 边界值
- `ERROR` - 异常处理
- `PRIORITY` - 优先级冲突
- `SEMANTIC` - 语义多解
- `DEPENDENCY` - 外部依赖缺失
- `METRIC` - 成功指标缺失
- `TERM` - 术语未定义
- `ROLE` - 角色/权限不清
- `TEMPORAL` - 时序/时限不清
- `DATA` - 数据来源/质量不清

---

## 动态生成流程（5 步）

### Step 1: 读取未闭环歧义

从 findings.md 或 PRD 中提取所有 `[NEEDS CLARIFICATION]` 标记。

### Step 2: 风险排序

按优先级排序：范围 > 安全 > 合规 > UX > 技术偏好

### Step 3: 过滤无效问题

删除"不会改变 FR/AC/NFR 结果"的问题。

**示例**：
- ❌ "用什么颜色？"（不影响功能）
- ✅ "超时时间多少？"（影响 NFR）

### Step 4: 合并重复问题

合并语义重复的问题，保证一轮最多 3 问。

### Step 5: 提供候选答案

每个问题提供 2-3 个可执行候选（A/B/C），避免开放式泛问。

---

## 提问示例

### 好的提问 ✅

```markdown
[NEEDS CLARIFICATION][BOUNDARY] FR-AUTH-001: 验证码有效期多少？候选 60s/120s/300s
```

**理由**：
- 明确问题类型（BOUNDARY）
- 关联具体 FR
- 提供可执行候选
- 影响 NFR（性能/安全）

### 不好的提问 ❌

```markdown
[NEEDS CLARIFICATION] 验证码怎么设计？
```

**问题**：
- 无类型标签
- 无 FR 关联
- 开放式泛问
- 无候选答案

---

## 回答处理

### 回答后立即写入

用户回答后，立即写入 `spec.md` 对应章节。

**示例**：
```markdown
## FR-AUTH-001: 短信验证码登录

验证码有效期：120s（用户确认于 2026-03-05）
```

### 更新歧义列表

从 findings.md 移除已回答问题，更新剩余歧义列表。

---

## 最佳实践

### DO ✅

1. **先推导再问** — 能从现有信息推导的不问
2. **提供候选** — 每个问题给 2-3 个候选答案
3. **关联 FR** — 问题关联具体 FR/NFR
4. **立即写入** — 回答后立即写入 spec.md
5. **更新列表** — 每轮结束更新剩余歧义

### DON'T ❌

1. **不要开放式泛问** — "怎么设计？"
2. **不要问无关问题** — 不影响 FR/AC/NFR 的不问
3. **不要超过 3 问/轮** — 超过 3 个合并或分轮
4. **不要重复问** — 检查是否已在前轮回答
5. **不要忘记写入** — 回答后必须落盘
