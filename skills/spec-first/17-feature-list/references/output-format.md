# 输出格式

> Feature-List Skill 的标准输出格式与示例

---

## 标准列表格式

```
📋 Feature 列表

项目: {project_name}
总数: {total_count} 个 Feature

---

| Feature ID | 标题 | 阶段 | 状态 | 更新时间 |
|-----------|------|------|------|----------|
| {id} | {title} | {stage} | {status} | {updated} |
...

---

💡 提示:
- 使用 /spec-first:feature-switch {id} 切换 Feature
- 使用 /spec-first:status 查看当前 Feature 详情
```

---

## 阶段显示

| 阶段代码 | 显示名称 | 图标 |
|---------|---------|------|
| 00_init | 初始化 | 🔧 |
| 01_specify | 需求规格 | 📝 |
| 02_design | 技术设计 | 🎨 |
| 03_plan | 任务拆解 | 📋 |
| 04_implement | 代码实现 | 💻 |
| 05_verify | 验证测试 | ✅ |
| 06_wrap_up | 归档复盘 | 📦 |
| 07_release | 发布上线 | 🚀 |
| 08_done | 已完成 | ✔️ |
| 09_cancelled | 已取消 | ❌ |

---

## 状态标识

| 状态 | 图标 | 说明 |
|------|------|------|
| 进行中 | 🔄 | 当前活跃 Feature |
| 已完成 | ✔️ | 终态：成功 |
| 已取消 | ❌ | 终态：取消 |

## 排序规则

**默认排序**: 按更新时间倒序（最新的在前）

**可选排序**:
1. 按创建时间
2. 按 Feature ID
3. 按阶段
4. 按状态
