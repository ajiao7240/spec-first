# Findings Schema

findings.md 标准格式，用于记录计划摘要、决策依据和执行证据。

## 标准格式

```markdown
# Findings & Decisions

## 计划摘要

| 字段 | 值 |
|------|-----|
| **目标阶段** | [00_init/01_specify/02_design/03_plan/04_implement/05_verify/06_wrap_up/07_release/08_done/09_cancelled] |
| **下一步动作** | [具体动作描述] |
| **阻塞项** | [当前阻塞的列表] |
| **风险等级** | [LOW/MEDIUM/HIGH/CRITICAL] |
| **建议命令** | [具体 CLI 命令] |

## 阶段状态

- **当前阶段**: [阶段名称]
- **阶段进度**: [百分比]
- **产物完整性**: [评估结果]
- **覆盖率状态**: [C1-C9 指标]

## 风险评估

| 风险项 | 等级 | 影响 | 缓解措施 |
|--------|------|------|----------|
| [风险描述] | [LOW/MEDIUM/HIGH/CRITICAL] | [影响范围] | [具体措施] |

## 下一步

1. [具体步骤 1]
2. [具体步骤 2]
3. [具体步骤 3]

## 研究发现

<!-- 调研/分析过程中发现的关键信息 -->

## 技术决策

<!-- 做出的技术选择及理由 -->

| 决策 | 理由 |
|------|------|
|      |      |

## 遇到的问题

<!-- 阻塞和问题及解决方案 -->

| 问题 | 解决方案 |
|------|----------|
|      |          |
```

## 字段定义

### 计划摘要字段

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| **目标阶段** | enum | 下一步要到达的阶段 | `01_specify` |
| **下一步动作** | string | 具体要执行的动作 | "生成 FR 定义" |
| **阻塞项** | array | 当前阻塞的问题列表 | `["AC-AUTH-001-01 歧义"]` |
| **风险等级** | enum | 整体风险等级 | `MEDIUM` |
| **建议命令** | string | 建议执行的 CLI 命令 | `/spec-first:spec` |

### 风险等级定义

| 等级 | 色标 | 说明 | 行动 |
|------|------|------|------|
| **LOW** | 🟢 | 风险可控，按计划推进 | 继续执行 |
| **MEDIUM** | 🟡 | 存在风险，需要关注 | 监控风险 |
| **HIGH** | 🟠 | 风险较高，需要缓解 | 制定缓解措施 |
| **CRITICAL** | 🔴 | 风险严重，可能阻断 | 必须解决才能继续 |

### 阶段状态

| 字段 | 说明 | 示例 |
|------|------|------|
| **当前阶段** | Feature 当前所处的阶段 | `01_specify` |
| **阶段进度** | 当前阶段的完成百分比 | `50%` |
| **产物完整性** | 当前阶段产物的完整程度 | "spec.md 已生成，AC 待补充" |
| **覆盖率状态** | C1-C9 覆盖率指标 | "C1=100%, C2=0%" |

## 与 orchestrate 的协同

### plan 产出

plan skill 产出计划摘要，写入 `findings.md`：

```markdown
## 计划摘要

| 字段 | 值 |
|------|-----|
| **目标阶段** | 01_specify |
| **下一步动作** | 生成 FR 定义 |
| **阻塞项** | 无 |
| **风险等级** | LOW |
| **建议命令** | /spec-first:spec |
```

### orchestrate 消费

orchestrate skill 读取计划摘要，作为编排输入：

1. 读取 `findings.md` 中的计划摘要
2. 根据 `下一步动作` 调度对应 skill
3. 根据 `风险等级` 决定确认策略
4. 根据 `建议命令` 生成执行序列
5. 执行完成后更新 `findings.md` 中的阶段状态

## 更新频率

| 场景 | 更新内容 |
|------|----------|
| **plan 执行后** | 计划摘要、风险评估、下一步 |
| **orchestrate 执行后** | 阶段状态、覆盖率状态、执行证据 |
| **发现问题后** | 遇到的问题、新的风险评估 |
| **做出决策后** | 技术决策、研究发现 |

## 2-Action Rule

每完成 2 个关键动作后，必须更新 `findings.md`：

- plan: 生成计划、识别风险、确认下一步
- orchestrate: 执行 skill、更新状态、检查阻塞
- research: 查阅资料、对比方案、得出结论

**最小落盘字段**：
- 当前结论
- 证据路径
- 下一步
