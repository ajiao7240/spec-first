# Research Checklist

调研质量检查清单，用于在输出 `research.md` 前自检。

## A. 调研目标（必查）

- [ ] 调研目标清晰、可衡量
- [ ] 调研范围明确（IN/OUT of scope）
- [ ] 调研结论可追溯到具体需求/设计决策

## B. 候选方案（必查）

- [ ] 至少 2 个候选方案
- [ ] 每个方案有明确名称/来源
- [ ] 排除方案有说明理由

## C. 对比维度（必查）

- [ ] 成本对比（直接成本 + 间接成本）
- [ ] 风险评估（技术风险、业务风险）
- [ ] 成熟度评估（社区、文档、案例）
- [ ] 生态完整性（依赖、工具、集成）

## D. 证据质量（必查）

- [ ] 每个结论有证据支撑
- [ ] 证据来源可追溯（链接/文档路径）
- [ ] 未验证假设已标记 `[NEEDS VERIFICATION][TYPE]`

## E. 推荐结论（必查）

- [ ] 推荐方案明确
- [ ] 推荐理由充分（基于对比分析）
- [ ] 风险与依赖已声明
- [ ] 下一步动作可执行

## F. 输出格式（必查）

- [ ] 使用统一的 research.md 模板
- [ ] 对比矩阵格式一致
- [ ] 术语与项目其他文档一致
- [ ] findings.md 已同步更新

## G. 特定场景检查（按需）

### G1. 性能敏感场景

- [ ] 包含 Benchmark 数据
- [ ] 测试环境与生产环境可比
- [ ] 边界条件已测试

### G2. 安全敏感场景

- [ ] 包含安全评估
- [ ] 已查阅安全白皮书/漏洞库
- [ ] 合规要求已考虑

### G3. 成本敏感场景

- [ ] TCO（总拥有成本）已计算
- [ ] 隐藏成本已识别
- [ ] 扩展成本已评估

### G4. 迁移/替换场景

- [ ] 迁移成本已评估
- [ ] 回滚方案已考虑
- [ ] 兼容性已验证

## H. 常见陷阱（自检）

- [ ] 无"我觉得"这类主观表述（替代为"根据 X 文档"）
- [ ] 无"大概"、"可能"等模糊词汇（替代为具体数值/范围）
- [ ] 无单一来源依赖（至少 2 个独立证据源）
- [ ] 无忽视反面证据的情况
- [ ] 无混淆营销话术与事实的情况
