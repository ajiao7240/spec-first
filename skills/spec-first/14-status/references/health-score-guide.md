# Health Score Guide

健康分计算与解读指南。

## 健康分定义

健康分是 Feature 整体状况的量化评估，范围 0-100 分，综合考虑覆盖率、质量、进度和风险。

## 分数段定义

| 分数段 | 等级 | 图标 | 说明 | 行动建议 |
|--------|------|------|------|----------|
| 90-100 | 优秀 | 🟢 | 状态良好，可推进 | 继续保持，准备推进阶段 |
| 70-89 | 良好 | 🟡 | 存在改进空间 | 关注风险项，优化覆盖率 |
| 50-69 | 中等 | 🟠 | 存在明显问题 | 优先修复，暂缓推进 |
| 0-49 | 较差 | 🔴 | 严重问题 | 立即处理，停止推进 |

## 计算公式

```
健康分 = 覆盖率完整性 × 40% + 质量门禁 × 30% + 任务进度 × 20% + 风险控制 × 10%
```

### 维度权重说明

| 维度 | 权重 | 理由 |
|------|------|------|
| 覆盖率完整性 | 40% | 最重要，决定功能完整性 |
| 质量门禁 | 30% | 次重要，决定质量保障 |
| 任务进度 | 20% | 重要，决定交付时间 |
| 风险控制 | 10% | 辅助，决定潜在问题 |

---

## 维度 1: 覆盖率完整性（40%）

### 计算方式

```
覆盖率完整性 = (达标指标数 / 总指标数) × 100
```

### 指标达标判定

| 指标 | 达标条件 | 得分 |
|------|----------|------|
| C1 | ≥ 阈值 | 100 |
| C2 | ≥ 阈值 | 100 |
| C3 | ≥ 阈值 | 100 |
| C4 | ≥ 阈值 | 100 |
| C5 | ≥ 阈值 | 100 |
| C6 | ≥ 阈值 | 100 |
| C7 | ≥ 阈值 | 100 |
| C8 | ≥ 阈值 | 100 |
| C9 | ≥ 阈值 | 100 |

### 示例

```
C1=100%, C2=100%, C3=80%, C4=85%, C5=92%, C6=60%

达标指标: C1, C2, C4, C5, C6 (5 个)
总指标: 6 个（当前阶段相关）

覆盖率完整性 = (5 / 6) × 100 = 83.3 分
```

---

## 维度 2: 质量门禁（30%）

### 计算方式

```
质量门禁 = (通过条件数 / 总条件数) × 100
```

### Gate 状态映射

| Gate 状态 | 得分 |
|-----------|------|
| PASS | 100 |
| PASS_WITH_WAIVER | 80 |
| FAIL | 0 |

### 示例

```
当前阶段: 03_plan
Gate 条件: G-PLAN-01, G-PLAN-02, G-PLAN-03

G-PLAN-01: PASS (100)
G-PLAN-02: PASS_WITH_WAIVER (80)
G-PLAN-03: FAIL (0)

质量门禁 = (100 + 80 + 0) / 3 = 60 分
```

---

## 维度 3: 任务进度（20%）

### 计算方式

```
任务进度 = (完成任务数 / 总任务数) × 100
```

### 任务状态权重

| 状态 | 权重 |
|------|------|
| complete | 100% |
| verified | 100% |
| in_progress | 50% |
| planned | 0% |
| blocked | 0% |

### 示例

```
总任务: 10 个
complete: 6 个
in_progress: 2 个
planned: 2 个

完成任务数 = 6 + 2 × 50% = 7
任务进度 = (7 / 10) × 100 = 70 分
```

---

## 维度 4: 风险控制（10%）

### 计算方式

```
风险控制 = 100 - (高风险数 × 30 + 中风险数 × 10 + 低风险数 × 5)
```

### 风险扣分规则

| 风险等级 | 扣分 |
|---------|------|
| 🔴 高风险 | -30 分/个 |
| 🟡 中风险 | -10 分/个 |
| 🟢 低风险 | -5 分/个 |

### 示例

```
高风险: 0 个
中风险: 2 个
低风险: 1 个

风险控制 = 100 - (0 × 30 + 2 × 10 + 1 × 5) = 75 分
```

## 综合示例

### 场景 1: 优秀状态（95 分）

```
覆盖率完整性: 100 分（所有指标达标）
质量门禁: 100 分（所有条件 PASS）
任务进度: 90 分（90% 完成）
风险控制: 95 分（1 个低风险）

健康分 = 100 × 40% + 100 × 30% + 90 × 20% + 95 × 10%
       = 40 + 30 + 18 + 9.5
       = 97.5 ≈ 98 分 (🟢 优秀)
```

### 场景 2: 良好状态（78 分）

```
覆盖率完整性: 83 分（5/6 达标）
质量门禁: 80 分（1 个 WAIVER）
任务进度: 70 分（70% 完成）
风险控制: 80 分（2 个中风险）

健康分 = 83 × 40% + 80 × 30% + 70 × 20% + 80 × 10%
       = 33.2 + 24 + 14 + 8
       = 79.2 ≈ 79 分 (🟡 良好)
```

### 场景 3: 中等状态（58 分）

```
覆盖率完整性: 50 分（3/6 达标）
质量门禁: 67 分（1 个 FAIL）
任务进度: 60 分（60% 完成）
风险控制: 70 分（3 个中风险）

健康分 = 50 × 40% + 67 × 30% + 60 × 20% + 70 × 10%
       = 20 + 20.1 + 12 + 7
       = 59.1 ≈ 59 分 (🟠 中等)
```

### 场景 4: 较差状态（35 分）

```
覆盖率完整性: 33 分（2/6 达标）
质量门禁: 33 分（2 个 FAIL）
任务进度: 40 分（40% 完成）
风险控制: 40 分（2 个高风险）

健康分 = 33 × 40% + 33 × 30% + 40 × 20% + 40 × 10%
       = 13.2 + 9.9 + 8 + 4
       = 35.1 ≈ 35 分 (🔴 较差)
```

## 健康分解读

### 90-100 分（优秀）

**特征**:
- 所有覆盖率指标达标
- Gate 条件全部通过
- 任务完成率 > 80%
- 无高风险，最多 1 个低风险

**建议**:
- ✅ 可以推进到下一阶段
- 继续保持当前节奏
- 准备阶段验收

### 70-89 分（良好）

**特征**:
- 大部分覆盖率指标达标
- Gate 条件部分通过或有 WAIVER
- 任务完成率 60-80%
- 无高风险，有少量中低风险

**建议**:
- ⚠️ 关注未达标指标
- 优化覆盖率
- 解决中风险项
- 评估是否可推进

### 50-69 分（中等）

**特征**:
- 部分覆盖率指标未达标
- Gate 条件有失败项
- 任务完成率 40-60%
- 可能有高风险

**建议**:
- ❌ 暂缓推进
- 优先修复失败项
- 补充覆盖率
- 解决高风险

### 0-49 分（较差）

**特征**:
- 多数覆盖率指标未达标
- Gate 条件多数失败
- 任务完成率 < 40%
- 存在多个高风险

**建议**:
- 🚫 停止推进
- 立即处理高风险
- 重新评估计划
- 考虑回退阶段
