# 质量门禁

本文档详细说明 spec skill 的质量门禁规则。

---

## Phase 0.4: PRD 自检门禁

**目标**: 确保 PRD 质量达标。

**评分标准**: C-PRD >= 85%

### C-PRD 评分规则

**必需章节**（各 20%）:
- 1. 业务目标
- 2. 功能需求
- 3. 非功能需求
- 4. 开放问题
- 5. 术语表

**评分公式**:
```
C-PRD = (章节完整性 × 40%) + (内容清晰度 × 30%) + (可验证性 × 30%)
```

**章节完整性**（40%）:
- 缺失必需章节: -20% / 章节
- 章节内容为空: -10% / 章节

**内容清晰度**（30%）:
- 存在模糊术语（"可能"/"大概"/"一般"）: -5% / 处
- 存在未定义术语: -5% / 处
- 存在多解语义: -10% / 处

**可验证性**（30%）:
- 缺少可量化指标: -10% / 项
- 缺少验收标准: -10% / 项
- 缺少边界定义: -5% / 项

### 门禁规则

- **C-PRD >= 85%**: 通过，进入 Phase 0.5
- **C-PRD < 85%**: 阻断，返回 Phase 0.3 修正

---

## Phase 0.2: 质量扫描门禁

**目标**: 确保输入信息质量达标。

**评分标准**: 初始质量评分

### 质量评分规则

```
初始评分 = (已明确项 / 必需项总数) × 100%
必需项: 业务目标(25%) + 功能边界(25%) + 约束条件(25%) + 成功标准(25%)
```

### 门禁规则

- **评分 < 40%**: 阻断，要求用户补充核心信息
- **评分 40-70%**: 警告，标记缺失项但允许继续
- **评分 > 70%**: 通过，进入 Phase 0.3

---

## Step 8: Gate Check 门禁

**目标**: 确保 spec 阶段所有产物完整且质量达标。

**执行命令**:
```bash
spec-first gate check <featureId>
```

### 检查项

1. **PRD 存在且 C-PRD >= 85%**
   - 文件路径: `specs/{featureId}/prd.md`
   - 评分标准: 见 Phase 0.4 门禁

2. **spec.md 存在且包含所有 FR/AC**
   - 文件路径: `specs/{featureId}/spec.md`
   - 必含内容: FR 列表 + AC 列表

3. **FR 已注册到追溯矩阵**
   - 文件路径: `specs/{featureId}/traceability-matrix.md`
   - 检查: 所有 FR ID 已注册

4. **spec-review.md 存在**（如需要）
   - 文件路径: `specs/{featureId}/spec-review.md`
   - 触发条件: 复杂度为 Moderate 或 Complex

### 失败处理

| 失败原因 | 处理方式 |
|---------|---------|
| C-PRD < 85% | 返回 Phase 0.3 修正 PRD，重新执行 Phase 0.4-0.6 |
| 缺少文件 | 补齐后重新执行 gate check |
| FR 未注册 | 执行注册后重新执行 gate check |

**阻断规则**: 不得跳过 gate check 进入下一阶段

### 通过后操作

1. 更新 `stage-state.json` 为 `02_design`
2. 记录 gate check 通过时间到 `findings.md`
3. 提示用户："Gate check 通过，可执行 /spec-first:design"

## 格式校验

**执行命令**:
```bash
spec-first validate format <featureId>
```

### 检查项

1. **PRD 章节格式**
   - 必含: `## 1. 业务目标` / `## 2. 功能需求` / `## 3. 非功能需求`

2. **ID 格式**
   - 无多余连字符
   - 符合规范: `FR-<ABBR>-<NNN>`

3. **文件路径完整性**
   - 所有引用文件存在

4. **必需字段**
   - Feature ID 存在

### 失败处理

- 校验失败时阻断
- 需修复后重新确认

---

## 门禁执行顺序

在 spec skill 中，门禁按以下顺序执行：

1. **Phase 0.2**: 质量扫描门禁（初始质量评分）
2. **Phase 0.4**: PRD 自检门禁（C-PRD >= 85%）
3. **Step 8**: Gate Check 门禁（所有产物完整性）
4. **Step 8**: 格式校验（自动执行）

所有门禁必须通过才能进入下一阶段。
