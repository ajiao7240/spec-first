## 深度分析: Phase 编号的语义

### 编号系统设计意图

**Phase 编号表达了分支逻辑**:

| 编号 | 类型 | 含义 |
|------|------|------|
| **整数 (0, 1, 2, 3)** | 主阶段 | 流程的主要里程碑 |
| **小数 (0.5, 2.5)** | 决策点 | 用户选择分支 |
| **小数 (2.8)** | 输出点 | 直接展示结果（可跳过确认） |

### 分支流程图

```
Phase 0 ── Phase 0.5 (决策)
              │
              ├── 快速模式 ── Phase 2.8 ── Phase 3
              │
              └── 完整模式 ── Phase 1 ── Phase 2 ── Phase 2.5 ── Phase 2.8 ── Phase 3
```

### 与 spec skill 对比

| 维度 | spec skill | onboarding skill |
|------|-----------|-----------------|
| **子阶段** | 连续 (0.0→0.6) | 跳跃 (0.5, 2.5, 2.8) |
| **风格** | 细粒度控制 | 分支决策 |
| **优点** | 精确 | 灵活 |

---

## 结论

**Phase 编号是有意设计的**，表达了:
- 分支决策点 (0.5, 2.5)
- 可跳过的阶段（快速模式）

**保持现有编号，无需修改。** ✅
