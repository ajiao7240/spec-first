# Subagent-Driven 架构设计

> 本文档描述 00-first skill 采用 Subagent-Driven 模式的架构设计方案。

---

## 设计目标

- **并行执行**：多个子 agent 同时工作，减少总执行时间
- **上下文隔离**：每个 agent 专注单一任务，避免主 context 爆炸
- **可观测性**：每个任务进度独立追踪，调试友好
- **容错性**：单个 agent 失败不阻塞其他 agent

---

## 架构概览

                              │
┌─────────────────────────────────────────────────────────────┐
│ P1a: 主线程 - 快速技术栈识别 (串行，<30秒)                        │
└─────────────────────────────────────────────────────────────┘
                              │
              ┌───────────────┴───────────────┐
              ▼                               ▼
┌─────────────────────────┐     ┌─────────────────────────┐
│ 第一波 Sub-Agents        │     │ P1b: Context7 收集     │
│ (并行，单 Agent 60s)      │     │ (与第一波并行)           │
├─────────────────────────┤     ├─────────────────────────┤
│ • A1-1: 扫描后端目录     │     │ • 映射 ESLint 规则      │
│ • A1-2: 扫描前端目录     │     │ • 映射 Prettier 规则    │
│ • A3-1: 后端调用链分析   │     │ • 映射框架最佳实践      │
│ • A3-2: 前端调用链分析   │     └─────────────────────────┘
│ • B-1: 扫描后端 API      │
│ • B-2: 扫描前端 API      │
│ • C1: 外部依赖扫描       │
│ • D: DB 检测（如有）     │
└─────────────────────────┘
             │
             ├─ 等待 A1 模块清单 → A2
             │
             ├─ 等待 P1b + C1 → C2-1~6 → C2-7

┌─────────────────────────────────────────────────────────────┐
│ 第二波 Sub-Agents                                            │
├─────────────────────────────────────────────────────────────┤

             ▼

└─────────────────────────────────────────────────────────────┘

## Agent 拆分策略

### 评估矩阵

| Agent | 并行潜力 | 复杂度 | 推荐拆分 | 拆分方案 |
|-------|----------|--------|----------|----------|
| **A1: codebase-overview** | ⭐⭐⭐⭐⭐ | 低 | ✅ 是 | 按前后端拆分：A1-Backend + A1-Frontend |
| **A2: architecture** | ⭐⭐⭐ | 中 | ❌ 否 | 依赖 A1，逻辑简单，保持单 Agent |
| **A3: call-graph** | ⭐⭐⭐⭐⭐ | 高 | ✅ 是 | 按前后端拆分：A3-Backend + A3-Frontend |
| **B: api-docs** | ⭐⭐⭐⭐⭐ | 低 | ✅ 是 | 按前后端拆分：B-Backend + B-Frontend |
| **C1: external-deps** | ⭐⭐ | 低 | ❌ 否 | 配置扫描，并行收益小 |
| **C2: dev-guidelines** | ⭐⭐⭐⭐ | 中 | ✅ 是 | 按模块拆分：C2-1~6 并行分析 |
| **D: database-er** | ⭐⭐ | 低 | ❌ 否 | DB 连接单点，无法并行 |
| **A4: domain-model** | ⭐⭐⭐⭐ | 中 | ✅ 是 | 已实现，4 个子任务并行 |

### 拆分原则

| 原则 | 说明 |
|------|------|
| **按模块拆分** | 优先按前后端/语言拆分，而非按步骤 |
| **独立产出** | 每个 sub-agent 产出独立的 JSON 结果 |
| **轻量传递** | 使用 JSON 上下文传递，避免文件 I/O |
| **降级保证** | sub-agent 失败时主线程可降级执行 |

## 数据流设计

### 中间数据格式

#### A1 模块清单（A1-1/A1-2 → A2/A4）

```json
{
  "modules": [
    {
      "name": "auth",
      "path": "backend/app/auth/",
      "responsibility": "认证与授权",
      "type": "backend"
    },
    {
      "name": "admin",
      "path": "frontend/admin/",
      "responsibility": "管理后台",
      "type": "frontend"
    }
  ]
}
```

#### API 文档片段（B-1/B-2 → A4）

```json
{
  "apis": [
    {
      "method": "POST",
      "path": "/api/users",
      "description": "创建用户",
      "file": "backend/app/api/admin/users.py",
      "line": 76
    }
  ]
}
```

#### DB Schema（D → A2/A4）

```json
{
  "tables": [
    {
      "name": "users",
      "columns": [...],
      "foreign_keys": [...]
    }
  ]
}
```

### 上下文传递

```
主线程
  ├─ tech_stack (P1a)
  ├─ context7_mappings (P1b)
  ├─ module_list (A1-1 + A1-2 合并)
  ├─ api_docs (B-1 + B-2 合并)
  ├─ external_deps (C1)
  └─ db_schema (D)

↓ 传递给后续 sub-agents

A2: 需要 tech_stack + module_list（如有 db_schema 则补充）
A3: 需要 tech_stack + module_list
A4: 需要 tech_stack + module_list + api_docs + db_schema
C2: 需要 context7_mappings + external_deps
```

## 超时与错误处理

### 超时策略

| 阶段 | 单 Agent 超时 | 总阶段超时 |
|------|---------------|-----------|
| 第一波 | 60s | 120s |
| 第二波 | 60s | 120s |
| 第三波 | 60s | 120s |

整体并行阶段最大超时：**300s**（与 `SKILL.md` 保持一致）。

### 错误处理

```
sub-agent 执行
  │
  ├─ 成功 → 收集结果
  │
  ├─ 超时 → 标记 [超时]，尝试降级或跳过
  │
  ├─ 失败 → 标记 [失败]，主线程降级执行
  │
  └─ 部分成功 → 使用已生成部分，标记 [部分完成]
```

### 降级示例

```
A3-Backend (LSP) 超时
  ↓ 降级
A3-Backend (静态 import 扫描)
  ↓ 输出
call-graph.md 标注 [依赖分析: 静态模式，LSP 超时]
```
