# 端类型产物映射配置

> 定义每种端类型应生成的产物列表，用于 quick/deep 模式的产物组合决策。
> **版本**: v1.0.0 | **更新**: {{DATE}}

---

## 映射规则

### 产物标记说明

| 标记 | 说明 |
|------|------|
| `[quick]` | quick 模式默认生成 |
| `[deep]` | 仅 deep 模式生成 |
| `[条件]` | 按条件生成（如检测到 DB） |
| `[可选]` | 可选产物 |

---

## 1. 后台服务（backend）

### quick 模式（5 个产物）

| 产物 | 状态 | 说明 |
|------|------|------|
| `tech-stack.md` | ✅ 必须 | 语言、框架、中间件 |
| `codebase-overview.md` | ✅ 必须 | 目录结构、模块划分 |

| `api-docs.md` | ✅ 必须 | 端点、请求/响应 |
| `database-er.md` | ✅ 条件 | 检测到 DB 时生成 |

### deep 模式追加（6 个产物）

| 产物 | 状态 | 说明 |
|------|------|------|
| `call-graph.md` | ✅ 必须 | 模块调用链 |
| `architecture.md` | ✅ 必须 | 分层架构、中间件 |
| `external-deps.md` | ✅ 必须 | 外部服务、依赖 |
| `local-setup.md` | ✅ 必须 | 环境搭建 |

**总计**：quick 模式 4-5 个，deep 模式 10-11 个

## 6. Monorepo

### quick 模式（按子包类型）

| 策略 | 说明 |
|------|------|
| **根级产物** | `tech-stack.md`（整体技术栈）、`README.md`（子包导航） |
| **子包产物** | 按各子包的端类型生成对应产物 |

**示例**：
```
monorepo/
├── packages/
│   ├── backend/    → 按 backend 类型生成产物
│   ├── admin/      → 按 frontend(Admin) 类型生成产物
│   └── mobile/      → 按 mobile 类型生成产物
├── docs/first/
│   ├── tech-stack.md         # 根级：整体技术栈
│   ├── backend/              # backend 子包产物
│   │   ├── codebase-overview.md
│   │   ├── domain-model.md
│   │   └── ...
│   ├── admin/                # admin 子包产物
│   │   ├── codebase-overview.md
│   │   └── ...
│   └── README.md             # 索引：指向各子包产物
```

### deep 模式

根级 + 子包的完整产物集。
