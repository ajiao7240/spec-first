## 1. spec-first-Requirements-Analysis (需求分析)

**定位**：将原始业务意图（Intent）解构为可规格化的逻辑实体。

| 维度 | 英文定义 | 说明 |
| --- | --- | --- |
| **核心目标** | **Logic Deconstruction** | 逻辑解构：剥离视觉表现，专注业务规则。 |
| **输入** | **Business Intent / User Pain-points** | 业务意图与用户痛点。 |
| **关键动作** | **Entity Modeling & State Mapping** | 实体建模与状态映射：定义系统中有哪些“物”，它们如何流转。 |
| **产出物** | **Domain Model & Logic Flow** | 领域模型与逻辑流程图。 |

## 2. spec-first-Requirements-PRD (需求结构化文档)

**定位**：一份“机器可读、人读无歧义”的需求契约，通常称为 **Structured Specification**。

### 📄 标准模板组成 (Template Components)

#### A. User Stories (As-I-So)

* **Definition**: Granular functional units defined from the user perspective.
* **Format**: *As a [User Role], I want to [Action], so that [Value/Goal].*

#### B. Acceptance Criteria (AC - Given-When-Then)

这是 Spec-First 的灵魂，将需求转化为测试的前置条件。

* **Scenario**: User initiates a high-value transaction.
* **Given**: The user has a verified account and balance > $1000.
* **When**: The user submits a transfer of $500.
* **Then**: The system should trigger MFA (Multi-Factor Authentication).

## 汇总：全链路节点英文化对照表

为了方便你在工具链（如 GitLab/Jira）中使用，建议采用以下标准命名：

| 原始阶段 | Spec-First 节点名称 (Standard English) | 关键产出物 (Key Deliverables) |
| --- | --- | --- |
| **需求分析** | `spec-first-Requirements-Analysis` | Domain Model, Logic Flow |
| **需求文档** | `spec-first-Structured-PRD` | AC (Acceptance Criteria), User Stories |
| **需求评审** | `spec-first-Requirements-Review` | DoR (Definition of Ready) Sign-off |
| **技术方案** | `spec-first-Technical-Design` | ADR (Architecture Decision Records) |
| **方案评审** | `spec-first-Design-Compliance-Review` | DCR Report, Baseline Locking |
| **任务拆解** | `spec-first-Task-Decomposition` | WBS (Work Breakdown Structure) |
| **任务评审** | `spec-first-Task-Review` | Estimated Velocity, Dependency Map |
| **根据任务开发** | `spec-first-Spec-Driven-Dev` | Code Implementation |
| **代码评审** | `spec-first-Spec-Code-CR` | Contract Consistency Audit |
| **需求变更** | `spec-first-Change-Management` | RFC (Request for Change), Impact Analysis |
| **初始化** | `spec-first-Project-Scaffolding` | Auto-generated SDKs/Boilerplate |

### 🚀 进阶建议
