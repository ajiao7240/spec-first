# Tech Comparison Template

技术对比标准模板，用于确保对比分析的结构化和完整性。

## 对比矩阵结构

```markdown
| 维度 | 方案 A | 方案 B | 方案 C |
|------|--------|--------|--------|
| 成本 | ¥X/月 | ¥Y/月 | ¥Z/月 |
| 成熟度 | ⭐⭐⭐⭐ | ⭐⭐⭐ | ⭐⭐ |
| 生态 | 丰富 | 中等 | 有限 |
| 学习曲线 | 低 | 中 | 高 |
| 风险 | 低 | 中 | 高 |
```

## 标准维度定义

### 1. 成本（Cost）

| 子维度 | 说明 | 示例指标 |
|--------|------|----------|
| 直接成本 | 许可费、订阅费 | ¥/月、$/user |
| 间接成本 | 学习成本、迁移成本 | 人天、工时 |
| 基础设施 | 服务器、存储 | CPU、内存 |
| 运维成本 | 监控、备份 | 人时/月 |

### 2. 风险（Risk）

| 子维度 | 说明 | 评估方式 |
|--------|------|----------|
| 技术风险 | 技术成熟度、社区活跃度 | GitHub stars、issue 响应时间 |
| 业务风险 | 厂商锁定、服务稳定性 | SLA、厂商历史 |
| 安全风险 | 已知漏洞、安全审计 | CVE 数量、安全报告 |
| 合规风险 | 数据处理、隐私保护 | GDPR、SOC2 认证 |

### 3. 成熟度（Maturity）

| 等级 | 标准 | 判断依据 |
|------|------|----------|
| ⭐⭐⭐⭐⭐ | 行业标准，大规模验证 | 500强企业广泛使用 |
| ⭐⭐⭐⭐ | 成熟稳定，案例丰富 | 多个行业标杆案例 |
| ⭐⭐⭐ | 基本可用，案例适中 | 有公开案例但不多 |
| ⭐⭐ | 早期阶段，谨慎使用 | 缺乏生产环境案例 |
| ⭐ | 实验性质，不建议生产 | 原型/概念验证 |

### 4. 生态（Ecosystem）

| 子维度 | 说明 | 示例指标 |
|--------|------|----------|
| 文档质量 | 官方文档完整性 | API 文档、教程、示例 |
| 社区活跃度 | 开发者参与度 | StackOverflow 问题数、响应速度 |
| 工具链 | 开发/运维工具支持 | IDE 插件、CI 集成 |
| 扩展能力 | 插件/扩展机制 | 插件市场、API 开放度 |

### 5. 学习曲线（Learning Curve）

| 等级 | 描述 | 评估时间 |
|------|------|----------|
| 低 | 概念简单，上手快 | < 1 天 |
| 中 | 需要培训，但有资源 | 1-3 天 |
| 高 | 复杂概念，需要专家 | > 1 周 |

## 风险等级定义

| 等级 | 色标 | 说明 |
|------|------|------|
| 低 | 🟢 | 可接受，无需特殊措施 |
| 中 | 🟡 | 需要缓解措施，定期回顾 |
| 高 | 🔴 | 需要详细预案，可能拒绝 |

## 证据要求

| 结论类型 | 必需证据 | 证据强度 |
|---------|---------|----------|
| 性能声明 | Benchmark 数据 | 🔴 强 |
| 安全声明 | 安全审计报告/官方文档 | 🔴 强 |
| 成本声明 | 官方报价单 | 🔴 强 |
| 成熟度声明 | 用户案例/社区数据 | 🟡 中 |
| 易用性声明 | 文档/教程质量 | 🟢 弱 |
