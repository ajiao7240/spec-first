进入 $spec-work，以完整实现为目标，但按可验证闭环顺序落地，不做范围裁剪。
  总原则
  - 目标不是“多接 GitNexus”，而是提升 AI coding 的上下文质量和证据质量。
  - review-pre-facts 只产出 deterministic facts；LLM 仍决定 scope、finding、root cause、task ordering。
  - 每个 operation 必须端到端完成：query-plan -> raw result -> normalization -> render -> summary -> tests。不能只改 buildQueryPlan。
  - 当前合同里的 current implementation boundary 必须保留，直到代码和测试证明能力已实现。
  建议执行顺序
  1. 先做 implementation reality sync
     重新核对：
      - review-pre-facts.js 当前只支持 doc-review/code-review + gitnexus.query
      - 技术方案和计划中所有“已支持”表述不能误导
      - contracts 已经区分 current / target capability
        验证：现有 review-pre-facts-helper.test.js 全绿。
  2. 做 context 端到端闭环
     这是第一条非 query 深能力，风险最低、价值高。
     实现：
      - buildQueryPlan 能为明确 symbol target 产出 gitnexus.context
      - ambiguous symbol degrade，不让脚本猜
      - raw result 校验 tool_name/operation/query_id
      - normalization 输出 fact_kind=context_symbol
      - render 输出 source-read pointer / relationship summary
      - summary 记录 capabilities_used / operation_counts
        验证：
      - query 旧行为不变
      - context plan 生成
      - context raw normalize
      - stale snapshot downgrade
      - unsafe path / oversized output fail closed
  3. 做 impact 端到端闭环
     只做 summary-first，不落完整 blast-radius dump。
     实现：
      - argument 必须有 target、direction
      - maxDepth、timeoutMs、结果大小有上限
      - normalize 成 fact_kind=impact_summary
      - 输出 affected module/process counts、by-depth counts、source_reads_required
      - full byDepth 不进 durable artifact
        验证：
      - oversized impact raw 触发 truncate/degrade
      - impact 不能扩大 review/work scope
      - finding 仍需 source/diff/test confirmation
  4. 做 detect_changes 端到端闭环
     重点是 diff scope 和 raw diff 安全。
     实现：
      - scope 必须显式：staged/unstaged/all/compare
      - compare 必须有 base_ref
      - normalize 成 fact_kind=detect_changes_summary
      - durable output 只保留 changed symbols / affected processes summary
      - raw diff 永不进入 durable docs
        验证：
      - raw diff redaction
      - scope mismatch degrade
      - affected processes 只作为 test/source-read candidate
  5. 扩 workflow-neutral rendering
     等三类非 query facts 稳定后，再启用 plan/debug profile。
     要求：
      - 不出现 Coverage / finding / reviewer wording
      - 输出 capabilities_used、key pointers、source_reads_required、limitations
      - debug root cause 必须由 reproduction/source/log/test 关闭
  6. 做 workflow-native session evidence envelope
     对 route_map/api_impact/shape_check/tool_map/cypher 不进 helper。
     实现统一 envelope：
      - capability
      - lane
      - tool_or_resource
      - arguments_or_uri
      - repo_scope
      - task_domain
      - provenance
      - freshness/readiness
      - summary
      - source_reads_required
      - limitations
      - redaction_status
        验证：session evidence 不写 canonical readiness，不替代 finding/root cause。
  7. 做 workspace/resource lane
     支持 repo/group resources 和 group-aware query/context/impact 的证据边界。
     核心限制：
      - 只能 orientation
      - 不能选择写入 repo
      - 不能扩大 scope
      - cross-repo claim 需要 source confirmation
  8. 做 utilization / Evaluation Harness
     不用 adoption 数量伪装效果，记录是否真的帮助决策。
     建议指标：
      - capabilities_used
      - operation_counts
      - degraded_reason_counts
      - source_reads_required_count
      - graph_findings_applied
      - graph_findings_rejected_after_source_read
      - graph_to_finding / graph_to_decision 摘要
      - spec-work：用 graph evidence 聚焦 source reads/tests，closeout 披露风险
      - spec-compound/refresh：只沉淀 source-confirmed graph-informed learning
  10. 最后处理 startup readiness snapshot
     只注入短 snapshot，不做治理逻辑：
  - query_ready
  - freshness
  - dirty/stale
  - available capability summary
  - limitations
    Hook 是 delivery surface，不是 source of truth。
  不要做
  - 不把 route_map/api_impact/shape_check/cypher 塞进 review-pre-facts-query-plan
  - 不让 GitNexus 决定 scope/finding/root cause
  - 不写 raw provider output 到 durable docs
  - 不运行 group_sync/rename/analyze/repair 于普通 workflow
  - 不手改 generated runtime mirrors
  验收口径
  完成后，至少要能证明：
  - review-pre-facts 能产出 query/context/impact/detect_changes 四类 bounded facts
  - 非 query facts 不伪造成 excerpt
  - stale/dirty/provider unavailable 都能 degraded
  - graph evidence 能帮助 plan/review/debug/work/task/knowledge，但不拥有 authority
  - tests 覆盖 helper、contracts、schema、downstream consumption
  - npm run typecheck、聚焦 unit tests、git diff --check 通过
