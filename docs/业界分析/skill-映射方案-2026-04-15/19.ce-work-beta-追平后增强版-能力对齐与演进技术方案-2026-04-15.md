# spec-work-beta 完整方案文档：`ce-work-beta` 基线对齐与可演进增强（2026-04-15）

## 文档定位

本文基于 `ce-work-beta` / `spec-work-beta` 主 skill、delegation reference 与 shipping workflow 的真实 diff 写成。

## Part A：上游基线与代码事实

### A.1 核对范围

- source:
  - `plugins/compound-engineering/skills/ce-work-beta/`
- target:
  - `skills/spec-work-beta/`
- 邻接：
  - `skills/spec-work/`
  - `skills/spec-review/`
  - `.spec-first/config.local.yaml` / `.compound-engineering/config.local.yaml` 路径约定

### A.2 关键差异

#### A.2.1 基础定位从 `ce:work-beta` 迁移为 `spec:work-beta`

这是正常 rename，不构成能力差距。

#### A.2.2 `spec-work-beta` 删除 bare prompt triage

与 `spec-work` 一致，`spec-work-beta` 当前也更偏向 plan-driven execution。

#### A.2.3 delegation config / scratch path 全面本地化

主要迁移：

- `.compound-engineering/config.local.yaml` → `.spec-first/config.local.yaml`
- `.context/compound-engineering/codex-delegation/...` → `.spec-first/workflows/work-beta/codex-delegation/...`

这属于本地 runtime contract。

#### A.2.4 platform gate 与本地执行表述收紧

`spec-work-beta` 把“Claude Code instead”收敛成“Execute locally instead”，并把 platform gate 改成：

- 当前 session 如果已经是 Codex 或其他会递归的 host，就禁用 delegation

这是更健壮的 host-awareness，而不是退化。

#### A.2.5 delegation reference 有若干执行约束变化

当前版本的关键变化包括：

- consent / sandbox 文案调整
- shell/polling 表述收紧
- `verify` 段删去“orchestrator will not re-run verification independently”之类更强表述
- 结果 handoff 文案更简洁

这些大多是 contract 收口，不是功能删除。

#### A.2.6 新增 swarm mode

与 `spec-work` 一样，`spec-work-beta` 新增了 Swarm Mode with Agent Teams。

### A.3 现状判断

#### 结论 1：按上游严格等价口径，`spec-work-beta` 未完全追平

原因主要是：

1. 删除 bare prompt triage
2. 一些 delegation 文案与 host assumptions 被本地重写

#### 结论 2：按当前产品边界，`spec-work-beta` 已经是更贴合 `spec-first` runtime 的 delegation executor

增强点：

1. `.spec-first/` config 与 scratch path
2. 更强的 platform gate
3. 与 `spec-work` 一致的 swarm / execution 收口

## Part B：当前产品边界下的正式实施方案

### B.1 目标

1. 保留 `spec-work-beta` 作为 plan-driven、delegation-aware executor
2. 明确当前 delegation contract 的本地不变式
3. 为未来 delegation 回归提供测试保护

### B.2 非目标

1. 不恢复 bare prompt triage
2. 不回退 `.spec-first/` config / scratch path
3. 不把 host-aware platform gate 改回上游表述

### B.3 分阶段方案

#### Phase 1：显式冻结 delegation invariants

建议在 `codex-delegation-workflow.md` 周边明确以下不变式：

1. scratch 路径固定在 `.spec-first/workflows/work-beta/...`
2. config 固定在 `.spec-first/config.local.yaml`
3. 当前 session 若会递归到 Codex，本次 delegation 必须关闭

#### Phase 2：补 contract test

建议新增：

- `tests/unit/spec-work-beta-contracts.test.js`

覆盖：

- source skill 内部命名
- `disable-model-invocation: true`
- argument-hint 与 delegation token
- config path / scratch path 指向 `.spec-first`
- shipping-workflow 指向 `spec-review`
- Claude / Codex runtime 变换正确

#### Phase 3：把 delegation reference 的关键 schema 固化

建议进一步保护：

- result schema path
- prompt batch path
- polling / rollback / circuit breaker 关键文案

### B.4 影响文件

- `skills/spec-work-beta/SKILL.md`
- `skills/spec-work-beta/references/codex-delegation-workflow.md`
- `skills/spec-work-beta/references/shipping-workflow.md`
- `tests/unit/spec-work-beta-contracts.test.js`

```bash
npx jest tests/unit/spec-work-beta-contracts.test.js --runInBand
npm run test:unit
```

### B.6 最终建议

`spec-work-beta` 当前最重要的不是“文本追平”，而是把 delegation runtime contract 固化下来：路径、host gate、consent/config、结果分类都需要测试保护。只要这些 contract 稳定，本地增强就能持续演进。
