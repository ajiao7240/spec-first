# 业务领域模型

## 核心概念

### Feature（需求特性）

一个完整的需求特性，从初始化到上线的完整生命周期单元。

**属性**:
- featureId: 唯一标识
- mode: N（新功能）/ I（迭代）
- size: S（小）/ M（中）/ L（大）
- currentStage: 当前所处阶段
- terminal: 是否已终止

### Stage（阶段）

Feature 生命周期的 8 个活跃阶段 + 2 个终态：

| 阶段 | 代码 | 说明 |
|------|------|------|
| 初始化 | 00_init | Feature 工作区初始化 |
| 需求规格 | 01_specify | 编写需求规格文档 |
| 技术设计 | 02_design | 技术方案设计 |
| 任务拆解 | 03_plan | 拆解为可执行任务 |
| 代码实现 | 04_implement | 编码与单元测试 |
| 验证测试 | 05_verify | 集成测试与验收 |
| 归档复盘 | 06_wrap_up | 文档归档与复盘 |
| 发布上线 | 07_release | 上线部署 |
| **已完成** | 08_done | 终态：成功完成 |
| **已取消** | 09_cancelled | 终态：取消 |

### 追溯 ID 体系

| ID 类型 | 说明 | 示例 |
|---------|------|------|
| Feature | 需求特性 | FEAT-20260305-001 |
| FR | 功能需求 | FR-001 |
| DS | 设计规格 | DS-001 |
| TASK | 开发任务 | TASK-001 |
| TC | 测试用例 | TC-UT-001 |
| RFC | 变更请求 | RFC-001 |

**TC 级别前缀**: UT（单元测试）/ IT（集成测试）/ E2E（端到端）/ ST（系统测试）

## 业务流程

### 1. Feature 生命周期流转

```
初始化 → 需求规格 → 技术设计 → 任务拆解 → 代码实现 → 验证测试 → 归档复盘 → 发布上线 → 已完成
   ↓                                                                                    ↓
已取消 ←────────────────────────────────────────────────────────────────────────────── 已取消
```

**状态机规则**:
- 只能单向前进（00→01→02→...→08）
- 任意阶段可取消（→09_cancelled）
- 终态不可逆转

### 2. 质量门禁（Gate）

每个阶段流转前必须通过质量门禁检查。

**Gate 状态**:
- PASS: 通过
- PASS_WITH_WAIVER: 豁免通过
- FAIL: 失败

**检查项**:
- 产物完整性（必需文档是否存在）
- 追溯完整性（ID 关联是否完整）
- 覆盖率指标（C1-C9）
- 安全扫描（SCA）

### 3. RFC 变更管理

**RFC 状态流转**:
```
draft（草稿）→ approved（已批准）→ closed（已关闭）
       ↓
   rejected（已拒绝）
```

**RFC 级别**:
- Minor: 小改动
- Major: 重大变更
- Critical: 关键变更（需豁免）

### 4. 缺陷管理

**Defect 状态流转**:
```
open（新建）→ fixing（修复中）→ fixed（已修复）→ verified（已验证）
       ↓
   wontfix（不修复）
```

**严重级别**: S1（严重）/ S2（高）/ S3（中）/ S4（低）

## 业务规则

### 规则 1: 阶段流转必须通过 Gate

- 执行 `spec-first stage advance` 时自动触发 Gate 检查
- Gate 失败时阻止流转
- 可通过 RFC 申请豁免

### 规则 2: 追溯 ID 必须关联

- FR 必须关联 DS
- TASK 必须关联 FR
- TC 必须关联 FR
- 覆盖率矩阵自动计算关联完整性

### 规则 3: RFC 豁免有时效

- 豁免必须设置过期时间
- 豁免必须指定回滚点
- 过期后自动失效

### 规则 4: 终态不可逆

- Feature 进入 08_done 或 09_cancelled 后不可再流转
- 需要重新 init 新 Feature

## 核心实体关系

```
Feature (1) ──┬── (N) FR
              ├── (N) DS
              ├── (N) TASK
              ├── (N) TC
              ├── (N) RFC
              └── (N) Defect

RFC (1) ──── (N) Waiver ──── (1) FR

Defect (1) ──┬── (0..1) FR
             └── (0..1) TC
```
