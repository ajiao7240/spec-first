## 任务 F：修正 registerAIHooks 格式 + 5 函数 dry-run 支持

**优先级**：P0（最高，阻塞所有后续任务）

### F1：修正 `registerAIHooks` 写入格式

**文件**：`src/core/tool-integration/ai-runtime-hook.ts`

**现状**（`ai-runtime-hook.ts:60-87`）：
- 写入 `{ "hooks": { "PreToolUse": [{ matcher, command }] } }` — 双重错误
- 错误 1：多了 `"hooks"` 包裹层（settings.json 事件名直接作为顶层 key）
- 错误 2：条目格式用了 `{ matcher, command }` 而非 `{ matcher, hooks: [{ type, command }] }`

**目标格式**（Claude Code 官方，Context7 确认）：

```json
{
  "PreToolUse": [
    {
      "matcher": "write|edit|create",
      "hooks": [
        { "type": "command", "command": "sh -c '...'" }
      ]
    }
  ]
}
```

**改动要点**：
1. 移除 `settings.hooks` 包裹层，改为直接读写顶层事件 key
2. 条目格式从 `{ matcher, command }` 改为 `{ matcher, hooks: [{ type: "command", command }] }`
3. 过滤逻辑：匹配 `item.hooks` 数组中包含 `npx spec-first` 的条目
4. 保留幂等性：先过滤已有 spec-first 条目，再追加

**验收**：
- `.claude/settings.json` 中 PreToolUse/PostToolUse/Stop 采用官方嵌套格式
- 无 `"hooks"` 包裹层
- 已有非 spec-first 的 hook 条目不受影响

### F2：5 个核心函数添加 `dryRun` 参数

**文件**（逐个）：

## 任务 L：viewer open 新增 --background 非阻塞模式

**优先级**：P0（阻塞任务 K）

**文件**：`src/cli/commands/viewer.ts`

**现状**（`viewer.ts:83-87`）：
```typescript
const result = spawnSync(process.execPath, [bootstrapScript, ...passthrough], {
  cwd: process.cwd(),
  stdio: 'inherit',
  env: process.env,
});
```

**改动**：当 `passthrough` 包含 `--background` 时，改用 `spawn` detached 模式：

```typescript
import { spawn } from 'node:child_process';

// --background 检测
if (passthrough.includes('--background')) {
  const idx = passthrough.indexOf('--background');
  passthrough.splice(idx, 1);
  const child = spawn(process.execPath, [bootstrapScript, ...passthrough], {
    cwd: process.cwd(),
    detached: true,
    stdio: 'ignore',
    env: process.env,
  });
  child.unref();
  return ExitCode.SUCCESS;
}
```

**插入位置**：`viewer.ts:83` 之前（`spawnSync` 调用之前）。

**验收**：
- `spec-first viewer open --print-url --background` 立即返回，不阻塞
- 非 spec-first 项目（无 bootstrap 脚本）返回 CONFIG_ERROR 但不阻塞
- 无 `--background` 时行为不变（仍用 spawnSync）

## 任务 C：新增 postinstall 入口

**优先级**：P0

**新建文件**：`src/postinstall.ts`

**实现**：

```typescript
import { execSync } from 'node:child_process';

function isGlobalInstall(): boolean {
  if (process.env.npm_config_global === 'true') return true;
  if (process.env.PNPM_HOME && process.execPath.includes(process.env.PNPM_HOME)) return true;
  return false;
}

if (isGlobalInstall()) {
  try {
    execSync('spec-first update --from-postinstall', { stdio: 'inherit' });
  } catch {
    // fallback：bin 链接可能尚未就绪，用 node 直接调用
    try {
      const entry = join(new URL('.', import.meta.url).pathname, 'index.js');
      execSync(`${process.execPath} ${entry} update --from-postinstall`, { stdio: 'inherit' });
    } catch {
      console.error('提示：请手动执行 spec-first update 完成注册。');
    }
  }
}
// 非全局安装或检测失败：静默退出（exit 0）
```

**验收**：
- `npm install -g` 时自动调用 `spec-first update --from-postinstall`
- 失败时输出提示但 exit 0（不阻塞安装）
- 作为项目依赖安装时静默跳过
