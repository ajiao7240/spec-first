# 平台集成概览

spec-first 支持多种 AI 编码工具，让您可以在喜欢的平台上使用规范驱动开发工作流。

## 支持的平台

| 平台 | 配置目录 | 特性支持 | 推荐场景 |
|------|----------|----------|----------|
| **Claude Code** | `.claude` | 完整支持（命令、代理、钩子） | 首选平台，功能最全 |
| **Cursor** | `.cursor` | Slash 命令 | Cursor IDE 用户 |
| **OpenCode** | `.opencode` | 命令、代理、插件 | 开源 CLI 工具 |
| **Codex** | `.codex` | Skills、代理、钩子 | OpenAI Codex 用户 |
| **iFlow** | `.iflow` | 完整支持（命令、代理、钩子） | iFlow CLI 用户 |
| **Kilo** | `.kilocode` | 工作流 | Kilo CLI 用户 |
| **Kiro** | `.kiro/skills` | Skills | Kiro Code 用户 |
| **Gemini** | `.gemini` | TOML 命令 | Google Gemini CLI |
| **Antigravity** | `.agent/workflows` | 工作流 | Antigravity 用户 |
| **Qoder** | `.qoder` | Skills | Qoder 用户 |
| **CodeBuddy** | `.codebuddy` | 嵌套命令 | CodeBuddy 用户 |

## 功能对比

### 完整功能支持
以下平台支持 spec-first 的全部功能：

- **Claude Code** - Anthropic 官方 CLI，功能最完整
- **iFlow** - 支持 Python 钩子和多代理流水线

### 命令 + 代理支持
- **OpenCode** - 开源替代方案，支持 JavaScript 插件

### 仅命令支持
- **Cursor** - 通过 `.cursor/commands/` 目录提供 slash 命令
- **CodeBuddy** - 使用嵌套目录结构

### Skills/工作流支持
- **Codex** - 使用 SKILL.md 格式，支持 `.agents/skills/` 开放标准
- **Kiro** - 使用 SKILL.md 格式
- **Kilo** - 工作流模式
- **Gemini** - TOML 格式命令
- **Antigravity** - Markdown 工作流
- **Qoder** - Skills 模式

## 选择指南

### 如果您是 Claude Code 用户
选择 **Claude Code** 平台配置。这是 spec-first 的首选平台，提供：
- 完整的 slash 命令集
- 多代理流水线（plan → research → implement → check）
- Python 钩子实现上下文注入
- 会话自动初始化

### 如果您使用 Cursor IDE
选择 **Cursor** 平台配置。特点：
- 通过 `Cmd+K` 或 `Cmd+L` 输入 `/spec-xxx` 命令
- 与 Cursor 原生功能无缝集成
- 简洁的命令格式

### 如果您偏好开源方案
选择 **OpenCode** 平台配置。特点：
- 完全开源的 CLI 工具
- JavaScript 插件系统
- 支持多代理协作

### 如果您使用其他平台
根据您的工具选择对应配置：
- **Codex** - OpenAI 生态系统
- **iFlow** - 类似 Claude Code 的完整体验
- **Gemini** - Google AI 生态系统
- **Kiro/Kilo** - 轻量级 Skills 模式

## 初始化命令

使用 spec-first CLI 初始化项目时指定平台：

```bash
# 单平台初始化
npx spec-first init --claude
npx spec-first init --cursor
npx spec-first init --opencode

# 多平台初始化
npx spec-first init --claude --cursor

# 交互式选择
npx spec-first init
```
