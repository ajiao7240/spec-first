## 基础问题

### Q: spec-first 是什么？

**A**: spec-first 是一个多平台 AI 编码框架，通过规范驱动开发（Spec-Driven Development）提升 AI 辅助编程的质量和一致性。

核心功能：
- 自动注入项目规范到 AI 会话
- 任务驱动的工作流管理
- 支持多 AI 编码平台
- 团队共享标准和记忆

### Q: spec-first 和 CLAUDE.md、AGENTS.md、.cursorrules 有什么区别？

**A**: 这些文件有用，但容易越写越大、越写越散。spec-first 在它们之外补充了：

| 功能 | 单文件 | spec-first |
|------|--------|------------|
| 分层组织 | ❌ | ✅ 多个 Spec 文件 |
| 任务管理 | ❌ | ✅ PRD、状态、上下文 |
| 会话记忆 | ❌ | ✅ Journal 系统 |
| 多平台复用 | ❌ | ✅ 统一结构 |

### Q: spec-first 只支持 Claude Code 吗？

**A**: 不是。spec-first 支持 10+ 平台：

- Claude Code
- Cursor
- OpenCode
- iFlow
- Codex
- Kilo
- Kiro
- Gemini CLI
- Antigravity
- Qoder
- CodeBuddy

### Q: spec-first 是免费的吗？

**A**: 是的，spec-first 是开源软件，基于 AGPL-3.0 许可证发布。

## 安装问题

### Q: 支持哪些 Node.js 版本？

**A**: 需要 Node.js 18.17.0 或更高版本。

```bash
# 检查版本
node --version

# 如果版本过低，使用 nvm 升级
nvm install 18
nvm use 18
```

### Q: 可以用 yarn 或 pnpm 安装吗？

**A**: 可以。

```bash
# yarn
yarn global add spec-first

# pnpm
pnpm add -g spec-first
```

### Q: 如何从源码安装？

**A**:

```bash
# 克隆仓库
git clone https://github.com/sunrain520/spec-first.git
cd spec-first

# 安装依赖
pnpm install

# 构建
pnpm build

# 全局链接
cd packages/cli
npm link
```

### Q: 安装后命令找不到怎么办？

**A**: npm 全局 bin 目录可能不在 PATH 中：

```bash
# 查看全局目录
npm config get prefix

# 添加到 PATH
export PATH="$(npm config get prefix)/bin:$PATH"

# 添加到 shell 配置文件
echo 'export PATH="$(npm config get prefix)/bin:$PATH"' >> ~/.bashrc
source ~/.bashrc
```

## 使用问题

### Q: 是不是每个 Spec 都得手写？

**A**: 不需要。建议的做法是：

1. 先让 AI 根据现有代码起草 Spec
2. 把真正关键的规则手动收紧
3. 持续迭代，不需要一次性写完

spec-first 的价值在于沉淀高信号规则并持续复用。

### Q: Spec 应该写什么内容？

**A**: 建议包含：

- **编码标准**: 命名规范、代码风格
- **架构指南**: 分层结构、模块划分
- **API 规范**: 接口格式、错误处理
- **测试规范**: 覆盖率要求、测试风格
- **审查规则**: PR 检查项、质量标准

### Q: 如何在现有项目中使用 spec-first？

**A**:

```bash
# 进入项目目录
cd /path/to/existing-project

# 初始化（跳过已存在文件）
spec-first init -s -u your-name

# 或者强制覆盖
spec-first init -f -u your-name
```

### Q: Monorepo 模式是什么？

**A**: Monorepo 模式用于多包仓库：

- 自动检测 `pnpm-workspace.yaml`、`lerna.json` 等
- 为每个包创建独立的 `.spec-first/`
- 共享根目录的 Spec

```bash
# 强制启用
spec-first init --monorepo -u user

# 禁用检测
spec-first init --no-monorepo -u user
```

### Q: 如何更新 spec-first？

**A**:

```bash
# 1. 更新 CLI
npm install -g spec-first@latest

# 2. 更新项目配置
cd /path/to/project
spec-first update

# 3. 预览变更
spec-first update --dry-run
```

## 平台问题

### Q: 如何选择平台？

**A**:

| 场景 | 推荐平台 |
|------|----------|
| 深度 AI 协作 | Claude Code |
| IDE 集成 | Cursor |
| 快速原型 | OpenCode |
| 企业开发 | Codex |

### Q: 如何在平台之间迁移？

**A**: 核心工作流是平台无关的：

```bash
# 1. 重新初始化新平台
spec-first init --<new-platform> -u user -f

# 2. Spec 和 Task 保持不变
# .spec-first/ 目录不需要改动

# 3. 使用新平台
# 新平台会读取相同的 Spec
```

### Q: 多个平台可以同时使用吗？

**A**: 可以。同时初始化多个平台：

```bash
spec-first init --cursor --claude --codex -u user
```

这会生成：
- `.cursor/` - Cursor 配置
- `.claude/` - Claude Code 配置
- `.codex/` - Codex 配置

## 团队协作问题

### Q: 团队一起用会不会经常冲突？

**A**: 不会。

- **共享内容**: Spec、Task、workflow.md - 提交到仓库，正常评审
- **个人内容**: workspace/journal - 在 .gitignore，不提交

### Q: 如何共享 Spec？

**A**:

1. Spec 文件提交到仓库
2. 团队成员 pull 获取最新 Spec
3. 修改 Spec 需要代码审查
4. 新成员入职先阅读 Spec

### Q: 如何分配任务？

**A**: 在任务目录中记录责任人：

```markdown
# tasks/TASK-001/prd.md

## 任务信息
- 负责人: 张三
- 优先级: 高
- 截止日期: 2024-01-20
```

或创建任务索引：

```markdown
# tasks/README.md

| 任务 | 负责人 | 状态 |
|------|--------|------|
| TASK-001 | 张三 | 实现中 |
| TASK-002 | 李四 | 测试中 |
```

### Q: 如何保持代码一致性？

**A**:

1. **共享 Spec**: 团队使用相同的编码规范
2. **代码审查**: 参考 Spec 进行审查
3. **持续迭代**: 发现问题更新 Spec
4. **自动化检查**: 配置 linter 和 formatter

## 其他问题

### Q: spec-first 会修改我的代码吗？

**A**: 不会。spec-first 只：
- 创建 `.spec-first/` 目录
- 生成平台配置文件
- 不会修改你的业务代码

### Q: 如何卸载 spec-first？

**A**:

```bash
# 卸载全局包
npm uninstall -g spec-first

# 删除项目配置（可选）
rm -rf .spec-first/
rm -rf .claude/ .cursor/ .codex/ .agents/
rm -f AGENTS.md
```
