## 安装问题

### 命令找不到

**症状**: 执行 `spec-first` 提示命令不存在

```bash
$ spec-first --version
command not found: spec-first
```

**原因**: npm 全局 bin 目录不在 PATH 中

**解决方案**:

```bash
# 1. 查看 npm 全局目录
npm config get prefix

# 2. 添加到 PATH
# bash (~/.bashrc)
echo 'export PATH="$(npm config get prefix)/bin:$PATH"' >> ~/.bashrc
source ~/.bashrc

# zsh (~/.zshrc)
echo 'export PATH="$(npm config get prefix)/bin:$PATH"' >> ~/.zshrc
source ~/.zshrc

# 3. 验证
spec-first --version
```

### 权限错误

**症状**: 安装或初始化时提示权限错误

```bash
$ npm install -g spec-first
npm ERR! Error: EACCES: permission denied
```

**解决方案**:

```bash
# 方案 1: 使用 sudo（不推荐）
sudo npm install -g spec-first

# 方案 2: 修改 npm 默认目录（推荐）
mkdir ~/.npm-global
npm config set prefix '~/.npm-global'
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc
source ~/.bashrc

# 方案 3: 使用 nvm 管理 Node.js
nvm install node
nvm use node
```

### 版本过低

**症状**: 提示 Node.js 版本不支持

```bash
$ spec-first init
Error: Node.js version 14.x is not supported
```

**解决方案**:

```bash
# 使用 nvm 升级
nvm install 18
nvm use 18
nvm alias default 18

# 或使用 n
n 18

# 验证版本
node --version  # 应该 >= 18.17.0
```

## 初始化问题

### 目录已存在

**症状**: 初始化时提示目录已存在

```bash
$ spec-first init -u user
Error: .spec-first already exists
```

**解决方案**:

```bash
# 方案 1: 强制覆盖
spec-first init -u user -f

# 方案 2: 跳过已存在文件
spec-first init -u user -s

# 方案 3: 先备份再初始化
mv .spec-first .spec-first.backup
spec-first init -u user
```

### 平台参数无效

**症状**: 指定的平台参数无效

```bash
$ spec-first init --invalid-platform
Error: Unknown platform: invalid-platform
```

**解决方案**:

```bash
# 查看支持的平台
spec-first init --help

# 支持的平台
--cursor --claude --opencode --iflow
--codex --kilo --kiro
--gemini --antigravity --qoder --codebuddy
```

### Monorepo 检测问题

**症状**: Monorepo 检测不正确

**解决方案**:

```bash
# 强制启用 monorepo 模式
spec-first init --monorepo -u user

# 禁用 monorepo 检测
spec-first init --no-monorepo -u user
```

## 命令问题

### 更新失败

**症状**: 更新项目配置失败

```bash
$ spec-first update
Error: Update failed
```

**解决方案**:

```bash
# 1. 检查版本
spec-first --version
cat .spec-first/.version

# 2. 先更新 CLI
npm install -g spec-first@latest

# 3. 预览变更
spec-first update --dry-run

# 4. 强制更新
spec-first update --force

# 5. 如果版本冲突，允许降级
spec-first update --allow-downgrade
```

### 版本不匹配

**症状**: CLI 和项目版本不匹配

```bash
Warning: CLI version (1.3.0) != Project version (0.4.0)
```

**解决方案**:

```bash
# 方案 1: 更新项目到最新
spec-first update

# 方案 2: 安装与项目匹配的 CLI 版本
npm install -g spec-first@0.4.0

# 方案 3: 允许版本差异
spec-first update --allow-downgrade
```

## 平台问题

### Claude Code Hooks 不生效

**症状**: Claude Code 的 hooks 没有执行

**检查步骤**:

```bash
# 1. 检查 settings.json
cat .claude/settings.json

# 2. 确认 hooks 配置
{
  "hooks": {
    "session-start": ".claude/hooks/session-start.py",
    "pre-tool-use": ".claude/hooks/inject-subagent-context.py"
  }
}

# 3. 检查 Python 环境
python3 --version

# 4. 检查 hooks 文件权限
ls -la .claude/hooks/

# 5. 手动测试 hook
python3 .claude/hooks/session-start.py
```

**解决方案**:

```bash
# 重新生成 hooks
spec-first init --claude -u user -f

# 确保 Python 可执行
chmod +x .claude/hooks/*.py
```

### Cursor Rules 不加载

**症状**: Cursor 不加载 `.cursorrules` 文件

**解决方案**:

```bash
# 1. 检查文件位置
ls -la .cursorrules
ls -la .cursor/rules

# 2. 确保 Cursor 版本支持
# Cursor 需要较新版本才支持 rules

# 3. 重新生成
spec-first init --cursor -u user -f

# 4. 重启 Cursor
```

### Codex Skills 不识别

**症状**: Codex 不识别自定义 Skills

**解决方案**:

```bash
# 1. 检查 skills 目录
ls -la .agents/skills/

# 2. 检查 skill 文件格式
cat .agents/skills/my-skill/skill.md

# 3. 确保 Codex 配置正确
cat .codex/config.json

# 4. 重新生成
spec-first init --codex -u user -f
```

## 调试技巧

### 启用调试模式

```bash
# 设置环境变量
DEBUG=spec-first:* spec-first init -u user

# 或
NODE_DEBUG=spec-first spec-first init -u user
```

### 查看详细日志

```bash
# 重定向输出到文件
spec-first init -u user 2>&1 | tee init.log

# 查看错误堆栈
spec-first init -u user --verbose
```

### 清理缓存

```bash
# 清理 npm 缓存
npm cache clean --force

# 重新安装
npm uninstall -g spec-first
npm install -g spec-first@latest
```

### 重置项目

```bash
# 备份现有配置
cp -r .spec-first .spec-first.backup

# 删除配置
rm -rf .spec-first

# 重新初始化
spec-first init -u user

# 如果需要恢复
# cp -r .spec-first.backup .spec-first
```
