## spec-first init

在当前项目中初始化 spec-first 工作流框架。

### 语法

```bash
spec-first init [选项]
```

### 选项说明

#### 平台选择选项

| 选项 | 说明 |
|------|------|
| `--cursor` | 包含 Cursor 命令配置 |
| `--claude` | 包含 Claude Code 命令配置 |
| `--iflow` | 包含 iFlow CLI 命令配置 |
| `--opencode` | 包含 OpenCode 命令配置 |
| `--codex` | 包含 Codex skills 配置 |
| `--kilo` | 包含 Kilo CLI 命令配置 |
| `--kiro` | 包含 Kiro Code skills 配置 |
| `--gemini` | 包含 Gemini CLI 命令配置 |
| `--antigravity` | 包含 Antigravity workflows 配置 |
| `--qoder` | 包含 Qoder 命令配置 |
| `--codebuddy` | 包含 CodeBuddy 命令配置 |

#### 交互模式选项

| 选项 | 简写 | 说明 |
|------|------|------|
| `--yes` | `-y` | 跳过所有提示，使用默认值 |
| `--user <name>` | `-u` | 指定开发者名称进行初始化 |

#### 文件处理选项

| 选项 | 简写 | 说明 |
|------|------|------|
| `--force` | `-f` | 强制覆盖已存在的文件，不进行询问 |
| `--skip-existing` | `-s` | 跳过已存在的文件，不进行询问 |

#### Monorepo 选项

| 选项 | 说明 |
|------|------|
| `--monorepo` | 强制启用 monorepo 模式 |
| `--no-monorepo` | 跳过 monorepo 检测 |

#### 模板选项

| 选项 | 简写 | 说明 |
|------|------|------|
| `--template <name>` | `-t` | 使用远程 spec 模板（如 `electron-fullstack`） |
| `--overwrite` | 使用模板时覆盖已存在的 spec 目录 |
| `--append` | 使用模板时仅添加缺失的文件 |

#### 注册表选项

| 选项 | 简写 | 说明 |
|------|------|------|
| `--registry <source>` | `-r` | 使用自定义模板注册表（如 `gh:myorg/myrepo/specs`） |

### 初始化流程

1. **显示欢迎横幅** - ASCII 艺术标题
2. **代理设置** - 自动检测并配置网络代理
3. **开发者识别** - 从 git 配置或选项获取开发者名称
4. **项目类型检测** - 自动检测 frontend/backend/fullstack
5. **Monorepo 检测** - 自动检测工作区配置
6. **平台选择** - 选择要配置的 AI 工具
7. **模板选择** - 选择空白模板或远程模板
8. **创建工作流结构** - 生成 `.spec-first/` 目录
9. **配置平台** - 复制平台特定配置
10. **创建引导任务** - 生成首个任务引导用户填写规范

## spec-first update

将 spec-first 配置和命令更新到最新版本。

### 语法

```bash
spec-first update [选项]
```

### 选项说明

| 选项 | 简写 | 说明 |
|------|------|------|
| `--dry-run` | | 预览变更而不实际应用 |
| `--force` | `-f` | 强制覆盖所有变更的文件，不进行询问 |
| `--skip-all` | `-s` | 跳过所有变更的文件，不进行询问 |
| `--create-new` | `-n` | 为所有变更的文件创建 `.new` 副本 |
| `--allow-downgrade` | | 允许降级到旧版本 |
| `--migrate` | | 应用待处理的文件迁移（重命名/删除） |

### 更新流程

1. **版本检查** - 比较项目版本与 CLI 版本
2. **变更扫描** - 检测新增、修改、未变更的文件
3. **用户修改检测** - 使用哈希追踪识别用户修改
4. **交互式冲突解决** - 对于用户修改的文件询问处理方式
5. **安全文件删除** - 自动删除已弃用的命令文件
6. **备份创建** - 更新前创建时间戳备份
7. **应用更新** - 写入新文件内容

### 冲突解决选项

当检测到用户修改的文件时，提供以下选择：

| 选项 | 说明 |
|------|------|
| `[1] Overwrite` | 用新版本替换当前版本 |
| `[2] Skip` | 保留当前版本 |
| `[3] Create copy` | 将新版本保存为 `.new` 文件 |
| `[a] Apply Overwrite to all` | 对所有冲突文件应用覆盖 |
| `[s] Apply Skip to all` | 对所有冲突文件应用跳过 |
| `[n] Apply Create copy to all` | 对所有冲突文件应用创建副本 |

## 使用示例

### 基础初始化

```bash
# 交互式初始化（推荐新用户使用）
spec-first init

# 使用默认值快速初始化（Cursor + Claude Code）
spec-first init -y

# 指定开发者名称
spec-first init -u "john-doe"
```

### 平台选择

```bash
# 仅配置 Cursor
spec-first init --cursor

# 配置多个平台
spec-first init --cursor --claude --codex

# 配置所有支持的 AI 工具
spec-first init --cursor --claude --iflow --opencode --codex --kilo --kiro --gemini
```

### Monorepo 项目

```bash
# 强制启用 monorepo 模式
spec-first init --monorepo

# 跳过 monorepo 检测
spec-first init --no-monorepo
```

### 使用模板

```bash
# 使用官方模板
spec-first init --template electron-fullstack

# 使用自定义注册表
spec-first init --registry gh:myorg/myrepo/specs

# 使用自定义注册表和特定模板
spec-first init -r gh:myorg/myrepo/specs -t my-template

# 覆盖现有 spec 目录
spec-first init --template electron-fullstack --overwrite

# 仅添加缺失文件
spec-first init --template electron-fullstack --append
```

### 文件处理模式

```bash
# 强制覆盖已存在的文件
spec-first init --force

# 跳过已存在的文件
spec-first init --skip-existing
```

### 更新操作

```bash
# 交互式更新
spec-first update

# 预览变更
spec-first update --dry-run

# 强制覆盖所有变更
spec-first update --force

# 跳过所有变更
spec-first update --skip-all

# 创建 .new 副本而非覆盖
spec-first update --create-new

# 应用文件迁移（重命名/删除）
spec-first update --migrate

# 允许降级
spec-first update --allow-downgrade
```

## 退出码说明

| 退出码 | 说明 |
|--------|------|
| `0` | 成功完成 |
| `1` | 发生错误（如无效参数、文件操作失败等） |

### 常见错误场景

| 场景 | 原因 | 解决方案 |
|------|------|----------|
| 无效注册表源 | `--registry` 参数格式错误 | 使用正确格式：`gh:org/repo/path` |
| 网络连接失败 | 无法访问模板注册表 | 检查网络连接或配置代理 |
| Monorepo 未检测到 | 使用 `--monorepo` 但未找到配置 | 确保项目有有效的 monorepo 配置 |
| 无工具选择 | 未选择任何 AI 工具 | 至少选择一个平台选项 |
