## 多代理概述

spec-first 支持多 Agent 并行工作，适用于：

- **大型任务拆分**: 将复杂任务分解给多个 Agent
- **并行开发**: 同时开发多个独立功能
- **角色分工**: 不同 Agent 负责不同领域

### 架构图

```
                    ┌─────────────────┐
                    │   调度器        │
                    │  (dispatch)     │
                    └────────┬────────┘
                             │
         ┌───────────────────┼───────────────────┐
         │                   │                   │
         ▼                   ▼                   ▼
   ┌───────────┐       ┌───────────┐       ┌───────────┐
   │  Plan     │       │  Research │       │ Implement │
   │  Agent    │       │  Agent    │       │  Agent    │
   └───────────┘       └───────────┘       └───────────┘
```

## Agent 类型

### 内置 Agent

spec-first 提供以下内置 Agent：

| Agent | 职责 | 使用场景 |
|-------|------|----------|
| `plan` | 规划和设计 | 需求分析、技术设计 |
| `research` | 调研和分析 | 技术选型、问题排查 |
| `implement` | 代码实现 | 功能开发、Bug 修复 |
| `check` | 质量检查 | 代码审查、测试验证 |
| `debug` | 调试修复 | 问题定位、错误修复 |
| `dispatch` | 任务调度 | 多 Agent 协调 |

### 调用方式

**Claude Code 平台**:

```bash
# 自动调度
> 请帮我实现用户登录功能

# 手动指定
> /agent:plan 设计 API 接口
> /agent:research 调研认证方案
> /agent:implement 实现登录逻辑
> /agent:check 检查代码质量
```

## 并行执行

### 独立任务并行

当多个任务相互独立时，可以并行执行：

```
Task A (登录)  ──→  Agent 1
Task B (注册)  ──→  Agent 2
Task C (支付)  ──→  Agent 3
```

### 使用 Worktree

git worktree 允许多个 Agent 在同一仓库的不同目录工作：

```bash
# 创建 worktree
git worktree add ../project-feature-a feature-a
git worktree add ../project-feature-b feature-b

# Agent 1 在 project-feature-a 工作
# Agent 2 在 project-feature-b 工作
```

### 并行流程

```bash
# 1. 创建任务分支
spec-first init -u user1  # 在主目录
cd ../project-feature-a
spec-first init -u user1  # 在 worktree

# 2. 分配任务
# Agent 1: 处理 feature-a
# Agent 2: 处理 feature-b

# 3. 合并结果
git checkout main
git merge feature-a
git merge feature-b
```

## Worktree 隔离

### 为什么需要隔离

多 Agent 同时工作时，需要隔离：

- **代码隔离**: 避免互相覆盖
- **状态隔离**: 各自的任务状态
- **依赖隔离**: 不同的依赖版本

### Worktree 结构

```
project/
├── .spec-first/          # 共享的 Spec
├── src/
└── ...

project-feature-a/       # Worktree 1
├── .spec-first/
│   └── tasks/
│       └── TASK-001/    # Agent 1 的任务
└── src/

project-feature-b/       # Worktree 2
├── .spec-first/
│   └── tasks/
│       └── TASK-002/    # Agent 2 的任务
└── src/
```

### 共享 Spec

Spec 文件可以在 worktree 之间共享：

```bash
# 使用符号链接共享 Spec
ln -s ../../project/.spec-first/spec .spec-first/spec
```

## 协作模式

### 模式 1: 流水线

任务按顺序经过多个 Agent：

```
需求 → Plan → Research → Implement → Check → 完成
```

适用于：需要严格流程的任务

### 模式 2: 并行开发

多个 Agent 同时开发独立功能：

```
        ┌→ Agent 1 → Feature A ─┐
需求 ──→├→ Agent 2 → Feature B ─┼→ 集成
        └→ Agent 3 → Feature C ─┘
```

适用于：功能模块独立的场景

### 模式 3: 主从协作

一个主 Agent 协调多个从 Agent：

```
                ┌→ Worker 1
主 Agent (dispatch) → Worker 2
                └→ Worker 3
```

适用于：复杂任务的分解和协调

### 模式 4: 专项分工

不同领域的 Agent 处理不同任务：

```
前端任务 → Frontend Agent
后端任务 → Backend Agent
测试任务 → QA Agent
```

适用于：跨职能团队协作

## 最佳实践

### 任务分配

1. **明确边界**: 每个任务有清晰的职责范围
2. **减少依赖**: 尽量选择独立的任务并行
3. **合理拆分**: 任务粒度适中，不过大或过小

### 冲突避免

1. **文件隔离**: 不同 Agent 修改不同文件
2. **频繁同步**: 定期合并和同步代码
3. **清晰沟通**: 记录每个 Agent 的工作内容

### 状态管理

1. **独立状态**: 每个 worktree 维护自己的任务状态
2. **共享进度**: 通过共享的 journal 了解整体进度
3. **定期汇总**: 定期汇总各 Agent 的成果

## 示例场景

### 场景: 并行开发三个功能

```bash
# 1. 创建 worktree
git worktree add ../app-auth auth-feature
git worktree add ../app-payment payment-feature
git worktree add ../app-notification notification-feature

# 2. 在各 worktree 初始化
cd ../app-auth && spec-first init -u dev1
cd ../app-payment && spec-first init -u dev2
cd ../app-notification && spec-first init -u dev3

# 3. 启动三个 Agent 会话
# Agent 1 在 app-auth: 实现认证功能
# Agent 2 在 app-payment: 实现支付功能
# Agent 3 在 app-notification: 实现通知功能

# 4. 完成后合并
git checkout main
git merge auth-feature
git merge payment-feature
git merge notification-feature

# 5. 清理 worktree
git worktree remove ../app-auth
git worktree remove ../app-payment
git worktree remove ../app-notification
```
