# 快速开始

本教程帮助您在 5 分钟内上手 spec-first。

## 前提条件

- 已安装 spec-first（参考 [安装指南](./安装指南.md)）
- 有一个 Git 仓库（可以是现有项目或新项目）

---

## 第一步：初始化项目

在您的项目根目录运行：

```bash
# 进入项目目录
cd your-project

# 初始化 spec-first
spec-first init -u your-name
```

### 初始化选项说明

| 参数 | 说明 |
|------|------|
| `-u, --user <name>` | 指定开发者名称，用于创建个人工作区 |
| `--cursor` | 配置 Cursor 支持 |
| `--opencode` | 配置 OpenCode 支持 |
| `--codex` | 配置 Codex 支持 |
| `--claude` | 配置 Claude Code 支持（默认） |
| `-y, --yes` | 跳过交互，使用默认配置 |
| `--force` | 强制覆盖现有文件 |
| `--template <id>` | 使用指定的 spec 模板 |
| `--registry <url>` | 从自定义仓库拉取模板 |

### 示例：多平台初始化

```bash
# 同时配置多个 AI 平台
spec-first init --cursor --opencode --codex -u your-name
```

### 示例：使用模板

```bash
# 使用社区模板
spec-first init --template react-typescript -u your-name

# 从自定义仓库拉取模板
spec-first init --registry https://github.com/your-org/spec-templates -u your-name
```

│   │   ├── frontend/            # 前端开发规范
│   │   │   ├── directory-structure.md
│   │   │   ├── component-guidelines.md
│   │   │   ├── hook-guidelines.md

### 核心目录说明

| 目录 | 用途 |
|------|------|
| `spec/` | 存放项目规范、编码标准、设计模式 |
| `tasks/` | 管理任务 PRD、上下文和状态 |
| `workspace/` | 个人会话日志和记忆 |
| `scripts/` | 自动化脚本工具 |

## 第三步：完成引导任务

初始化后，spec-first 会自动创建一个引导任务，帮助您填充项目规范：

```bash
# 查看当前任务
cat .spec-first/tasks/00-bootstrap-guidelines/prd.md
```

### 填充规范的步骤

1. **分析现有代码**

   让 AI 帮助您从代码中提取规范：

   ```
   请分析我的代码库，提取编码规范和设计模式，填充到 .spec-first/spec/ 目录
   ```

2. **导入现有规范**

   如果项目已有 `CLAUDE.md`、`.cursorrules` 等文件：

   ```
   读取 CLAUDE.md 和 .cursorrules，将相关规范导入到 .spec-first/spec/
   ```

3. **完成任务**

   ```bash
   # 完成引导任务
   python3 ./.spec-first/scripts/task.py finish

   # 归档任务
   python3 ./.spec-first/scripts/task.py archive 00-bootstrap-guidelines
   ```

## 第四步：创建第一个任务

使用任务脚本创建新任务：

```bash
# 创建新任务
python3 ./.spec-first/scripts/task.py create my-first-feature

# 查看任务状态
python3 ./.spec-first/scripts/task.py status

# 切换任务
python3 ./.spec-first/scripts/task.py switch my-first-feature
```

### 任务目录结构

每个任务包含：

```text
.spec-first/tasks/my-first-feature/
├── task.json        # 任务元数据（状态、优先级、负责人等）
├── prd.md           # 产品需求文档
├── context.md       # 实现上下文（可选）
└── review.md        # 评审检查清单（可选）
```

### 任务 JSON 示例

```json
{
  "id": "my-first-feature",
  "name": "我的第一个功能",
  "description": "功能描述",
  "status": "in_progress",
  "priority": "P1",
  "dev_type": "feature",
  "creator": "your-name",
  "assignee": "your-name",
  "createdAt": "2026-03-25",
  "subtasks": [],
  "relatedFiles": []
}
```

## 常用命令速查

### spec-first CLI

```bash
# 初始化项目
spec-first init -u <name>

# 更新 spec-first 模板
spec-first update

# 查看版本
spec-first --version

# 查看帮助
spec-first --help
```

### 任务管理脚本

```bash
# 创建任务
python3 ./.spec-first/scripts/task.py create <task-name>

# 查看所有任务
python3 ./.spec-first/scripts/task.py list

# 查看当前任务状态
python3 ./.spec-first/scripts/task.py status

# 切换到任务
python3 ./.spec-first/scripts/task.py switch <task-name>

# 完成任务
python3 ./.spec-first/scripts/task.py finish

# 归档任务
python3 ./.spec-first/scripts/task.py archive <task-name>
```
