# 安装指南

本文档介绍如何在您的系统上安装 spec-first。

## 系统要求

在安装 spec-first 之前，请确保您的系统满足以下要求：

| 依赖 | 最低版本 | 说明 |
|------|---------|------|
| Node.js | >= 18.17.0 | 运行环境 |
| pnpm | >= 8.0.0 | 包管理器（源码安装时需要） |
| Git | >= 2.0.0 | 版本控制 |
| Python | >= 3.8 | 可选，用于运行 hook 脚本 |

### 检查环境

```bash
# 检查 Node.js 版本
node --version

# 检查 pnpm 版本
pnpm --version

# 检查 Git 版本
git --version

# 检查 Python 版本
python3 --version
```

---

## 安装方式

### 方式一：npm 全局安装（推荐）

最简单的安装方式是通过 npm 全局安装：

```bash
npm install -g spec-first@latest
```

安装完成后，验证安装：

```bash
spec-first --version
```

### 方式二：源码安装

如果您需要最新开发版本或想要贡献代码，可以从源码安装：

#### 1. 克隆仓库

```bash
git clone https://github.com/sunrain520/spec-first.git
cd spec-first
```

#### 2. 安装依赖

```bash
pnpm install
```

#### 3. 构建项目

```bash
pnpm build
```

#### 4. 链接到全局

```bash
cd packages/cli
pnpm link --global
```

#### 5. 验证安装

```bash
spec-first --version
```

---

## 验证安装

安装成功后，运行以下命令验证：

```bash
# 查看版本
spec-first --version

# 查看帮助信息
spec-first --help

# 初始化一个测试项目
mkdir test-project && cd test-project
spec-first init -u your-name
```

如果一切正常，您将看到 spec-first 的 ASCII 艺术 banner 和初始化向导。

## 常见安装问题

### 1. `command not found: spec-first`

**原因**：全局安装的 npm 包不在 PATH 环境变量中。

**解决方案**：

```bash
# 检查 npm 全局安装路径
npm config get prefix

# 将 npm 全局路径添加到 PATH（添加到 ~/.bashrc 或 ~/.zshrc）
export PATH="$(npm config get prefix)/bin:$PATH"

# 重新加载配置
source ~/.bashrc  # 或 source ~/.zshrc
```

### 2. `EACCES: permission denied`

**原因**：npm 全局目录权限不足。

**解决方案**：

```bash
# 方案 A：使用 nvm 管理 Node.js（推荐）
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash
nvm install 18
nvm use 18

# 方案 B：修改 npm 全局目录权限
mkdir ~/.npm-global
npm config set prefix '~/.npm-global'
export PATH=~/.npm-global/bin:$PATH
```

### 3. `pnpm: command not found`

**原因**：未安装 pnpm。

**解决方案**：

```bash
# 使用 npm 安装 pnpm
npm install -g pnpm

# 或使用 corepack（Node.js 16.9+）
corepack enable
corepack prepare pnpm@latest --activate
```

### 4. Node.js 版本过低

**原因**：spec-first 需要 Node.js >= 18.17.0。

**解决方案**：

```bash
# 使用 nvm 升级 Node.js
nvm install 18
nvm use 18

# 或使用 nvm 安装 LTS 版本
nvm install --lts
nvm use --lts
```

### 5. Windows 上 Python hook 不工作

**原因**：Windows 上 Python 命令可能是 `python` 而不是 `python3`。

**解决方案**：

spec-first 会自动检测 Python 命令。如果仍有问题，请确保：

```bash
# 确认 Python 已安装
python --version

# 如需，创建 python3 别名
mklink python3.exe python.exe
```

## 更新 spec-first

### npm 安装方式

```bash
npm update -g spec-first@latest
```

### 源码安装方式

```bash
cd spec-first
git pull origin master
pnpm install
pnpm build
```
