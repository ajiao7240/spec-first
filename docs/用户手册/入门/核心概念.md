# 核心概念

本文档介绍 spec-first 的核心概念和设计理念。

---

## spec-first 是什么？

spec-first 是一个**规范驱动的 AI 开发框架**，旨在通过可执行的项目标准提升交付质量。

### 核心价值

| 能力 | 带来的变化 |
|------|-----------|
| **自动注入 Spec** | 把规范写进 `.spec-first/spec/` 之后，spec-first 会在每次会话里注入当前任务真正需要的上下文，不用反复解释 |
| **任务驱动工作流** | PRD、实现上下文、检查上下文和任务状态都放进 `.spec-first/tasks/`，AI 开发不会越做越乱 |
| **并行 Agent 执行** | 用 git worktree 同时推进多个 AI 任务，不需要把一个分支挤成大杂烩 |
| **项目记忆** | `.spec-first/workspace/` 里的 journal 会保留上一次工作的脉络，让新会话不是从空白开始 |
| **团队共享标准** | Spec 跟着仓库一起版本化，一个人总结出来的规则和流程，可以直接变成整个团队的基础设施 |
| **多平台复用** | 同一套 spec-first 结构可以带到 10 个 AI coding 平台上，而不是每换一个工具就重搭一次工作流 |

### 与其他工具的区别

| 工具 | 特点 | spec-first 的改进 |
|------|------|------------------|
| `CLAUDE.md` | 单文件，容易变大变散 | 分层 Spec，按需注入 |
| `AGENTS.md` | 缺乏结构化管理 | 任务上下文 + 规范分离 |
| `.cursorrules` | 仅限 Cursor 使用 | 多平台统一结构 |

## 核心概念

### 1. Spec（规范）

Spec 是项目的"宪法"，定义了编码标准、设计模式和工作流规则。

#### Spec 目录结构

```text
.spec-first/spec/
├── backend/                  # 后端规范
│   ├── directory-structure.md    # 目录结构约定
│   ├── database-guidelines.md    # 数据库规范
│   ├── error-handling.md         # 错误处理规范
│   ├── logging-guidelines.md     # 日志规范
│   └── quality-guidelines.md     # 质量标准
├── frontend/                 # 前端规范
│   ├── directory-structure.md    # 目录结构约定
│   ├── component-guidelines.md   # 组件规范
│   ├── hook-guidelines.md        # Hook 规范
│   ├── state-management.md       # 状态管理规范
│   ├── type-safety.md            # 类型安全规范
│   └── quality-guidelines.md     # 质量标准
└── guides/                   # 思考指南
    ├── before-implementation.md  # 实现前思考
    ├── cross-layer-guide.md      # 跨层一致性检查
    └── ralph-loop.md             # Ralph Loop 验证循环
```

#### Spec 的作用

1. **上下文注入**：根据当前任务自动加载相关规范
2. **代码审查**：作为代码检查的依据
3. **团队协作**：统一团队编码风格

#### 如何编写 Spec

```markdown
# 组件规范示例

## 命名约定

- 组件文件：PascalCase（如 `UserProfile.tsx`）
- Hook 文件：camelCase，以 `use` 开头（如 `useAuth.ts`）
- 工具函数：camelCase（如 `formatDate.ts`）

## 文件结构

```text
components/
├── UserProfile/
│   ├── index.ts           # 导出入口
│   ├── UserProfile.tsx    # 主组件
│   ├── UserProfile.test.tsx
│   ├── useUserProfile.ts  # 相关 Hook
│   └── types.ts           # 类型定义
```

## 代码示例

```tsx
// 正确示例
export const UserProfile: React.FC<UserProfileProps> = ({ userId }) => {
  const { user, loading } = useUserProfile(userId);

  if (loading) return <Skeleton />;

  return <div>{user.name}</div>;
};
```

## 反模式

- 避免在组件中直接调用 API
- 避免使用 any 类型
- 避免内联样式（使用 Tailwind 或 CSS Modules）
```

### 2. Task（任务）

Task 是工作的组织单元，包含 PRD、上下文和状态跟踪。

#### Task 目录结构

```text
.spec-first/tasks/
├── 00-bootstrap-guidelines/     # 引导任务
│   ├── task.json
│   └── prd.md
├── feature-user-auth/           # 功能任务
│   ├── task.json
│   ├── prd.md                   # 需求文档
│   ├── context.md               # 实现上下文
│   └── review.md                # 评审清单
└── bugfix-login-error/          # Bug 修复任务
    ├── task.json
    └── prd.md
```

#### Task JSON 结构

```json
{
  "id": "feature-user-auth",
  "name": "用户认证功能",
  "description": "实现用户登录、注册和密码重置功能",
  "status": "in_progress",
  "dev_type": "feature",
  "priority": "P1",
  "creator": "alice",
  "assignee": "bob",
  "createdAt": "2026-03-25",
  "completedAt": null,
  "commit": null,
  "subtasks": [
    { "name": "实现登录 API", "status": "done" },
    { "name": "实现注册 API", "status": "in_progress" },
    { "name": "编写单元测试", "status": "pending" }
  ],
  "children": [],
  "parent": null,
  "relatedFiles": [
    "src/api/auth.ts",
    "src/middleware/auth.ts"
  ],
  "notes": "",
  "meta": {}
}
```

#### 任务状态

| 状态 | 说明 |
|------|------|
| `pending` | 待处理 |
| `in_progress` | 进行中 |
| `done` | 已完成 |
| `blocked` | 被阻塞 |
| `cancelled` | 已取消 |

#### 任务类型

| 类型 | 说明 |
|------|------|
| `feature` | 新功能 |
| `bugfix` | Bug 修复 |
| `refactor` | 重构 |
| `docs` | 文档 |
| `test` | 测试 |
| `chore` | 杂项 |

### 3. Workspace（工作区）

Workspace 是个人工作空间，保存会话日志和个人记忆。

#### Workspace 目录结构

```text
.spec-first/workspace/
├── alice/                     # 开发者 alice 的工作区
│   ├── journal.md             # 会话日志
│   └── memory/                # 个人记忆
│       ├── project-context.md
│       └── decisions.md
└── bob/                       # 开发者 bob 的工作区
    ├── journal.md
    └── memory/
```

#### Journal（会话日志）

Journal 记录每次 AI 会话的关键信息：

```markdown
# 会话日志 - 2026-03-25

## 当前任务
feature-user-auth

## 完成内容
- 实现了登录 API
- 编写了登录 API 的单元测试
- 修复了 token 过期问题

## 待办事项
- [ ] 实现注册 API
- [ ] 添加邮箱验证
- [ ] 编写集成测试

## 遇到的问题
1. JWT token 在跨域请求中丢失
   - 解决方案：配置 CORS 允许 credentials

## 决策记录
- 使用 bcrypt 进行密码哈希（cost factor: 12）
- Token 有效期：7 天
```

#### Memory（记忆）

Memory 保存项目级别的持久知识：

```markdown
# 项目上下文

## 技术栈
- 前端：React 18 + TypeScript + Tailwind
- 后端：Node.js + Express + PostgreSQL
- 认证：JWT + bcrypt

## 关键决策
- 2026-03-20: 选择 PostgreSQL 而非 MySQL（需要 JSONB 支持）
- 2026-03-22: 使用 pnpm 而非 npm（monorepo 支持）

## 已知问题
- 暂无
```

## .spec-first 目录详解

```text
.spec-first/
├── spec/                    # 项目规范（团队共享）
│   ├── backend/
│   ├── frontend/
│   └── guides/
├── tasks/                   # 任务管理（团队共享）
│   └── <task-name>/
│       ├── task.json
│       ├── prd.md
│       └── context.md
├── workspace/               # 工作区（个人）
│   └── <username>/
│       ├── journal.md
│       └── memory/
├── scripts/                 # 工具脚本
│   ├── task.py              # 任务管理
│   ├── init_developer.py    # 开发者初始化
│   └── ...
├── workflow.md              # 工作流规则
├── config.yaml              # 配置文件
└── .version                 # 版本追踪
```

### 配置文件 (config.yaml)

```yaml
# 项目配置
project:
  name: my-project
  type: fullstack

# Monorepo 配置（可选）
packages:
  web:
    path: packages/web
  api:
    path: packages/api
default_package: web

# 平台配置
platforms:
  claude:
    enabled: true
  cursor:
    enabled: true
```

## 工作流程

### 整体流程

```text
┌─────────────────────────────────────────────────────────────────┐
│                        spec-first 工作流                          │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  1. 初始化项目                                                     │
│     $ spec-first init -u your-name                               │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  2. 填充项目规范                                                   │
│     - 分析现有代码                                                 │
│     - 导入现有规范（CLAUDE.md, .cursorrules 等）                    │
│     - 编写编码标准                                                 │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  3. 创建任务                                                       │
│     $ python3 .spec-first/scripts/task.py create <task-name>     │
│     - 编写 PRD                                                    │
│     - 定义子任务                                                   │
│     - 关联相关文件                                                 │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  4. AI 开发                                                        │
│     - spec-first 自动注入相关规范                                   │
│     - AI 根据规范生成代码                                           │
│     - 使用 Thinking Guides 进行思考                                │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  5. 验证与迭代                                                      │
│     - Ralph Loop 验证循环                                          │
│     - 代码审查                                                     │
│     - 更新 Journal                                                │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  6. 完成任务                                                       │
│     $ python3 .spec-first/scripts/task.py finish                 │
│     $ python3 .spec-first/scripts/task.py archive <task-name>    │
└─────────────────────────────────────────────────────────────────┘
```

### Ralph Loop（验证循环）

Ralph Loop 是 spec-first 的核心验证机制：

```text
┌──────────────┐
│   编码       │
└──────┬───────┘
       │
       ▼
┌──────────────┐     不通过
│   验证       │ ──────────────┐
└──────┬───────┘              │
       │                      │
       │ 通过                  │
       ▼                      │
┌──────────────┐              │
│   完成       │              │
└──────────────┘              │
                              │
       ┌──────────────────────┘
       │
       ▼
┌──────────────┐
│   修复       │
└──────┬───────┘
       │
       └──────► 返回验证步骤
```
