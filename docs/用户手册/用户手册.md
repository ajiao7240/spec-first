## 概述

**spec-first** 是一个多平台 AI 编码框架，通过规范驱动开发（Spec-Driven Development）提升 AI 辅助编程的质量和一致性。

### 核心特性

- **自动注入规范**: 在 `.spec-first/spec/` 中定义规范，自动注入到每个会话
- **任务驱动工作流**: 在 `.spec-first/tasks/` 中管理 PRD、实现上下文和任务状态
- **并行 Agent 执行**: 使用 git worktree 并行运行多个 AI 任务
- **项目记忆**: 在 `.spec-first/workspace/` 中保存工作日志和会话连续性
- **团队共享标准**: 规范随仓库版本化，一人总结的规则可惠及全团队
- **多平台支持**: 同一套 spec-first 结构可在 10+ AI 编码平台上使用

## 快速开始

### 1. 在项目中初始化

```bash
# 进入你的项目目录
cd /path/to/your-project

# 基础初始化
spec-first init -u your-name

# 或指定多个平台
spec-first init --cursor --claude --codex -u your-name
```

### 2. 初始化选项说明

| 选项 | 说明 |
|------|------|
| `-u, --user <name>` | 设置开发者身份，创建个人工作空间 |
| `--cursor` | 启用 Cursor 平台支持 |
| `--claude` | 启用 Claude Code 平台支持 |
| `--opencode` | 启用 OpenCode 平台支持 |
| `--iflow` | 启用 iFlow 平台支持 |
| `--codex` | 启用 Codex 平台支持 |
| `--kilo` | 启用 Kilo 平台支持 |
| `--kiro` | 启用 Kiro 平台支持 |
| `--gemini` | 启用 Gemini CLI 平台支持 |
| `--antigravity` | 启用 Antigravity 平台支持 |
| `--qoder` | 启用 Qoder 平台支持 |
| `--codebuddy` | 启用 CodeBuddy 平台支持 |
| `-y, --yes` | 跳过提示，使用默认配置 |
| `-f, --force` | 强制覆盖现有文件 |
| `-s, --skip-existing` | 跳过已存在的文件 |

### 3. 生成的目录结构

```
your-project/
├── .spec-first/
│   ├── spec/              # 项目规范、模式和指南
│   ├── tasks/             # 任务 PRD、上下文文件和状态
│   ├── workspace/         # 工作日志和开发者会话
│   │   └── your-name/     # 你的个人工作空间
│   ├── workflow.md        # 共享工作流规则
│   └── .version           # 版本信息
└── [平台特定文件]          # 根据启用的平台生成
    ├── .claude/           # Claude Code 配置
    ├── .cursor/           # Cursor 配置
    ├── AGENTS.md          # Agent 说明文件
    └── ...
```

## 初始化配置

### 使用模板初始化

spec-first 支持使用远程模板快速初始化：

```bash
# 使用内置模板
spec-first init -u your-name -t electron-fullstack

# 使用自定义模板仓库
spec-first init -u your-name --registry gh:myorg/myrepo/specs

# 完整示例
spec-first init --cursor --codex -u your-name \
  -t electron-fullstack \
  --registry gh:myorg/spec-templates
```

### 模板选项

| 选项 | 说明 |
|------|------|
| `-t, --template <name>` | 使用指定的远程模板 |
| `-r, --registry <source>` | 使用自定义模板仓库 |
| `--overwrite` | 使用模板时覆盖现有 spec 目录 |
| `--append` | 仅添加缺失的文件 |

### Monorepo 模式

如果你的项目是 monorepo：

```bash
# 强制启用 monorepo 模式
spec-first init --monorepo -u your-name

# 禁用 monorepo 检测
spec-first init --no-monorepo -u your-name
```

## 更新升级

### 检查更新

spec-first 会在每次运行时自动检查更新：

```bash
# 进入项目目录时会自动检查
cd /path/to/your-project

# 如果有更新，会显示提示：
# ⚠️  spec-first update available: 0.3.0 → 1.3.0
#    Run: spec-first update
```

### 更新 CLI 工具

```bash
# 更新到最新版本
npm install -g spec-first@latest

# 使用 pnpm
pnpm update -g spec-first

# 使用 yarn
yarn global upgrade spec-first
```

### 更新项目配置

当你更新 CLI 后，需要更新项目中的配置：

```bash
# 在项目目录中执行
cd /path/to/your-project

# 预览变更（不实际应用）
spec-first update --dry-run

# 交互式更新（会询问每个文件）
spec-first update

# 强制更新所有文件
spec-first update --force

# 跳过所有变更
spec-first update --skip-all

# 创建 .new 副本而不覆盖
spec-first update --create-new

# 应用文件迁移（重命名/删除）
spec-first update --migrate
```

### 更新选项说明

| 选项 | 说明 |
|------|------|
| `--dry-run` | 预览变更，不实际应用 |
| `-f, --force` | 强制覆盖所有变更文件 |
| `-s, --skip-all` | 跳过所有变更文件 |
| `-n, --create-new` | 为所有变更文件创建 .new 副本 |
| `--allow-downgrade` | 允许降级到旧版本 |
| `--migrate` | 应用待处理的文件迁移 |

## 配置说明

### .spec-first/ 目录

这是 spec-first 的核心目录，包含所有规范、任务和工作流配置。

#### spec/ 目录

存放项目规范文件，包括：

- 编码标准
- 架构指南
- 代码审查规则
- 工作流偏好

```markdown
# 示例：spec/coding-standards.md

## 编码规范

### TypeScript

- 使用严格的类型检查
- 优先使用 interface 而非 type
- 避免使用 any，使用 unknown 替代
```

#### tasks/ 目录

管理任务相关的文档：

- PRD（产品需求文档）
- 实现上下文
- 任务状态跟踪

#### workspace/ 目录

开发者个人工作空间：

- 工作日志
- 会话记录
- 个人笔记

#### workflow.md

共享的工作流规则，定义团队的工作方式。

### 平台特定配置

根据启用的平台，会生成相应的配置文件：

| 平台 | 配置文件 |
|------|----------|
| Claude Code | `.claude/` |
| Cursor | `.cursor/` |
| Codex | `.agents/`, `.codex/` |
| OpenCode | 配置集成到项目 |
| 其他 | 根据平台需求生成 |

## 常见问题

### Q1: 安装后命令找不到？

**问题**: 执行 `spec-first --version` 提示命令不存在

**解决方案**:

```bash
# 检查 npm 全局 bin 目录是否在 PATH 中
npm config get prefix

# 将 npm bin 目录添加到 PATH
# 对于 bash (~/.bashrc)
export PATH="$(npm config get prefix)/bin:$PATH"

# 对于 zsh (~/.zshrc)
export PATH="$(npm config get prefix)/bin:$PATH"

# 重新加载配置
source ~/.bashrc  # 或 source ~/.zshrc
```

### Q2: 初始化时提示权限错误？

**问题**: `spec-first init` 提示没有权限

**解决方案**:

```bash
# 检查当前目录权限
ls -la

# 如果需要，修改目录所有者
chown -R $USER:$USER /path/to/your-project

# 或使用 sudo（不推荐）
sudo spec-first init -u your-name
```

### Q3: 更新时版本冲突？

**问题**: CLI 版本与项目版本不匹配

**解决方案**:

```bash
# 查看 CLI 版本
spec-first --version

# 查看项目版本
cat .spec-first/.version

# 更新 CLI 到最新
npm install -g spec-first@latest

# 然后更新项目
spec-first update
```

### Q4: 如何在现有项目中使用？

**问题**: 项目已经存在，如何集成 spec-first

**解决方案**:

```bash
# 进入项目目录
cd /path/to/existing-project

# 初始化（会保留现有文件）
spec-first init -u your-name

# 或使用跳过现有文件模式
spec-first init -s -u your-name
```

### Q5: 如何在不同平台间迁移？

**问题**: 从 Claude Code 迁移到 Cursor

**解决方案**:

```bash
# 重新初始化，添加新平台
spec-first init --cursor --claude -u your-name -f

# 或仅更新
spec-first update --force
```

### Q6: 支持哪些 Node.js 版本？

**回答**: spec-first 需要 Node.js 18.17.0 或更高版本。

```bash
# 检查 Node.js 版本
node --version

# 如果版本过低，升级 Node.js
# 使用 nvm
nvm install 18
nvm use 18

# 或使用 n
n 18
```

## 故障排除

### 调试模式

如果遇到问题，可以启用调试输出：

```bash
# 设置环境变量
DEBUG=spec-first:* spec-first init -u your-name

# 或
NODE_DEBUG=spec-first spec-first init -u your-name
```

### 清理缓存

```bash
# 清理 npm 缓存
npm cache clean --force

# 重新安装
npm uninstall -g spec-first
npm install -g spec-first@latest
```

### 查看详细错误

```bash
# 初始化时查看详细输出
spec-first init -u your-name 2>&1 | tee init.log

# 更新时查看详细输出
spec-first update --dry-run 2>&1 | tee update.log
```

### 重置项目

如果需要完全重置 spec-first 配置：

```bash
# 备份现有配置
cp -r .spec-first .spec-first.backup

# 删除配置
rm -rf .spec-first

# 重新初始化
spec-first init -u your-name
```
