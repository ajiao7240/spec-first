## 核心内容

### 第一层：项目定位（顶部横幅）
```
Spec-First — AI 时代的规范驱动开发引擎
Spec-Driven AI Development Workflow Engine

一句话：阶段状态机 + 质量门禁 + 追溯体系 = 可审计的 AI 协作
```

### 第二层：核心问题与解决方案（左右对比）

**左侧 - 痛点（红色标注）：**
- ❌ AI 生成代码无状态，每次会话丢失上下文
- ❌ 需求变更无法追溯，"为什么这样写"无人知晓
- ❌ 代码直达生产，缺少验证层
- ❌ 团队成员各自提示 AI，评审不可靠

**右侧 - 方案（绿色标注）：**
- ✅ `specs/<featureId>/` 作为跨会话真理源
- ✅ 14 类追溯 ID（FR→DS→TASK→TC）全链路可追溯
- ✅ 9 阶段状态机 + 19 条质量门禁强制校验
- ✅ 20 个 Skill 统一执行协议（P0-P5 六阶段）

### 第三层：核心架构（中央主体，分三列）

#### 左列：阶段状态机（Stage State Machine）
```
单向不可逆流转：

                                                         ↓
                                              09_cancelled ←┘
                                              （任意阶段可取消）

每个箭头处标注：Gate Check（质量门禁）

#### 右列：Skill 执行协议（P0-P5）
```
每个 Skill 的标准执行流程：

P0 LOCATE       定位 Feature + 校验阶段
      ↓
P1 CONTEXT      加载规范目录 + 历史产物
      ↓
P2 GENERATE     AI 推理生成结构化草稿
      ↓
P3 CONFIRM      用户评审（支持多轮迭代）
      ↓
P4 WRITE        写入最终产物 + 注册追溯 ID
      ↓
P5 SIDE EFFECTS 同步矩阵 + 触发 Gate + 更新状态

特点：可定位、可审计、可确认、可追溯
```

14 类追溯 ID 全覆盖：

```

## 视觉元素建议

1. **配色方案**：
   - 主色：#2563EB（蓝色，代表技术与可靠）
   - 辅色：#10B981（绿色，代表通过/成功）
   - 警示：#EF4444（红色，代表阻塞/失败）
   - 背景：#F8FAFC（浅灰，干净专业）

2. **图标使用**：
   - 状态机：使用圆角矩形 + 箭头
   - 质量门禁：使用盾牌图标
   - 追溯 ID：使用链条图标
   - Skill：使用齿轮图标

3. **连接线**：
   - 实线：强制流程
   - 虚线：可选流程
   - 粗箭头：数据流向
   - 细箭头：依赖关系

4. **分层边框**：
   - 每层使用浅色边框分隔
   - 核心模块使用卡片式设计
   - 重要概念使用高亮背景

5. **文字层级**：
   - 标题：24-32px，粗体
   - 模块名：16-20px，中等
   - 说明文字：12-14px，常规

## 关键信息强调

在图中突出显示以下核心价值：

1. **单向不可逆**：状态机只能前进，不能回退（用粗箭头 + 锁图标）
2. **质量门禁**：每个阶段转换必须通过 Gate Check（用盾牌图标）
3. **全链路追溯**：从 Feature 到 TC 的完整追溯链（用链条连接）
4. **六阶段协议**：P0-P5 标准化执行流程（用数字标签）
5. **14 个核心引擎**：分层展示，突出职责分离
