## Cross-Review Delta

Prompt field mapping: `has_examples` is represented as `has_examples_or_evals`; `suspected_boundary_issue` is represented by this Cross-Review Delta plus the dedicated boundary tables in `05-boundary-overlap-report.md`. The source scan did not find any `skills/**/contract.yaml` files.

## Inventory Judgment

- Core public workflows are present, but most contracts are prose-only.
- No `skills/**/contract.yaml` files were found, so machine-readable stage contracts are absent.
- Strongest artifact readiness is in `spec-graph-bootstrap`, `spec-write-tasks`, `spec-code-review`, `spec-doc-review`, `spec-plan`, `spec-app-consistency-audit`, and `spec-skill-audit`.
- Standalone helper skills and older creative/productivity skills remain prompt-like and should either receive a compact contract block or be explicitly marked as non-Harness auxiliary skills.
- Four skills are not merely low-score files; they are semantic P0s because they expose internal helpers, central autonomous flow, or hidden mutating routes.
