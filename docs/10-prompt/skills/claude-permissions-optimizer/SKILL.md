# Claude 权限优化器

分析历史命令，找出可以安全放入 allowlist 的命令模式，并更新 Claude Code 的权限配置。

## 目标

减少重复的权限确认，同时保持权限范围可控。

## 工作流

### 第 1 步：收集命令历史

读取最近会话中的命令，重点关注高频、低风险、重复出现的命令。

优先考虑：

- 只读命令
- 明确受限的测试命令
- 固定前缀的开发命令
- 已被多次批准且行为稳定的命令

避免纳入：

- 破坏性命令
- 过宽泛的解释器入口
- 带任意脚本能力的前缀
- 会扩大权限边界的模糊模式

### 第 2 步：归一化命令

使用配套脚本对命令做清洗和归一化，提取适合作为 `prefix_rule` 的前缀。

### 第 3 步：评估安全性

对每个候选前缀做人工审查：

- 是否只覆盖稳定、可预期的行为
- 是否过宽
- 是否会无意中放行危险变体
- 是否真的值得长期免审批

### 第 4 步：更新配置

把通过审查的规则写入 Claude Code 对应的 `settings.json`。

更新时应：

- 保留已有用户配置
- 避免重复规则
- 保持配置结构合法

### 第 5 步：汇报结果

说明：

- 新增了哪些 allowlist 规则
- 为什么这些规则是安全的
- 哪些候选被拒绝，以及拒绝原因

## 判断标准

优先选择“窄而有用”的规则，而不是“广而方便”的规则。

一个好的规则应当：

- 能覆盖真实高频工作流
- 不允许任意代码执行
- 不会放大文件系统或网络风险
- 即使长期保留，也仍然合理
