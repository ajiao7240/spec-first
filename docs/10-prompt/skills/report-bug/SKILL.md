# 报告 Bug

当你确认 `spec-first` CLI、安装器、技能包或生成资产存在缺陷时，使用此技能整理一份高质量 bug 报告。

## 目标

产出一份可复现、可定位、可分派的缺陷报告，减少来回补充信息的成本。

## 收集内容

至少整理以下信息：

- 问题现象
- 期望行为
- 实际行为
- 复现步骤
- 影响范围
- 相关环境
- 相关日志、报错或截图

环境信息通常包括：

- 操作系统
- Shell / 终端环境
- `spec-first` 版本
- Node.js 版本
- 相关 CLI 命令
- 是否在 Claude Code / Codex / 其他宿主环境中运行

## 排查前置

提交报告前，先确认：

1. 问题可以稳定复现，或至少给出复现概率
2. 行为不是预期限制或配置问题
3. 已检查相关文档与近期变更
4. 报告中包含最小可复现场景

## 输出格式

建议使用如下结构：

```md
# Bug: <一句话标题>

## Summary
<问题概述>

## Environment
- OS:
- Shell:
- Node.js:
- spec-first:
- Host:

## Steps to Reproduce
1.
2.
3.

## Expected
<期望行为>

## Actual
<实际行为>

## Evidence
<日志、报错、截图、相关文件路径>

## Suspected Scope
<可能涉及的模块、skill、脚本或命令>
```

## 质量要求

- 标题要具体，避免笼统
- 复现步骤要可执行，避免主观描述
- 日志要贴关键片段，不要只说“报错了”
- 如果怀疑是回归问题，注明大致回归区间
