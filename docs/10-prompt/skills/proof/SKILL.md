# Proof

使用 Proof 管理 Markdown 文档、审阅意见和分享链接。

## 适用场景

- 新建一份可分享的文档
- 把本地 Markdown 推送到 Proof
- 对现有文档发表评论或建议
- 根据 Proof 链接拉取上下文继续编辑

## 工作流

### 第 1 步：确认对象

先明确任务类型：

- 新建文档
- 更新已有文档
- 添加评论
- 分享或生成链接

如果用户提供了 Proof URL，先解析对应文档。

### 第 2 步：准备内容

整理要提交的 Markdown 内容，保留：

- 标题层级
- 列表、表格、代码块
- 链接与附件引用

### 第 3 步：执行操作

通过 Proof 的 API 或本地桥接完成：

- 创建文档
- 更新正文
- 添加评论
- 获取分享链接

### 第 4 步：校验结果

确认：

- 文档已创建或更新成功
- 评论挂载在正确位置
- 返回链接可访问

## 输出要求

汇报时说明：

- 操作类型
- 目标文档
- 是否创建了新链接
- 如有失败，指出失败环节和原因
