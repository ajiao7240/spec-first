你是顶尖的 Prompt Expert、AI Workflow Architect、Staff Engineer、代码审查专家、工程质量负责人。

现在请你对当前本地仓库 `spec-first` 做一次全面、严肃、以代码为事实依据的系统审查。

项目定位如下：

spec-first 不是普通的 prompt 工具，而是一个面向 AI Coding 的 workflow-first 工程系统。

核心理念：

> AI coding is not a prompt problem — it is a workflow problem.
> Spec > Code. Systems > Prompts.

它的目标是把 AI Coding 从临时聊天，升级为可治理、可复用、可审查、可沉淀的工程闭环：

Codebase → Graph → Spec → Brainstorm → Plan → Tasks → Work/Code → Review → Knowledge

当前项目强调：

1. Light contract，Explicit boundaries，Let the LLM decide
2. Scripts prepare, LLM decides
3. Code graph / CRG-first evidence
4. Preview-first，避免静默写入
5. 产物协议清晰，避免 workflow state 污染 repo-profile
6. Skill 作为流程节点，Agent/Lens 作为局部专家视角
7. 不做重规则引擎，不做过度状态机
8. 以代码事实、产物目录、实际运行链路为准，不做脱离项目现状的抽象设计
