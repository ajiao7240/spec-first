# Spec-First Framework 目录结构

## 1. 目录视角

这个项目同时存在两种目录视角：

1. 仓库源码视角：`spec-first` npm 包本身包含的目录结构
2. 目标项目运行态视角：用户执行 `spec-first init --claude` 后，在目标项目里生成的 `.claude/` 目录

## 2. 仓库源码视角

```text
spec-first/
├── bin/
│   └── spec-first.js
├── src/
│   └── cli/
│       ├── index.js
│       ├── commands/
│       │   ├── doctor.js
│       │   └── init.js
│       ├── spec-commands.js
│       └── templates.js
├── templates/
│   └── claude/commands/spec/
│       ├── brainstorm.md
│       ├── plan.md
│       ├── work.md
│       ├── review.md
│       └── compound.md
├── docs/
├── tests/
│   ├── smoke/
│   │   ├── cli.sh
│   │   └── install-local.sh
│   └── integration/
│       └── e2e.sh
├── package.json
└── README.md
```

## 3. 目标项目运行态视角

`spec-first init --claude` 会在目标项目里生成：

```text
your-project/
└── .claude/
    ├── commands/
    │   └── spec/
    │       ├── brainstorm.md
    │       ├── plan.md
    │       ├── work.md
    │       ├── review.md
    │       └── compound.md
    ├── tasks/
    │   └── <task-id>/
    │       ├── task.yaml
    │       ├── 01-brainstorm.md
    │       ├── 02-plan.md
    │       ├── 03-work.md
    │       ├── 04-review.md
    │       └── 05-compound.md
    └── assets/
        ├── prompts/
        ├── workflows/
        ├── domain/
        └── cases/
```

## 4. 说明

- `templates/claude/commands/spec/` 是 npm 包内部模板来源，不是目标项目运行态目录
- `.claude/commands/spec/` 是用户真正执行 `/spec:*` 时的命令来源
- `.claude/tasks/` 和 `.claude/assets/` 属于目标项目的运行态数据，不属于 npm 包发布产物
- `skills/`、`agents/` 和 `scripts/` 属于仓库级开发与历史资产，不属于 npm 包发布产物
