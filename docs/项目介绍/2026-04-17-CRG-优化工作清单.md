## 消费路径说明（决定每项的真实受益链条）

`/spec:code-review`、`/spec:plan`、`/spec:work` **主路径是"预编译文档 + 静态路由"**，不直接调 CRG retrieval 管道：

```
spec-graph-bootstrap（周期性）
  ├─ 跑 CRG build 建图
  └─ 生成 docs/contexts/<slug>/*.md 预编译视图 + injection-index.yaml 路由
                │
                v
三个工作流运行时：evaluator.js 按 stage 挑 .md 文件 → 注入 prompt
```

CRG 的 `retrieveContext`（seed/expand/rerank/pack 管道）**只在** `crg search` / `crg context` / `crg review-context` CLI 被调用时生效。

**两种受益链条**：

- 🟢 **直接影响主路径**：改动立即让 `/spec:code-review|plan|work` 体感不同；
- 🟡 **间接基建**：改动要**重跑 `spec-graph-bootstrap --force` 重新生成 `docs/contexts/*.md`** 才会在主路径体感到；单独改完而不 bootstrap，主路径无变化；
- 🔵 **仅惠及 CLI**：只影响直接跑 `crg search / context / review-context` 的场景，不改主路径。

每项标签见下面分区。

### C2. `resolveEdges` AMBIGUOUS 缓存失效

- **受益**：🟡 间接基建（下次 bootstrap 建图更准）+ 🔵 CLI（resolveEdges 速度）
- **位置**：`src/crg/graph.js:255-280`
- **现象**：`name` 全局重名时每次调用都重跑 `getAllSymbolRows.all(name)`，缓存只省了 `rows.length` 判断。对 `run`/`build`/`parse` 等高频重名 symbol 是主瓶颈。
- **修复**：缓存从 `status` 改为 `{ status, rows }`，AMBIGUOUS 分支从缓存 rows 侧 filter srcFile，不再重查 SQL。

```js
const getSymbolId = (name, srcFile) => {
  let c = symbolCache[name];
  if (c === undefined) {
    const rows = getAllSymbolRows.all(name);
    c = rows.length === 0 ? { status: 'none' }
      : rows.length === 1 ? { status: 'unique', id: rows[0].id }
      : { status: 'ambiguous', rows };
    symbolCache[name] = c;
  }
  if (c.status === 'none') return null;
  if (c.status === 'unique') return c.id;
  const same = c.rows.filter(r => r.file_path === srcFile);
  return same.length === 1 ? same[0].id : null;
};
```

- **验证**：`tests/unit/crg-resolve-edges-cache.test.js`——spy `getAllSymbolRows.all`，断言同一 AMBIGUOUS name 多次调用只产生 1 次 SQL。

- **受益**：💎 **Phase 1a 里唯一直接打在主路径**——`crg review-context` 被 spec-code-review 的 reviewer subagent 直接调用做变更影响分析。C7 修完 `/spec:code-review` 立刻降噪。

// changes.js 新增
function getChangedHunksFromGit(repoRoot, since) {
  const out = execFileSync('git', ['diff', '--unified=0', since, 'HEAD'],
    { cwd: repoRoot, encoding: 'utf8', maxBuffer: 64 * 1024 * 1024 });
  const result = [];
  let file = null, hunks = [];
  for (const line of out.split('\n')) {
    const fm = line.match(/^\+\+\+ b\/(.+)$/);
    if (fm) { if (file) result.push({ file, hunks }); file = fm[1]; hunks = []; continue; }
    const hm = line.match(/^@@ -\d+(?:,\d+)? \+(\d+)(?:,(\d+))? @@/);
    if (hm) {
      const start = parseInt(hm[1], 10);
      const count = hm[2] ? parseInt(hm[2], 10) : 1;
      hunks.push({ start, end: start + count - 1 });
    }
  }
  if (file) result.push({ file, hunks });
  return result;
}

// changes.js:306 assessNodePriorities 签名改动
function assessNodePriorities(filePath, hunks, db) {
  const nodes = db.prepare(
    `SELECT id, name, kind, line_start, line_end FROM nodes
     WHERE file_path = ? AND kind IN ('function', 'method', 'class')`
  ).all(filePath);
  // 关键：先按 hunk 相交过滤，再打分
  const scoped = nodes.filter(n =>
    hunks.some(h => !(n.line_end < h.start || n.line_start > h.end))
  );
  if (scoped.length === 0) return { review_priorities: [], test_gaps: [] };
  // ... 对 scoped 打分
}
