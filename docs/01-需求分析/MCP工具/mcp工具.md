# MCP 工具分层建议（全栈项目）
**面向 Claude Code / Codex / Cursor / Windsurf 的默认安装建议**

## 决策结论

- 默认不要全装分析类 MCP
- 核心默认集控制在 `Serena + Sequential Thinking + Context7`
- `GitNexus`、`ABCoder`、`Code Index` 按项目复杂度启用
- `Playwright MCP` 保持测试场景可选

## 工具分层与边界

这组 MCP 不建议默认“全装全开”。

其中 `Serena`、`GitNexus`、`ABCoder`、`Code Index` 都参与代码理解，但关注层级不同：

- `Serena`：符号级定位与精确修改
- `GitNexus`：架构关系、调用图、依赖影响
- `ABCoder`：跨语言语义理解、复杂逻辑分析
- `Code Index`：轻量全局检索与上下文筛选

因此，推荐采用“核心默认 + 按需增强”模型，而不是把所有分析类工具都定义为默认必装。

| 工具 | 角色 | 边界 | 默认建议 |
|------|------|------|----------|
| **Serena** | 符号级精确编辑 | 不负责架构图谱 | ✅ **核心默认** |
| **Sequential Thinking** | 多步骤推理 | 不负责代码索引或文档查询 | ✅ **核心默认** |
| **Context7** | 最新官方文档查询 | 不负责本地代码理解 | ✅ **核心默认** |
| **GitNexus** | 架构关系与调用图 | 不替代精确编辑 | ⭐ **按需增强** |
| **ABCoder** | 跨语言语义理解 | 不替代架构图谱 | ⭐ **按需增强** |
| **Code Index** | 轻量全局检索 | 更适合作为补充，不应与上面两者一起默认全开 | 📌 **可选补充** |
| **Playwright MCP** | 前端 / App 自动化测试 | 不参与代码分析 | 📌 **场景可选** |

## 安装与接入约束

### 安装语义

这里的“全局安装”不是操作系统层面的“一次安装，所有客户端自动生效”，而是指：把 MCP 工具注册到每个已检测到的 AI 客户端的全局配置区，让该客户端后续会话可见。

### 前置条件

| 能力 | 说明 |
|---|---|
| Node.js | 仅对 Node 系列 MCP 运行时必要，版本以工具官方要求为准 |
| Python / uv / pip | 仅对 Python 系列 MCP 运行时必要，版本以工具官方要求为准 |
| 写权限 | 目标客户端的全局配置目录必须可写 |
| 网络 | 首次安装或更新时，允许拉取依赖与元数据 |

### 安装来源

- 以 `mcp工具.md` 的工具表作为需求输入源。
- 每个工具必须有可机器解析的安装元数据，包括安装源、启动命令、参数、验证方式。
- 安装器不得把“看起来像包名”的内容当成事实来源；缺少元数据时，必须跳过并提示补充。

### 标准流程

1. 探测本机已安装的 AI 客户端。
2. 只把“已检测到且可写”的客户端展示给用户选择。
3. 用户确认后，按客户端逐个执行安装或更新。
4. 写入前备份现有配置。
5. 已存在且定义一致的 MCP 跳过。
6. 已存在但定义变化的 MCP 执行更新。
7. 不存在的 MCP 执行安装。
8. 写入后重新读取并校验结果。
9. 失败时仅回滚当前客户端，不影响其他客户端。

### 异常处理

| 异常 | 处理方式 |
|---|---|
| 配置文件不存在 | 创建新配置或新目录后继续 |
| 配置文件不可写 | 跳过该客户端并记录原因 |
| 包解析失败 | 停止当前工具安装，保留备份 |
| 网络失败 | 重试一次，仍失败则记录失败并继续 |
| 格式校验失败 | 回滚当前客户端配置 |
| 并发修改 | 重新读取后再合并一次，仍冲突则退出当前客户端 |

### 验证规则

安装完成后，必须验证以下内容：

1. 目标 MCP 条目已存在。
2. 启动命令与参数完整。
3. 其他既有 MCP 未被破坏。
4. 如果客户端支持列出已安装 MCP 的命令，则优先做二次验证。

### v1 默认安装集

v1 默认只处理以下 3 项：

| 工具 | 角色 |
|---|---|
| `Serena` | 符号级精确编辑 |
| `Sequential Thinking` | 反思性多步骤推理 |
| `Context7` | 最新框架文档查询 |

### v1 增强安装集

以下工具不建议默认全装，按项目复杂度启用：

| 工具 | 适用场景 |
|---|---|
| `GitNexus` | 需要架构关系、调用图、依赖影响分析 |
| `ABCoder` | 需要跨语言语义理解、复杂逻辑梳理 |
| `Code Index` | 需要更轻量的全局检索，或作为上面两者的补充 |
| `Playwright MCP` | 需要前端/App 自动化测试 |

### 非目标

- 不将“客户端识别”扩展为“客户端自动安装”。
- 不承诺单一命令可让所有客户端自动生效。
- 不对未检测到的客户端执行写入。

## 实战搭配

最小可用组合：
- `Serena` + `Sequential Thinking` + `Context7`

复杂仓库组合：
- 先看架构 → `GitNexus`
- 再理解复杂逻辑 → `ABCoder`
- 最后精准改代码 → `Serena`

轻量仓库组合：
- 先全局检索 → `Code Index`
- 再定位与修改 → `Serena`
