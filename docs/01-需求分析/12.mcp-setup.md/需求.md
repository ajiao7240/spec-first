**生产版增强方案**

1. **跨平台兼容**：macOS / Linux / WSL / Windows 分开处理
2. **路径差异**：按官方系统级路径写 Claude Code；Codex 用用户级配置，Unix 额外兼容 `/etc/codex/config.toml` 的优先级模型 ([OpenAI开发者][1])
3. **健壮性**：同名 MCP 段替换、管理员权限不足自动回退、支持卸载

我先给结论：

* **Codex**

  * 用户级配置：`~/.codex/config.toml`。官方明确支持。([OpenAI开发者][1])
  * Unix 上还存在系统级 `/etc/codex/config.toml` 配置层。([OpenAI开发者][1])
* **Claude Code**

  * 系统级 MCP 托管文件 `managed-mcp.json`：

    * macOS: `/Library/Application Support/ClaudeCode/managed-mcp.json`
    * Linux / WSL: `/etc/claude-code/managed-mcp.json`
    * Windows: `C:\Program Files\ClaudeCode\managed-mcp.json`。([Claude][2])
  * 用户级常规设置在 `~/.claude/settings.json`，但 **MCP 的系统级独占托管** 用的是 `managed-mcp.json`。([Claude][2])

你这套 MCP 的官方启动路径我按下面固化：

* **Serena**：`uv` + repo + `uv run --directory ... serena-mcp-server [project.yml]`。([GitHub][3])
* **Context7 本地版**：本地 MCP 模式可用；Context7 官方说明同时支持 `CLI + Skills` 与 `MCP`。([GitHub][4])
* **Playwright MCP**：官方文档支持 MCP；官方仓库也明确提到 coding agent 场景下 CLI+skills 可能更省。([Playwright][5])
* **Sequential Thinking**：官方 servers 仓库的 `@modelcontextprotocol/server-sequential-thinking`。([GitHub][6])

# 一、Unix 生产版安装器

适用：**macOS / Linux / WSL**

特点：

* 幂等执行
* 同名 `[mcp_servers.xxx]` 会替换，不会无限追加
* Claude Code 优先写系统级 `managed-mcp.json`
* Codex 默认写用户级 `~/.codex/config.toml`
* 统一用 wrapper，后续改参数只改 wrapper

```bash
#!/usr/bin/env bash
set -Eeuo pipefail

MCP_ROOT="${MCP_ROOT:-/opt/mcp}"
BIN_DIR="${BIN_DIR:-/usr/local/bin}"
SERENA_DIR="${SERENA_DIR:-$MCP_ROOT/serena}"
SERENA_PROJECT_YML="${SERENA_PROJECT_YML:-}"
INSTALL_USER="${SUDO_USER:-$USER}"
INSTALL_HOME="$(eval echo "~$INSTALL_USER")"

OS="$(uname -s)"
case "$OS" in
  Darwin)
    CLAUDE_MANAGED="/Library/Application Support/ClaudeCode/managed-mcp.json"
    ;;
  Linux)
    CLAUDE_MANAGED="/etc/claude-code/managed-mcp.json"
    ;;
  *)
    echo "Unsupported OS: $OS"
    exit 1
    ;;
esac

CODEX_USER_CONFIG="$INSTALL_HOME/.codex/config.toml"
CLAUDE_USER_MCP="$INSTALL_HOME/.claude.json"

need_cmd() {
  command -v "$1" >/dev/null 2>&1 || {
    echo "Missing command: $1"
    exit 1
  }
}

ensure_uv() {
  if ! command -v uv >/dev/null 2>&1; then
    curl -LsSf https://astral.sh/uv/install.sh | sh
    export PATH="$HOME/.local/bin:$PATH"
  fi
  need_cmd uv
}

replace_toml_block() {
  local file="$1"
  local section="$2"
  local content="$3"

  mkdir -p "$(dirname "$file")"
  touch "$file"

  python3 - "$file" "$section" "$content" <<'PY'
import re, sys
from pathlib import Path

path = Path(sys.argv[1])
section = sys.argv[2]
content = sys.argv[3].strip() + "\n"

text = path.read_text(encoding="utf-8") if path.exists() else ""
pattern = rf'(?ms)^\[mcp_servers\.{re.escape(section)}\]\n.*?(?=^\[|\Z)'

if re.search(pattern, text):
    text = re.sub(pattern, content + "\n", text)
else:
    if text and not text.endswith("\n"):
        text += "\n"
    text += "\n" + content + "\n"

path.write_text(text, encoding="utf-8")
PY
}

write_claude_json_merge() {
  local target="$1"

  python3 - "$target" "$BIN_DIR" <<'PY'
import json, sys
from pathlib import Path

target = Path(sys.argv[1])
bin_dir = sys.argv[2]

data = {}
if target.exists():
    try:
        data = json.loads(target.read_text(encoding="utf-8"))
    except Exception:
        data = {}

mcp = data.get("mcpServers", {})
mcp["serena"] = {"type": "stdio", "command": f"{bin_dir}/serena-mcp-global", "args": []}
mcp["context7"] = {"type": "stdio", "command": f"{bin_dir}/context7-mcp-global", "args": []}
mcp["playwright"] = {"type": "stdio", "command": f"{bin_dir}/playwright-mcp-global", "args": []}
mcp["sequential-thinking"] = {"type": "stdio", "command": f"{bin_dir}/sequential-thinking-mcp-global", "args": []}
data["mcpServers"] = mcp

target.parent.mkdir(parents=True, exist_ok=True)
target.write_text(json.dumps(data, indent=2, ensure_ascii=False) + "\n", encoding="utf-8")
PY
}

echo "==> Checking dependencies"
need_cmd git
need_cmd curl
need_cmd node
need_cmd npm
need_cmd npx
need_cmd python3
ensure_uv

echo "==> Preparing directories"
sudo mkdir -p "$MCP_ROOT" "$BIN_DIR"
sudo chown -R "$INSTALL_USER":"$(id -gn "$INSTALL_USER")" "$MCP_ROOT"

echo "==> Installing/updating Serena"
if [[ ! -d "$SERENA_DIR/.git" ]]; then
  sudo -u "$INSTALL_USER" git clone https://github.com/oraios/serena.git "$SERENA_DIR"
else
  sudo -u "$INSTALL_USER" git -C "$SERENA_DIR" pull --ff-only
fi

if [[ ! -f "$SERENA_DIR/serena_config.yml" && -f "$SERENA_DIR/serena_config.template.yml" ]]; then
  sudo -u "$INSTALL_USER" cp "$SERENA_DIR/serena_config.template.yml" "$SERENA_DIR/serena_config.yml"
fi

echo "==> Writing wrappers"
sudo tee "$BIN_DIR/serena-mcp-global" >/dev/null <<EOF
#!/usr/bin/env bash
set -euo pipefail
SERENA_DIR="${SERENA_DIR}"
PROJECT_YML="\${SERENA_PROJECT_YML:-${SERENA_PROJECT_YML}}"
if [[ -n "\$PROJECT_YML" ]]; then
  exec uv run --directory "\$SERENA_DIR" serena-mcp-server "\$PROJECT_YML"
else
  exec uv run --directory "\$SERENA_DIR" serena-mcp-server
fi
EOF

sudo tee "$BIN_DIR/context7-mcp-global" >/dev/null <<'EOF'
#!/usr/bin/env bash
set -euo pipefail
exec npx -y @upstash/context7-mcp
EOF

sudo tee "$BIN_DIR/playwright-mcp-global" >/dev/null <<'EOF'
#!/usr/bin/env bash
set -euo pipefail
exec npx -y @playwright/mcp@latest
EOF

sudo tee "$BIN_DIR/sequential-thinking-mcp-global" >/dev/null <<'EOF'
#!/usr/bin/env bash
set -euo pipefail
exec npx -y @modelcontextprotocol/server-sequential-thinking
EOF

sudo chmod +x \
  "$BIN_DIR/serena-mcp-global" \
  "$BIN_DIR/context7-mcp-global" \
  "$BIN_DIR/playwright-mcp-global" \
  "$BIN_DIR/sequential-thinking-mcp-global"

echo "==> Installing Playwright browser dependency"
npx -y playwright install chromium || true

echo "==> Updating Codex user config"
replace_toml_block "$CODEX_USER_CONFIG" "serena" "
[mcp_servers.serena]
command = \"$BIN_DIR/serena-mcp-global\"
startup_timeout_sec = 30
tool_timeout_sec = 180
"
replace_toml_block "$CODEX_USER_CONFIG" "context7" "
[mcp_servers.context7]
command = \"$BIN_DIR/context7-mcp-global\"
startup_timeout_sec = 30
tool_timeout_sec = 180
"
replace_toml_block "$CODEX_USER_CONFIG" "playwright" "
[mcp_servers.playwright]
command = \"$BIN_DIR/playwright-mcp-global\"
startup_timeout_sec = 30
tool_timeout_sec = 240
"
replace_toml_block "$CODEX_USER_CONFIG" "sequential-thinking" "
[mcp_servers.sequential-thinking]
command = \"$BIN_DIR/sequential-thinking-mcp-global\"
startup_timeout_sec = 30
tool_timeout_sec = 180
"

chown -R "$INSTALL_USER":"$(id -gn "$INSTALL_USER")" "$(dirname "$CODEX_USER_CONFIG")"

echo "==> Updating Claude Code config"
if sudo test -w "$(dirname "$CLAUDE_MANAGED")" 2>/dev/null || [[ ! -e "$CLAUDE_MANAGED" ]]; then
  sudo mkdir -p "$(dirname "$CLAUDE_MANAGED")"
  sudo python3 - "$CLAUDE_MANAGED" "$BIN_DIR" <<'PY'
import json, sys
from pathlib import Path
target = Path(sys.argv[1])
bin_dir = sys.argv[2]

data = {}
if target.exists():
    try:
        data = json.loads(target.read_text(encoding="utf-8"))
    except Exception:
        data = {}

mcp = data.get("mcpServers", {})
mcp["serena"] = {"type": "stdio", "command": f"{bin_dir}/serena-mcp-global", "args": []}
mcp["context7"] = {"type": "stdio", "command": f"{bin_dir}/context7-mcp-global", "args": []}
mcp["playwright"] = {"type": "stdio", "command": f"{bin_dir}/playwright-mcp-global", "args": []}
mcp["sequential-thinking"] = {"type": "stdio", "command": f"{bin_dir}/sequential-thinking-mcp-global", "args": []}

target.write_text(json.dumps(data, indent=2, ensure_ascii=False) + "\n", encoding="utf-8")

  chown "$INSTALL_USER":"$(id -gn "$INSTALL_USER")" "$CLAUDE_USER_MCP"

fi

echo
echo "Done."
echo "Codex user config: $CODEX_USER_CONFIG"
echo "Wrappers:"
echo "  $BIN_DIR/serena-mcp-global"
echo "  $BIN_DIR/context7-mcp-global"
echo "  $BIN_DIR/playwright-mcp-global"
echo "  $BIN_DIR/sequential-thinking-mcp-global"
```

```bash
  "$BIN_DIR/serena-mcp-global" \
  "$BIN_DIR/context7-mcp-global" \
```

# 三、Windows 生产版安装器

适用：**原生 Windows PowerShell，建议管理员运行**

特点：

* 写 `C:\Program Files\ClaudeCode\managed-mcp.json`；无权限时回退到用户级 `~\.claude.json`。Claude Code 官方系统级文件路径就是这里。([Claude][2])
* Codex 写用户级 `%USERPROFILE%\.codex\config.toml`，这是官方公开路径。([OpenAI开发者][1])
* wrapper 用 `.cmd`

```powershell
$ErrorActionPreference = "Stop"

$McpRoot = if ($env:MCP_ROOT) { $env:MCP_ROOT } else { "C:\ProgramData\MCP" }
$BinDir = Join-Path $McpRoot "bin"
$SerenaDir = Join-Path $McpRoot "serena"
$SerenaProjectYml = $env:SERENA_PROJECT_YML

$CodexConfig = Join-Path $HOME ".codex\config.toml"
$ClaudeManaged = "C:\Program Files\ClaudeCode\managed-mcp.json"
$ClaudeUser = Join-Path $HOME ".claude.json"

function Need-Cmd($name) {
  if (-not (Get-Command $name -ErrorAction SilentlyContinue)) {
    throw "Missing command: $name"
  }
}

function Ensure-Uv {
  if (-not (Get-Command uv -ErrorAction SilentlyContinue)) {
    irm https://astral.sh/uv/install.ps1 | iex
    $env:PATH += ";$HOME\.local\bin"
  }
  Need-Cmd uv
}

function Set-TomlBlock {
  param(
    [string]$Path,
    [string]$Section,
    [string]$Content
  )
  $dir = Split-Path $Path
  if (-not (Test-Path $dir)) { New-Item -ItemType Directory -Force -Path $dir | Out-Null }
  if (-not (Test-Path $Path)) { New-Item -ItemType File -Force -Path $Path | Out-Null }

  $text = Get-Content $Path -Raw
  $pattern = "(?ms)^\[mcp_servers\." + [regex]::Escape($Section) + "\]`r?`n.*?(?=^\[|\z)"
  if ([regex]::IsMatch($text, $pattern)) {
    $text = [regex]::Replace($text, $pattern, ($Content.Trim() + "`r`n`r`n"))
  } else {
    $text += "`r`n" + $Content.Trim() + "`r`n"
  }
  Set-Content -Encoding UTF8 $Path $text
}

function Merge-JsonMcp {
  param(
    [string]$Path
  )
  $dir = Split-Path $Path
  if (-not (Test-Path $dir)) { New-Item -ItemType Directory -Force -Path $dir | Out-Null }

  $data = @{}
  if (Test-Path $Path) {
    try { $data = Get-Content $Path -Raw | ConvertFrom-Json -AsHashtable } catch { $data = @{} }
  }
  if (-not $data.ContainsKey("mcpServers")) { $data["mcpServers"] = @{} }

  $data["mcpServers"]["serena"] = @{
    type = "stdio"; command = "$BinDir\serena-mcp-global.cmd"; args = @()
  }
  $data["mcpServers"]["context7"] = @{
    type = "stdio"; command = "$BinDir\context7-mcp-global.cmd"; args = @()
  }
  $data["mcpServers"]["playwright"] = @{
    type = "stdio"; command = "$BinDir\playwright-mcp-global.cmd"; args = @()
  }
  $data["mcpServers"]["sequential-thinking"] = @{
    type = "stdio"; command = "$BinDir\sequential-thinking-mcp-global.cmd"; args = @()
  }

  $data | ConvertTo-Json -Depth 8 | Set-Content -Encoding UTF8 $Path
}
```

# 五、这版比前一版更稳的原因

第一，**Claude Code 系统级路径按官方三平台路径处理**，而不是把 Linux 路径硬套到所有系统。Claude 官方文档明确列出了 `managed-mcp.json` 在 macOS、Linux/WSL、Windows 的位置。([Claude][2])

第二，**Codex 只把“公开文档确认过”的路径当作确定项**：用户级 `~/.codex/config.toml` 是确定的；Unix 上系统级 `/etc/codex/config.toml` 也在官方优先级文档里。Windows 的系统级 Codex 配置路径我没有硬编，因为官方页面里没有给出。([OpenAI开发者][1])

第三，**Serena 单独走 repo + uv**，因为它官方 MCP 安装说明就是这一路，不适合和其他纯 `npx` server 完全等同。([GitHub][3])

第四，**Context7 / Playwright / Sequential Thinking 都保留 wrapper 层**。这样你后面改参数、加 env、加日志，不需要重新改 Codex 和 Claude 两套配置。Context7 官方支持 MCP 模式，Playwright 官方支持 MCP 且单独维护了相关文档，Sequential Thinking 则来自 MCP 官方 servers 仓库。([GitHub][4])

第五，**Codex 其实还支持 Skills**。所以你后面如果想把 Context7 或 Playwright CLI 的工作流做得更强，可以在现有 MCP 之外再叠加 Codex skills，而不需要推翻这套安装器。([OpenAI开发者][7])
