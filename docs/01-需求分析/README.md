# docs/01-需求分析 说明

本目录收录 spec-first 的历史需求分析、探索方案、阶段设计稿和归档技术文档。

## 使用边界

- 本目录主要用于回溯背景、设计取舍和演进过程
- 其中部分文档描述的是历史阶段的运行时路径、命名或流程，**不保证**与当前实现完全一致
- 读取本目录时，应将其视为“设计史料”而不是“当前执行手册”

## 当前实现以哪些文档为准

1. `skills/` 与 `agents/`：当前运行时 source-of-truth
2. `docs/plans/`：当前实施计划与完成记录
3. `docs/02-架构设计/02-目录结构.md`：当前目录结构与运行时产物布局
4. `CLAUDE.md` / `AGENTS.md`：当前宿主治理规则
5. `CHANGELOG.md`：变更时间线与迁移记录

## 维护原则

- 新的功能实现不要直接修改本目录中的历史设计稿来追平现状
- 如果某类历史文档长期被误当成现行规范，应优先在目录入口补充说明或增加指向当前 source-of-truth 的链接
- 只有当某篇历史文档仍承担现行规范职责时，才应将其迁移到正式文档目录并持续维护
