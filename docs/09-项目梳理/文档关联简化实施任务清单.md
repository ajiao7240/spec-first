# 文档关联简化实施任务清单

> 执行要求：先审查本清单，再按 TDD 逐任务落地。每个任务都必须先有失败测试，再写最小实现，再回归。

## 1. 改造目标

本次改造的终态不是“矩阵瘦身”，而是**彻底移除矩阵模型**：

- 删除 `traceability-matrix.md`
- 删除 `FR / DS / TASK / TC` 作为运行时追踪真相源
- 删除 coverage/matrix 驱动的 gate、status、validate、trace 逻辑
- 新增 `document-links.yaml` 作为**唯一文档关联索引**
- 保留 `stage-state.json` 作为**唯一阶段状态真相源**
- 所有运行态检查只基于：
  - `stage-state.json`
  - 文档文件是否存在
  - `document-links.yaml` 中的文档引用关系
  - `reports/` 等证据文件是否存在

本次改造**不考虑向下兼容**。旧命令、旧字段、旧文件可以直接删除。

## 2. 非妥协约束

- 严格 DRY：新增的文档关联解析/校验逻辑只能有一份共享实现。
- 测试无妥协：优先补边界测试、回归测试、坏输入测试。
- 适度工程化：拒绝脆弱 hack，也拒绝为未来假设做抽象树。
- 周全优先速度：优先覆盖缺文件、坏 YAML、坏 stage、坏引用、半初始化 Feature。
- 显式优先巧妙：优先直接 if/switch，不引入隐式魔法层。
- 最小变更：先改最少文件打通主链路，再删除死代码。
- 代码与文档同步：代码行为改完，同批次更新 skill 文档、CLI 文档、测试夹具。

## 4. 目标模型

### 4.1 `stage-state.json`

只保留阶段状态，不再承载矩阵阈值和条目级追踪语义。

建议收敛为：

```ts
interface StageState {
  featureId: string;
  mode: Mode;
  size: Size;
  platforms: string[];
  backgroundInputStatus?: BackgroundInputStatus;
  stageStatus?: StageStatus;
  autoAdvancePolicy?: AutoAdvancePolicy;
  lastVerifiedAt?: string;
  lastSuggestedCommand?: string;
  currentStage: Stage;
  history: StageHistoryEntry[];
  terminal: boolean;
  title?: string;
  createdAt: string;
  updatedAt: string;
}
```

### 4.2 `document-links.yaml`

新增统一文档关联模型，建议在 `src/core/document-links.ts` 中集中实现。

```yaml
version: 1
featureId: FSREQ-20260323-AUTH-001
documents:
  - path: spec.md
    kind: requirements
    stage: 01_specify
    references: []
  - path: design.md
    kind: design
    stage: 02_design
    references:
      - spec.md
  - path: task_plan.md
    kind: plan
    stage: 03_plan
    references:
      - spec.md
      - design.md
```

### 4.3 允许的检查维度

只允许以下检查：

- 必需文档是否存在
- 文档引用目标是否存在
- 文档引用是否满足阶段要求
- Release / Verify 证据文档是否存在
- Gate 历史、findings、reports 是否存在且格式可读

不再允许以下运行时检查：

- FR 是否有 DS/TASK/TC
- coverage 是否达到矩阵阈值
- matrix status 是否终态
- ID 链是否完整
