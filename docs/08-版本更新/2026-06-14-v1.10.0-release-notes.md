# spec-first v1.10.0 版本说明

`v1.10.0` 是一次 AI Coding Harness 的治理与运行时架构升级，重点提升 workflow 的证据可信度、运行边界、双宿主一致性和任务收口质量。

## 主要变化

- 退役旧的 GitNexus / graph-bootstrap 主路径。`/spec:graph-bootstrap` 和 `$spec-graph-bootstrap` 不再作为当前公开入口；workflow 默认回到直接源码、git diff、测试、日志和用户证据。
- Runtime Setup 升级为统一的 harness readiness 检查。`/spec:mcp-setup` / `$spec-mcp-setup` 负责 MCP/helper/provider readiness facts，并保持 provider 输出为 advisory evidence。
- 新增 `spec-first update` CLI 命令。它会执行 `npm install -g spec-first@latest`，升级成功后提示运行 `spec-first init` 刷新本项目 runtime assets。
- `spec-first init` 体验优化：会记住上次选择的 Claude Code / Codex host，并可复用全局 developer profile，减少重复提问。
- Codex / Claude 双宿主治理加强。Codex `$spec-*` workflow 不再默认授权 subagent dispatch，缺少显式授权时会走 documented fallback。
- Work closeout 和验证闭环增强。新增 verification profile、verification run summary 和 honest closeout 机制，避免只用自然语言声明“已验证”。
- PRD、Plan、Work、Knowledge workflow 都有增强：更强调 source evidence、change boundary、task handoff、doc-review 续接和 `docs/solutions/` 知识回源确认。
- Eval readiness 规范化。workflow eval fixture 改为基于 declared coverage tags / bucket 的结构化证据，减少靠文件名推断 readiness 的脆弱性。
- 文档运营内容迁出：`docs/11-文章系列` 已迁移到独立文档运营仓，本仓更聚焦 spec-first source、contract、workflow 和 runtime 资产。

## 升级指引

通过 npm 安装的用户运行：

```bash
spec-first update
spec-first init
```

通过 Claude Code plugin 安装的用户，请使用：

```bash
claude plugin update
spec-first init
```

升级后请重启 Claude Code / Codex，确保宿主加载新的 runtime assets。

## 兼容性说明

这是一个用户可见的治理升级版本。依赖 `$spec-graph-bootstrap`、`/spec:graph-bootstrap`、`$spec-update` 或旧 GitNexus readiness artifacts 的使用方式，需要迁移到新的 Runtime Setup 与 direct evidence workflow 口径。
