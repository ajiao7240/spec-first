## 摘要

`spec-first@1.11.0` 是一次以「治理可信度、Runtime Setup、跨工作流验证证据」为主线的功能版本。相比 v1.10.0，本版让 provider（CodeGraph / Graphify）的安装与版本来源更收敛、让各工作流在收尾时留下可核对的证据、并打磨了从需求（PRD）到计划、任务、知识沉淀的链路体验。`update` 命令的行为也有一处用户可感知的增强（见升级注意事项）。

## 亮点

- **Runtime Setup 走向 provider-native**：CodeGraph / Graphify 作为显式 opt-in 能力，安装口径、运行时可见性与版本钉选更稳定一致。
- **跨工作流的收尾证据**：多个工作流在结束时会留下可核对的 closeout 证据，让「做完了什么、验证到哪一步」更透明。
- **审查→整改闭环可追踪**：评审发现可被登记并追踪到对应的整改，知识判断会回流到项目基线。

## 新增

- `update` 命令现在会执行依赖升级并引导你跑 `init`，不再只是「检查版本」。
- Runtime Setup 收口 provider-native 方案：CodeGraph / Graphify 的安装、可见性探测与版本钉选集中管理。
- 各工作流新增收尾证据沉淀，并支持 provider 协调与任务交接。
- `spec-write-tasks` 补齐 agent-native 能力，对高风险任务包可自动续接文档评审。
- `init` 会记住并预勾选上次选择的 host（Claude / Codex）。
- `spec-plan` 新增 plan guard 生命周期；`spec-prd` 增强多源输入指引、PRD 净化与 Markdown 功能切片，并引入自适应「产品专家」质量闭环。
- 落地 Knowledge Harness，让经验沉淀与跨会话复用更系统。

## 修复

- 修复 Codex 侧 SessionStart hook 的重复注入问题，新会话上下文更干净。
- 收敛 `using-spec-first` 的分流与 Codex 派遣治理，路由更可预期。
- 修复 Graphify 在 Runtime Setup 中的可见性与版本钉选若干问题。
- 移除多余的安装期 lifecycle 脚本，安装更干净、无额外告警。
- 一批治理与双宿主（Claude / Codex）一致性的闭环修复。

## 升级注意事项

- **`update` 行为变化**：升级到本版后，`spec-first update` 会实际执行升级并引导你运行 `init`（此前仅做检查）。首次运行请留意它会提示初始化步骤。
- CodeGraph / Graphify 为显式 opt-in；如需图能力，按引导用 `--only codegraph,graphify` 安装，当前钉选版本为 `@colbymchenry/codegraph@1.0.1`、`graphifyy==0.8.39`。
- 本版自 v1.10.0 起变更较多，上面仅列面向使用者的要点；完整改动清单请参阅仓库 CHANGELOG 或 `git log v1.10.0..v1.11.0`。
