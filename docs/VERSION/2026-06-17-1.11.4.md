## 摘要

本版是 `spec-first` 自 v1.11.3 起的一次治理 prose 收口发布，核心是同步 `AGENTS.md` 的「Workflow 入口治理」与「任务分级」措辞，使 host 入口文档与 `using-spec-first` 路由策略对齐。无破坏性变更，无 CLI 行为变化。

## 亮点

- substantial work 阈值补「明确单点、低风险的小代码/文案/config 修正可直接执行」语义，减少不必要的 workflow 触发，同时保留 CHANGELOG、最窄验证与 source/runtime 边界纪律。
- 任务分级中的「小任务」明确为默认直接执行，措辞与 `CLAUDE.md`、`skills/using-spec-first/SKILL.md` 的判据保持一致。

## 修复

- 收口 `AGENTS.md` Workflow 入口治理与任务分级两段 prose 的措辞漂移，使其与已落地的 routing/codex-hook 行为一致（路由与 codex-hook 的具体实现细化已分别记录在各 `v1.11.3` CHANGELOG 条目）。

## 升级注意事项

- 升级方式不变：`spec-first update` 或重新 `npm i -g spec-first@1.11.4`。
- 本版仅涉及 host 入口文档 prose，无需迁移；如本地已自定义 `AGENTS.md`，升级后建议确认 managed bootstrap block 是否需重新生成（`spec-first init`）。
