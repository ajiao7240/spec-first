## 摘要

`spec-first@1.8.2` 是 v1.8.1 的稳定性与能力补强更新。本版主要带来发布包 evidence 归档能力（P2-007）、AI 开发基准 fixture 套件（P2-003 v1）、以及多处工作流加固修复——包括 pnpm 发布流水线修复、graph bootstrap 稳定性、跨平台打包验证强化，以及 `spec-mcp-setup` 和 `spec-standards` 的 workspace 初始化加固。无破坏性变更。

## 亮点

- 发布时自动生成包内容清单（package content manifest）与 tarball-installed init dry-run 验证报告，为每次发布留下可审计的 evidence 归档
- 新增 AI 开发基准 fixture suite v1，含 advisory gate 聚合脚本与 deterministic result artifact，支撑可度量的 workflow 质量评估
- graph-bootstrap 新增 timing 和 freshness facts，提升外部 graph provider 就绪状态的可观测性

## 新增

- `release package evidence`（P2-007）：每次发布后自动产出 package content manifest、tarball-installed init dry-run 日志与 release artifact summary，路径约定写入 `docs/contracts/release-package-evidence.schema.json`
- `AI dev benchmark fixture suite v1`（P2-003）：包含 3 个基准 fixture、advisory gate 聚合与脚本/LLM 职责边界，支撑 spec-first workflow 的可度量质量评估
- `graph-bootstrap` timing 和 freshness facts：向下游 workflow 提供 graph provider 的时间戳和新鲜度信息
- `spec-standards` workspace targeting boundaries：为多仓 workspace 场景添加明确的范围边界约束

## 修复

- `spec-mcp-setup` provider setup 和 workspace init 加固（TASK-MCP-SETUP-001/002）：修复 graph bootstrap 和 init 边缘案例
- 发布流水线：给 `pnpm publish` 加入 `--no-git-checks`，修复版本 bump 后工作区变脏导致发布被拦截的问题
- 发布包跨平台验证强化：改善 Windows/macOS/Linux 打包一致性检查
- `using-spec-first` 入口治理收紧（TASK-GOVERNANCE-002）：收敛路由合同测试，减少误分流
- graph bootstrap 与 doc review 稳定性：修复 GitNexus 宿主 block 刷新和证据稳定性问题
- 多项 P1 全面审查 finding 关闭：包含 release gate 前移、managed state atomic write、managed path containment 等

## 升级注意事项

- 本版无 API 或 CLI 行为的破坏性变更，可直接升级。
- 发布包 evidence 功能（P2-007）需 `docs/VERSION/` 已从 `npm pack` 排除（通过 `.npmignore` 或 `package.json#files`），否则 `release:publish` preflight 将拒绝发布——若遇此问题请先更新 `.npmignore`。
- 本次变更较多，完整清单请参阅 CHANGELOG 或 `git log v1.7.3..v1.8.2`。
