## 摘要

本版在 `spec-first` 1.11.4 基础上做了一轮治理能力扩展与工作流质量收口。最大的新增是团队开发规范治理层：你现在可以把团队规范当作 source 文档来查询、审计、起草候选并准备晋升/废弃提案。同时 PRD、Plan、Write-Tasks、Brainstorm、Debug 等工作流的质量信号和入口边界都得到加强，整体让从需求到落地的链路更可治理、可验证。无破坏性变更，可直接升级。

## 亮点

- 新增团队开发规范治理能力：以确认规范为准查询、审计规范健康度、管理候选规范并准备晋升/废弃提案，候选内容保持 advisory，不会被当成既定事实。
- PRD 工作流引入需求澄清前置环节，并增强质量信号，帮助更早暴露需求歧义与范围风险。
- Debug 工作流接入历史经验检索，定位问题时能复用 `docs/solutions` 中沉淀的过往结论。
- 初始化时支持同步用户语言偏好，新仓库的输出语言更贴合实际使用习惯。

## 新增

- 团队规范治理 standalone 能力：支持规范查询、初始化、审计、候选起草与晋升/废弃提案流程。
- PRD 工作流的需求澄清前置环节，进入正式撰写前先收敛模糊点。
- Debug 工作流的历史经验召回，把已沉淀的解决方案接入问题定位。
- 初始化时的用户语言偏好同步。

## 修复

- 修正初始化时对 standalone 规范类能力命名投影的处理，使其在 Claude 与 Codex 两套宿主下保持一致。
- 规范团队规范治理能力在 Claude（`/spec:*`）与 Codex（`$spec-*`）下的命令写法表述。
- 加强版本提醒的冷却控制与帮助输出的静默行为。

## 升级注意事项

- 直接升级即可，无破坏性变更：`npm install -g spec-first`。
- 升级后如需刷新本地宿主运行时资产，运行 `spec-first init`。
- 本次变更较多，完整清单请参阅 CHANGELOG 或 git log。
