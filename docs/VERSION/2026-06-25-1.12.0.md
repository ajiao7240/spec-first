## 摘要

本版在 `spec-first` 1.11.5 基础上聚合了一轮 PRD 工作流的韧性重构，并新增了用于编写 skill 的 standalone 能力 `spec-write-skill`（source skill 数量从 36 增至 37）。`$spec-prd` 现在在写 PRD 之前会显式区分写作模式与澄清证据，对 owner 必答问题、design-source 阅读完整性和完成态伪装做了更强的闭环约束。同时 task-pack 与用户手册按多轮评审收敛。无破坏性变更，可直接升级。

## 亮点

- PRD 工作流引入 `write_mode` 与 `clarification_evidence` 双轴：写 PRD 前先判定 ask-owner-first / checkpoint-prd / final-prd / route-out，并记录澄清证据来源，更早暴露未闭合的需求与范围风险。
- 新增 Pre-Write Closure Gate 与 source-proven 反仪式化边界，避免把未澄清问题写成「看似完成」的 PRD。
- Figma/design-source 升级为 `design_source_inventory` + 阅读覆盖率 readiness：影响 UI 结构、状态、交互、验收或范围的设计节点未读完时阻塞 readiness。
- 新增 `spec-write-skill`：编写、改写、迁移或按 audit findings 修复 spec-first source skill 的 standalone 能力。

## 新增

- `spec-write-skill` standalone skill，覆盖 skill 触发、边界、I/O、渐进披露、resources/evals 与治理验证。
- `$spec-prd` 的 write_mode / clarification_evidence 判定、Pre-Write Closure Gate，以及 `check-prd-artifact.js` 的 advisory facts 与 findings。
- Product Expert Lens 三段清晰度重构与 grill-first 澄清实现。

## 修复

- 校准 `$spec-prd` 的 deep clarification 与 grill-with-docs 触发策略。
- 收敛 task-pack：修正 README 越界与 review_gate 状态化回归，折入 doc-review 的 P1/P2 findings。
- 校准用户手册 PRD 质量增强流程图与 Context/ADR Topology Adapter 边界。

## 升级注意事项

- 直接升级即可，无破坏性变更：`npm install -g spec-first`。
- 升级后如需刷新本地宿主运行时资产，运行 `spec-first init`。
- 本次变更较多，完整清单请参阅 CHANGELOG 或 git log。
