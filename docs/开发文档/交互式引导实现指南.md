# 交互式引导实现指南

> 如何在 Spec-First Skill 中实现单选、多选、自定义输入和多步骤交互

---

## 核心工具：AskUserQuestion

Claude Code 提供的交互工具，支持：
- ✅ 单选（radio）
- ✅ 多选（checkbox）
- ✅ 自定义输入（自动提供 "Other" 选项）
- ✅ 多步骤（一次最多 4 个问题）

---

## 基础用法

### 1. 单选（默认）

```markdown
使用 `AskUserQuestion` 工具收集用户信息：

**Q1: 你的主要角色是？**

选项：
- 开发者（写代码）
- 产品经理（写需求）
- 测试工程师（写测试）
- 架构师（做设计）
```

**AI 执行时的调用**：
```javascript
{
  "questions": [
    {
      "header": "角色",
      "question": "你的主要角色是？",
      "multiSelect": false,  // 单选（默认）
      "options": [
        {
          "label": "开发者",
          "description": "写代码"
        },
        {
          "label": "产品经理",
          "description": "写需求"
        },
        {
          "label": "测试工程师",
          "description": "写测试"
        },
        {
          "label": "架构师",
          "description": "做设计"
        }
      ]
    }
  ]
}
```

### 3. 自定义输入

**自动支持**：用户总是可以选择 "Other" 并输入自定义文本

```markdown
**Q1: 你的主要角色是？**

选项：
- 开发者（写代码）
- 产品经理（写需求）
- 测试工程师（写测试）
- 架构师（做设计）

**注意**：用户可以选择"Other"自定义角色名称（如：运维工程师、技术经理等）
```

**用户选择 "Other" 后**：
- 系统自动提供文本输入框
- 用户输入自定义内容（如："运维工程师"）
- AI 收到的答案就是用户输入的文本

### 4. 多步骤交互

#### 方式 A：一次调用多个问题（推荐，最多 4 个）

```markdown
使用 `AskUserQuestion` 工具一次收集 3 个问题：

**Q1: 你的主要角色是？**
**Q2: 你想做什么？**
**Q3: 项目规模？**
```

**AI 执行时的调用**：
```javascript
{
  "questions": [
    {
      "header": "角色",
      "question": "你的主要角色是？",
      "multiSelect": false,
      "options": [...]
    },
    {
      "header": "任务",
      "question": "你想做什么？",
      "multiSelect": false,
      "options": [...]
    },
    {
      "header": "规模",
      "question": "项目规模？",
      "multiSelect": false,
      "options": [...]
    }
  ]
}
```

#### 方式 B：多次调用（用于条件分支）

```markdown
### Phase 1: 第一步问题

**Q1: 你是否已有项目分析数据？**

选项：
- 是，已运行过 /spec-first:first
- 否，还没有

### Phase 2: 根据答案决定后续

**若用户选择"否"**：
- 引导执行 /spec-first:first
- 完成后继续下一步

**若用户选择"是"**：
- 直接进入场景识别
- 询问角色、任务、规模
```

## 最佳实践

### ✅ DO

1. **合并相关问题**：一次调用收集多个相关问题（最多 4 个）
2. **清晰的 header**：用简短标签（如"角色"、"任务"）
3. **完整的 description**：每个选项都有说明
4. **提示自定义输入**：在 SKILL.md 中说明用户可以选择 "Other"

### ❌ DON'T

1. **不要超过 4 个问题**：一次调用最多 4 个
2. **不要少于 2 个选项**：每个问题至少 2 个选项
3. **不要超过 4 个选项**：每个问题最多 4 个选项
4. **不要忘记 multiSelect**：多选时必须设置 `multiSelect: true`

---

## 参数说明

### questions 数组

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| header | string | ✅ | 短标签（最多 12 字符），如"角色"、"任务" |
| question | string | ✅ | 完整问题，如"你的主要角色是？" |
| multiSelect | boolean | ✅ | true=多选，false=单选（默认） |
| options | array | ✅ | 选项列表（2-4 个） |

### options 数组

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| label | string | ✅ | 选项显示文本（1-5 词） |
| description | string | ✅ | 选项说明，解释含义或影响 |

## 实战案例：init Skill 的逐步引导

参考 `skills/spec-first/01-init/SKILL.md`，展示如何实现复杂的多步骤交互：

### 设计原则

```markdown
## 交互引导要求（强制）
- P2 必须使用**逐步交互式引导**，禁止一次性要求用户提交全部字段
- 推荐顺序：feat → mode → size → platforms → title → feature-id → bootstrap
- mode/size 必须给出编号选项与默认值（支持直接回车采用默认）
- platforms 必须展示可选列表，以多选交互收集结果
- 参数确认阶段必须回显最终选择后再执行
```

### 实现步骤

**Step 1: 收集 feat（文本输入）**
```markdown
**Q1: Feature 缩写（2-16 个大写字母）**

示例：AUTH、REPORT、URPT

约束：必须匹配 ^[A-Z][A-Z0-9]{0,15}$
```

**Step 2: 收集 mode（单选 + 默认值）**
```markdown
**Q2: 开发模式？**

选项：
1. Normal - 正常开发（默认）
2. Incremental - 增量开发

提示：直接回车使用默认值（Normal）
```

**Step 3: 收集 size（单选 + 默认值）**
```markdown
**Q3: 项目规模？**

选项：
1. Small - 小型（默认）
2. Medium - 中型
3. Large - 大型

提示：直接回车使用默认值（Small）
```

**Step 4: 收集 platforms（多选 + 动态列表）**
```markdown
**Q4: 目标平台？（可多选）**

从 .spec-first/layer2/*.yaml 读取可选平台：
- h5
- java-backend
- react-native
- python-api

要求：至少选择 1 项
```

**Step 5: 参数确认**
```markdown
请确认以下参数：

- Feature: AUTH
- Mode: Normal
- Size: Small
- Platforms: h5, java-backend
- Bootstrap: 否

是否继续？
- 确认并执行
- 修改参数
- 取消
```

## 调试技巧

1. **在 SKILL.md 中明确说明**：AI 会按照你的描述调用工具
2. **测试时观察 AI 的调用**：检查参数是否正确
3. **处理用户跳过**：提供默认值或降级方案
