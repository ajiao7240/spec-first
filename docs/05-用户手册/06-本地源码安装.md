# 本地源码安装指南

这份指南描述的是“下载源码后，本地打包并安装到目标项目里生成 `/spec:*` 命令”的流程。

## 前置要求

- Node.js >= 20
- npm >= 8
- Claude Code CLI 已安装

## 模式说明

当前仓库已与 `spec-first` CLI 模型对齐：

- 发布入口是 npm 包 `spec-first`
- 本地安装模型是 `npm install -g spec-first` 后执行 `spec-first init --claude`
- `/spec:*` 命令来自项目内 `.claude/commands/spec/`
- `.claude/commands/spec/*.md` 会显式委托给对应 `.claude/skills/spec-*/SKILL.md`
- `spec-first init --claude` 还会把随包发布的 `skills/` 同步到项目内 `.claude/skills/`
- `spec-first init --claude` 还会把随包发布的 `agents/` 同步到项目内 `.claude/agents/`
- 发布包内还包含 `.claude-plugin/plugin.json` 作为统一资产清单
- 不再推荐把当前仓库直接注册成 Claude plugin

## 推荐安装

### 1. 获取源码

```bash
git clone https://github.com/sunrain520/spec-first.git
cd spec-first
```

### 2. 打包源码

```bash
npm pack
```

这一步会生成一个本地 tarball，例如 `spec-first-1.3.10.tgz`。

### 3. 安装本地 tarball

```bash
npm install -g ./spec-first-<version>.tgz
```

这一步就是“全局安装”到系统 PATH。安装成功后，直接运行：

```bash
which spec-first
spec-first --version
```

如果 `which spec-first` 仍然指向旧路径，先执行：

```bash
hash -r
```

再重复上面的检查。

### 4. 诊断宿主环境并初始化 Claude

```bash
spec-first doctor
spec-first init --claude
```

如果你之前装过 pnpm 版本或旧的全局 shim，先在新终端里执行 `hash -r`，或者直接重开一个 shell，再运行上面的命令。这样可以避免命令缓存继续指向旧路径。

### 5. 重启 Claude Code

**重要**：`spec-first init --claude` 写入项目内 `.claude/commands/spec/` 后，必须完全退出并重启 Claude Code 才能稳定识别新命令。

**macOS 退出方式：**
- 使用 `Cmd+Q` 完全退出应用
- 或运行 `pkill -f "claude"` 杀掉进程
- 不是关闭窗口，必须退出应用

然后重新启动：
```bash
claude
```

### 6. 验证安装

```bash
# 检查项目内命令目录
ls .claude/commands/spec

# 检查项目内技能目录
ls .claude/skills

# 检查项目内代理目录
find .claude/agents -type f | head

# 在 Claude Code 中测试
/spec:brainstorm --help
```

如果看到帮助信息，说明安装成功。

> 提示：`npm install -g ./spec-first-<version>.tgz` 里的 `<version>` 应替换成你刚刚 `npm pack` 生成的实际版本号。

如果需要确认全局命令版本，可执行：
```bash
spec-first --version
```

## 源码模式

如果只是想从当前源码仓库确认安装路径，可运行：

```bash
./install-local.sh
```

这个脚本现在只输出与源码打包安装一致的本地验证步骤，不再写入 `~/.claude/plugins`。

## 测试安装

运行 E2E 测试：

```bash
npm test
```

应该看到：
```
=== E2E 测试全部通过 ✓ ===
```

## 开发模式

如果需要修改代码：

### 快速重载（推荐）

```bash
# 修改代码后运行
./dev-reload.sh

# 然后按提示执行 doctor/init
```

此脚本现在会提醒使用与 npm 发布版一致的本地验证流程。

### 手动重载

```bash
# 1. 重新诊断并生成项目命令
spec-first doctor
spec-first init --claude

# 2. 完全退出 Claude Code
pkill -f "claude"

# 3. 重新启动
claude
```

### 测试修改

```bash
# 运行测试
npm test

# 在 Claude Code 中测试
/spec:brainstorm "测试功能"
```

**注意**：Claude Code 不支持热加载，每次修改后必须重启

## 卸载

```bash
# 卸载全局 CLI
npm uninstall -g spec-first

# 删除项目内命令
rm -rf .claude/commands/spec

# 删除源码
cd ..
rm -rf spec-first
```

重启 Claude Code 完成卸载。

## 故障排查

### 问题1：源码安装失败

```bash
# 检查 Node.js 是否安装
node --version

# 如果没有，安装 Node.js
brew install node  # macOS
```

### 问题2：找不到 skills

```bash
# 检查项目命令目录
ls -la .claude/commands/spec

# 检查项目技能目录
ls -la .claude/skills

# 重新生成
spec-first init --claude

# 运行环境诊断
spec-first doctor

# 重新打包并重新安装
npm pack
npm install -g ./spec-first-<version>.tgz
```

### 问题3：修改不生效

```bash
# 重新生成项目命令
spec-first init --claude

# 完全退出并重启 Claude Code
```

### 问题4：`spec-first doctor` 仍然指向旧路径

如果报错路径里出现 `packages/cli/bin/spec-first.js`，说明你的 shell 还缓存着旧的 pnpm shim。处理方式：

```bash
hash -r
which spec-first
spec-first doctor
```

如果还不对，直接关闭当前终端并重新打开，再执行 `which spec-first` 和 `spec-first doctor`。
