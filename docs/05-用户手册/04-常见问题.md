# 常见问题

## 安装问题

### Q: 如何验证安装成功？

A: 在 Claude Code 中输入：
```bash
/spec:brainstorm --help
```
如果显示帮助信息，说明安装成功。

### Q: 安装后找不到 skills？

A: 检查插件目录：
```bash
ls ~/.claude/plugins/
```

## 使用问题

### Q: 如何查看当前任务状态？

A: 查看 task.yaml：
```bash
cat .claude/tasks/<task-id>/task.yaml
```

### Q: Review 不通过怎么办？

A:
1. 查看 04-review.md 中的问题
2. 修复代码
3. 重新运行 /spec:work
4. 再次 /spec:review

### Q: 如何跳过某个阶段？

A: 不建议跳过。如果必须：
```bash
# 手动创建产物文件
touch .claude/tasks/<task-id>/0X-stage.md

# 更新状态
./scripts/task-manager.sh update --task-id <id> --stage <next-stage>
```

### Q: 多个任务如何管理？

A: 每个任务有独立的 task-id：
```bash
ls .claude/tasks/
```

## 高级问题

### Q: 如何自定义 Review 标准？

A: 修改 scripts/review-judge.sh 中的判定规则。

### Q: 如何扩展新的 skill？

A: 参考现有 skills 创建新的 skill.md。

### Q: 如何集成到 CI/CD？

A: 在 CI 中运行测试：
```bash
npm test
```
