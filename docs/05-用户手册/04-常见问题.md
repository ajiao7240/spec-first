# 常见问题

## 安装问题

### Q: 如何验证安装成功？

A: 先在目标项目执行：
```bash
spec-first doctor
spec-first init --claude
```

然后在 Claude Code 中输入：
```bash
/spec:brainstorm --help
```
如果显示帮助信息，说明安装成功。

如果 `doctor` 提示 `.claude/commands/spec` 缺失，先在目标项目执行 `spec-first init --claude`。

### Q: 安装后找不到 `/spec:*` 命令？

A:
1. 检查项目内命令目录：
```bash
ls .claude/commands/spec
```

2. 重新执行项目初始化：
```bash
spec-first init --claude
```

3. 运行环境诊断：
```bash
spec-first doctor
```

4. 完全退出并重启 Claude Code

### Q: 本地安装后 `/spec:*` 不生效？

A:
1. 确认使用的是 npm CLI 模型，而不是旧的 plugin 安装模型

2. 重新执行：
```bash
spec-first doctor
spec-first init --claude
```

3. 完全退出 Claude Code（不是关闭窗口）

4. 重新启动 Claude Code

## 使用问题

### Q: 如何查看生成的命令？

A: 查看项目内的命令目录：
```bash
ls .claude/commands/spec
```

### Q: 如何重新生成命令？

A: 重新运行初始化，命令会默认覆盖现有文件：
```bash
spec-first init --claude
```

### Q: Review 不通过怎么办？

A:
1. 查看 04-review.md 中的问题
2. 修复代码
3. 重新运行 /spec:work
4. 再次 /spec:review

### Q: 多个任务如何管理？

A: 每个任务有独立的 task-id：
```bash
ls .claude/tasks/
```

## 仓库维护问题

以下问题只适用于你正在维护这个仓库源码，而不是普通 npm 用户。

### Q: 如何跳过某个阶段？

A: 不建议跳过。如果你是在仓库源码里调试任务流程，可以手动操作：
```bash
# 手动创建产物文件
touch .claude/tasks/<task-id>/0X-stage.md

# 更新状态
./scripts/task-manager.sh update --task-id <id> --stage <next-stage>
```

### Q: 如何自定义 Review 标准？

A: 如果你在仓库源码里调试本地流程，可以修改 `scripts/review-judge.sh` 中的判定规则；日常 npm 用户通常不需要改这个脚本。

### Q: 如何扩展新的 skill？

A: 如果你在仓库源码里扩展能力，可以参考现有 `skills/` 创建新的 `skill.md`；日常 npm 用户通常只需要使用现有 `/spec:*` 命令。

### Q: 如何集成到 CI/CD？

A: 在 CI 中运行测试：
```bash
npm test
```
