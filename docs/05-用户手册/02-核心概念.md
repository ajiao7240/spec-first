# 核心概念

Spec-First 的当前入口是 `spec-first doctor` 和 `spec-first init --claude`。
`spec-first init --claude` 会同时生成项目级命令文件、项目级 `.claude/skills/`、项目级 `.claude/agents/`；发布包内的 `.claude-plugin/plugin.json` 负责描述这些资产的统一清单。

## 五步闭环

Spec-First 框架基于五步闭环工作流：

```
Brainstorm → Plan → Work → Review → Compound
    ↑                                    ↓
    └────────────── 循环 ─────────────────┘
```

### 1. Brainstorm（需求探索）

**目标**：明确要做什么

**输入**：功能想法、问题描述

**输出**：`docs/brainstorms/YYYY-MM-DD-<topic>-requirements.md`（需求文档）

**关键活动**：
- 探索需求
- 讨论方案
- 确定范围
- 明确验收标准

### 2. Plan（方案规划）

**目标**：明确怎么做

**输入**：需求文档

**输出**：`docs/plans/YYYY-MM-DD-NNN-<type>-<name>-plan.md`（实施计划）

**关键活动**：
- 技术选型
- 任务拆解
- 风险识别
- 时间估算

### 3. Work（执行实施）

**目标**：完成开发

**输入**：实施计划

**输出**：代码、测试与必要的实现记录

**关键活动**：
- 编写代码
- 运行测试
- 记录问题
- 更新文档

### 4. Review（质量评审）

**目标**：确保质量

**输入**：实施产物

**输出**：结构化评审结论与问题列表

**评审维度**：
- Correctness（正确性）
- Completeness（完整性）
- Executability（可执行性）
- Reusability（可复用性）

**判定规则**：
- 关键维度 ≥ 3
- 无阻断问题
- 总分 ≥ 12

### 5. Compound（知识沉淀）

**目标**：提取复用知识

**输入**：评审通过的产物

**输出**：`docs/solutions/<category>/<filename>.md`

**沉淀内容**：
- 提示词模板
- 工作流程
- 领域知识
- 优秀案例
