# 完整示例

## 场景：添加用户登录功能

### 步骤1：Brainstorm

```bash
/spec:brainstorm "添加用户登录功能"
```

**对话过程**：

```
Claude: 让我们探索一下登录功能的需求。请问：
1. 需要支持哪些登录方式？
   A. 用户名密码
   B. 邮箱密码
   C. 第三方登录（Google/GitHub）

用户: A 和 B

Claude: 好的。关于安全性：
1. 是否需要验证码？
2. 是否需要记住登录状态？

用户: 都需要

Claude: 明白了。我会生成需求文档...
```

**生成文档**：`.claude/tasks/2026-03-29-user-login/01-brainstorm.md`

```markdown
---
task_id: 2026-03-29-user-login
stage: brainstorm
---

# 用户登录功能需求

## 核心需求
- 支持用户名/邮箱 + 密码登录
- 登录验证码
- 记住登录状态

## 验收标准
- 用户可以成功登录
- 错误提示清晰
- 登录状态持久化
```
