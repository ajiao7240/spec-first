# 快速开始指南

本指南默认你使用的是 `spec-first` 的 npm CLI 版本。

![Spec-First 总览图](../assets/svg/spec-first-overview.svg)

## 1. 安装 CLI

推荐直接从 npm 安装：

```bash
npm install -g spec-first
```

如果你要验证源码发布物，也可以使用：

```bash
git clone https://github.com/sunrain520/spec-first.git
cd spec-first
npm pack
npm install -g ./spec-first-<version>.tgz
```

如果 shell 之前缓存过旧路径，执行：

```bash
hash -r
```

详细说明见 [本地源码安装指南](./06-本地源码安装.md)。

## 2. 初始化目标项目

在目标项目目录执行：

```bash
spec-first doctor
spec-first init --claude -u <name> --lang <zh|en>
```

完成后，项目内会生成：

- `.claude/commands/spec/*.md`
- `.claude/skills/*`
- `.claude/agents/*`
- `.claude/spec-first/state.json`
- `.claude/spec-first/.developer`

其中 `.claude/spec-first/.developer` 记录本次项目初始化使用的开发者名称、语言偏好、初始化时间和 CLI 版本。

## 3. 启动第一个任务

![Spec-First 五阶段工作流](../assets/svg/spec-first-workflow.svg)

从需求澄清开始：

```bash
/spec:brainstorm "添加用户登录功能"
```

然后按标准链路推进：

```bash
/spec:plan
/spec:work
/spec:review
/spec:compound
```

这条链路会分别产出：

- `docs/brainstorms/*.md`
- `docs/plans/*.md`
- 代码与测试变更
- 结构化评审结论
- `docs/solutions/**/*.md`

## 4. 验证是否可用

在目标项目里再次执行：

```bash
spec-first doctor
```

如果 `.claude/commands/spec`、`.claude/skills` 和 `.claude/agents` 都通过检查，说明项目级工作流已经安装完成。
