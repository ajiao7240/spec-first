# 快速开始指南

本指南默认你使用的是 `spec-first` 的 npm CLI 版本。

![Spec-First 总览图](../assets/svg/spec-first-overview.svg)

## 1. 安装 CLI

推荐直接从 npm 安装：

```bash
npm install -g spec-first
```

如果你要验证源码发布物，也可以使用：

```bash
git clone https://github.com/sunrain520/spec-first.git
cd spec-first
npm pack
npm install -g ./spec-first-<version>.tgz
```

如果 shell 之前缓存过旧路径，执行：

```bash
hash -r
```

详细说明见 [本地源码安装指南](./06-本地源码安装.md)。

## 2. 检查环境并初始化

安装完成后，**第一个命令**应该是 `spec-first doctor`，确认当前环境状态，再执行 `spec-first init`：

```bash
# 安装后第一个命令：检查环境
spec-first doctor
# 或显式指定平台
spec-first doctor --claude
spec-first doctor --codex

# 确认版本
spec-first -v

# 初始化目标项目
spec-first init --claude
# 或
spec-first init --codex
```

完成后，项目内会生成：

- Claude Code：`.claude/commands/spec`、`.claude/skills`、`.claude/spec-first/workflows`、`.claude/agents`
- Codex：`.codex/commands/spec`、`.agents/skills`、`.codex/agents`
- 平台状态：`.claude/spec-first/*` 或 `.codex/spec-first/*`

其中 `.claude/spec-first/.developer` 或 `.codex/spec-first/.developer` 记录本次项目初始化使用的开发者名称、语言偏好、初始化时间和 CLI 版本。

`spec-first -v` 会稳定显示欢迎页和版本信息；`npm install -g spec-first` 的安装期提示不作为对外保证。

当前推荐的事实准备入口是 CRG query-first 路径：

- `spec-graph-bootstrap`：建立 CRG 图索引，并提示后续 workflow 使用 `spec-first crg hook / locate / path / explain / review-context` 按需查询
- `spec-compound`：在完成工作后捕获稳定知识，沉淀到 `docs/solutions/`

旧 Stage-0 文档包、`minimal-context` 和 `injection-index.yaml` 不再是当前入口或 fallback。

如果你想显式指定身份，可以改成：

```bash
spec-first init --claude -u <name> --lang <zh|en>
spec-first init --codex -u <name> --lang <zh|en>
```

如果不传 `-u/--user`，系统会优先回退到全局 `~/.spec-first/.developer`，再回退到 `git config user.name`。

如果 `doctor` 报告 `legacy managed state`，或者 `clean` 提示当前安装属于 legacy，不要先手动删目录，直接重新运行对应平台的 `spec-first init`。当前版本会由 `init` 自动执行 managed hard reset 并重建运行时。

## 3. 启动第一个任务

![Spec-First 五阶段工作流](../assets/svg/spec-first-workflow.svg)

从候选发散开始：

```bash
# Claude Code
/spec:ideate "添加用户登录功能"
/spec:brainstorm "添加用户登录功能"

# Codex
$spec-ideate "添加用户登录功能"
$spec-brainstorm "添加用户登录功能"
```

然后按标准链路推进：

```bash
# Claude Code
/spec:graph-bootstrap  # CRG 图索引与 query-first 决策输入
/spec:ideate
/spec:plan
/spec:work
/spec:code-review
/spec:compound

# Codex
$spec-graph-bootstrap  # CRG 图索引与 query-first 决策输入
$spec-ideate
$spec-plan
$spec-work
$spec-code-review
$spec-compound
```

这条链路会分别产出：

- `docs/brainstorms/*.md`
- `docs/plans/*.md`
- 代码与测试变更
- 结构化评审结论
- `docs/solutions/**/*.md`

## 4. 验证是否可用

在目标项目里再次执行：

```bash
spec-first doctor
# 或显式指定当前平台
spec-first doctor --claude
spec-first doctor --codex
```

如果 Claude Code 的 `commands/spec`、`skills`、`agents` 或 Codex 的 `.codex/commands/spec`、`.agents/skills`、`.codex/agents` 都通过检查，说明项目级工作流已经安装完成。

如需确认当前 CRG workflow 资产已落地，可额外检查：

- Claude Code：`.claude/commands/spec/graph-bootstrap.md`
- Claude Code：`.claude/spec-first/workflows/spec-graph-bootstrap/SKILL.md`
- Codex：`.agents/skills/spec-graph-bootstrap/SKILL.md`

如果宿主内仍看不到 `spec-graph-bootstrap`，说明本地安装需要重新 `init` 并完全重启宿主。
