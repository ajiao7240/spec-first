# 快速开始指南

本指南默认你使用的是 `spec-first` 的 npm CLI 版本。

## 安装

### 方式1：从 npm 安装（推荐）

```bash
npm install -g spec-first
spec-first doctor
spec-first init --claude
```

### 方式2：本地源码安装

```bash
git clone https://github.com/sunrain520/spec-first.git
cd spec-first
npm pack
npm install -g ./spec-first-<version>.tgz
```

本地源码安装的真实流程是先打包成 tarball，再从 tarball 做全局安装。
如果只想查看步骤说明，可以运行 `./install-local.sh`。

如果你的 shell 之前装过 pnpm 版本或旧的全局 shim，先执行：
```bash
hash -r
```
再运行 `spec-first doctor`。

详细说明见 [本地源码安装指南](./06-本地源码安装.md)

## 验证安装

```bash
# 先在目标项目执行
spec-first doctor
spec-first init --claude

# 再在 Claude Code 中验证
/spec:brainstorm --help
```

如果看到帮助信息，说明项目级命令已生成成功。
`spec-first init --claude` 也会同步发布包内的 skills 到 `.claude/skills/`。

## 第一个任务

### 1. 启动 Brainstorm

```bash
/spec:brainstorm "添加用户登录功能"
```

Claude 会基于项目内 `.claude/commands/spec/brainstorm.md` 引导你：
- 探索需求
- 讨论方案
- 确定范围

实际执行时，命令会进一步委托到项目内 `.claude/skills/spec-brainstorm/SKILL.md`，并以这个 skill 的要求决定需求文档路径。

### 2. 制定计划

```bash
/spec:plan
```

生成实施计划到 `docs/plans/YYYY-MM-DD-NNN-<type>-<name>-plan.md`。

### 3. 执行实施

```bash
/spec:work
```

按计划执行开发工作。

### 4. 质量评审

```bash
/spec:review
```

自动评审代码质量。

### 5. 知识沉淀

```bash
/spec:compound
```

提取可复用知识。
