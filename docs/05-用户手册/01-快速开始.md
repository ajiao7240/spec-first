# 快速开始指南

本指南默认你使用的是 `spec-first` 的 npm CLI 版本。

![Spec-First 总览图](../assets/svg/spec-first-overview.svg)

## 1. 安装 CLI

推荐直接从 npm 安装：

```bash
npm install -g spec-first
```

如果你要验证源码发布物，也可以使用：

```bash
git clone https://github.com/sunrain520/spec-first.git
cd spec-first
npm pack
npm install -g ./spec-first-<version>.tgz
```

如果 shell 之前缓存过旧路径，执行：

```bash
hash -r
```

详细说明见 [本地源码安装指南](./06-本地源码安装.md)。

## 2. 初始化目标项目

在目标项目目录执行：

```bash
spec-first doctor
# 或显式指定平台
spec-first doctor --claude
spec-first doctor --codex
spec-first -v
spec-first init --claude
# 或
spec-first init --codex
```

完成后，项目内会生成：

- Claude Code：`.claude/commands/spec`、`.claude/skills`、`.claude/agents`
- Codex：`.codex/commands/spec`、`.agents/skills`、`.codex/agents`
- 平台状态：`.claude/spec-first/*` 或 `.codex/spec-first/*`

其中 `.claude/spec-first/.developer` 或 `.codex/spec-first/.developer` 记录本次项目初始化使用的开发者名称、语言偏好、初始化时间和 CLI 版本。

`spec-first -v` 会稳定显示欢迎页和版本信息；`npm install -g spec-first` 的安装期提示不作为对外保证。

Stage-0 当前有两个入口：

- `spec-bootstrap`：默认稳定入口
- `spec-graph-bootstrap`：阶段 1 并行验证入口

如果你要真正生成项目上下文，优先使用 `bootstrap`；如果你只是在验证新入口是否已完成安装，再使用 `graph-bootstrap`。

如果你想显式指定身份，可以改成：

```bash
spec-first init --claude -u <name> --lang <zh|en>
spec-first init --codex -u <name> --lang <zh|en>
```

如果不传 `-u/--user`，系统会优先回退到全局 `~/.spec-first/.developer`，再回退到 `git config user.name`。

## 3. 启动第一个任务

![Spec-First 五阶段工作流](../assets/svg/spec-first-workflow.svg)

从候选发散开始：

```bash
# Claude Code
/spec:ideate "添加用户登录功能"
/spec:brainstorm "添加用户登录功能"

# Codex
$spec-ideate "添加用户登录功能"
$spec-brainstorm "添加用户登录功能"
```

然后按标准链路推进：

```bash
# Claude Code
/spec:bootstrap
/spec:graph-bootstrap  # 阶段 1 并行验证入口
/spec:ideate
/spec:plan
/spec:work
/spec:review
/spec:compound

# Codex
$spec-bootstrap
$spec-graph-bootstrap  # 阶段 1 并行验证入口
$spec-ideate
$spec-plan
$spec-work
$spec-review
$spec-compound
```

这条链路会分别产出：

- `docs/brainstorms/*.md`
- `docs/plans/*.md`
- 代码与测试变更
- 结构化评审结论
- `docs/solutions/**/*.md`

## 4. 验证是否可用

在目标项目里再次执行：

```bash
spec-first doctor
# 或显式指定当前平台
spec-first doctor --claude
spec-first doctor --codex
```

如果 Claude Code 的 `commands/spec`、`skills`、`agents` 或 Codex 的 `.codex/commands/spec`、`.agents/skills`、`.codex/agents` 都通过检查，说明项目级工作流已经安装完成。

如需确认双入口并行期资产已落地，可额外检查：

- Claude Code：`.claude/commands/spec/bootstrap.md` 与 `.claude/commands/spec/graph-bootstrap.md`
- Claude Code：`.claude/skills/spec-bootstrap/SKILL.md` 与 `.claude/skills/spec-graph-bootstrap/SKILL.md`
- Codex：`.codex/commands/spec/bootstrap.md` 与 `.codex/commands/spec/graph-bootstrap.md`
- Codex：`.agents/skills/spec-bootstrap/SKILL.md` 与 `.agents/skills/spec-graph-bootstrap/SKILL.md`
