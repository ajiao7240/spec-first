### A. 真实代码 bug（P1，最该先修）

#### SRB-001 — spec-plan phantom drift（confirmed, P1 → **RESOLVED**，见 §0.5）
- **契约要求**：§6「source 为真相源，init 收敛重生成，不手改 runtime mirror」，前提是 generator/校验逻辑与 source 一致。
- **实际**：`src/cli/plugin.js:62/86` 的 `HIGH_VALUE_SKILL_ANCHORS / HIGH_VALUE_COMMAND_ANCHORS` 对 spec-plan 仍硬编码 `'Context Orientation Anchor'`，但该锚点已从 source 移除（`skills/spec-plan/SKILL.md` 与 `templates/claude/commands/spec/plan.md` grep 计数=0，被 `skills/spec-plan/SKILL.md:64` 的 `governance-boundaries.md` 引用取代；git commit `8561262b` 移除）。因 init 忠实从 source 重生成，三份 mirror 也都不含该锚点 → `doctor --claude/--codex` **永久报** `missing_command_anchor/missing_anchor: Context Orientation Anchor`，且 `spec-first init` 永远无法清除。对照 spec-work/spec-code-review（source 仍各含 1 个该锚点、未误报），反证 spec-plan 是锚点清单漏更新。
- **证据**：`src/cli/plugin.js:57/62/86/1236/1246`、`skills/spec-plan/SKILL.md:64`、`templates/claude/commands/spec/plan.md`
- **修法**：更新 plugin.js 中 spec-plan 的锚点清单（移除已不存在的字面量或替换为当前真实锚点）；补 contract/unit test 断言锚点清单是 source 真子集。

#### SRB-002 — phantom drift 触发每次 re-init 的 managed hard reset（confirmed, P2 → **RESOLVED**，见 §0.5）
- **实际**：`init.js:836-847` 在已初始化仓库 re-init 时调用 `inspectCurrentRuntimeDrift`，把 SRB-001 的 phantom 锚点 drift 计入 reasons，恒为真 → 每次 `spec-first init` 走 `planHardResetManagedAssets`（`init.js:841/844`）做无谓破坏性重置，且重置后下次 doctor 仍报同一 drift，不收敛。验证员在 /tmp 干净沙箱实测复现（连续 re-init 均输出 `Detected current spec-first runtime drift; performing managed hard reset`）。
- **证据**：`src/cli/commands/init.js:836/841/844/1746`
- **修法**：修复 SRB-001 后自动消除；补"source 未变 + 已初始化 → detected=false"幂等回归测试。
