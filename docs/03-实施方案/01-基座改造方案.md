## 2. 改造策略

### 2.1 核心原则

**复用优先**：
- 保留 Every 的 prompt 正文内容
- 保留 Every 的 agents 实现
- 保留 Every 的核心逻辑

**最小改动**：
- 只改造命名空间（ce: → spec:）
- 只改造目录结构（适配 .claude/tasks/）
- 只增加必需的新机制

**依赖管理**：
- 识别 skills 之间的依赖关系
- 识别 skills 与 agents 的依赖关系
- 保持依赖链完整

### 2.2 改造范围

**必须改造**：
1. Skills 命名（ce-* → spec-*）
2. 产物目录（适配 .claude/tasks/）
3. 增加 task.yaml 管理
4. 增加阶段契约机制
5. 增加 review 双重判定

**保持不变**：
1. Prompt 正文内容
2. Agents 实现
3. 核心工作流逻辑
4. 模板结构

## 4. 实施步骤

### 4.1 第一步：项目初始化

```bash
# 1. 创建项目目录
mkdir -p spec-first
cd spec-first

# 2. 从基座复制核心文件
cp -r /Users/kuang/xiaobu/compound-engineering-plugin/plugins/compound-engineering/.claude-plugin .
cp -r /Users/kuang/xiaobu/compound-engineering-plugin/plugins/compound-engineering/agents .

# 3. 创建目录结构
mkdir -p skills scripts templates resources docs
mkdir -p .claude/tasks .claude/assets/{prompts,workflows,domain,cases}
```

### 4.2 第二步：Skills 改造

```bash
# 复制并重命名 skills
cp -r /path/to/ce-brainstorm skills/spec-brainstorm
cp -r /path/to/ce-plan skills/spec-plan
cp -r /path/to/ce-work skills/spec-work
cp -r /path/to/ce-review skills/spec-review
cp -r /path/to/ce-compound skills/spec-compound

# 批量修改 skill 名称
find skills -name "*.md" -exec sed -i '' 's/ce-brainstorm/spec-brainstorm/g' {} \;
find skills -name "*.md" -exec sed -i '' 's/ce-plan/spec-plan/g' {} \;
# ... 其他替换
```

### 4.3 第三步：增加新机制

**创建 task.yaml 管理脚本**：
```bash
# scripts/task-manager.sh
# 实现 create/read/update task.yaml 功能
```

**创建阶段契约检查脚本**：
```bash
# scripts/stage-gate.sh
# 实现六步契约检查
```

**创建 review 判定脚本**：
```bash
# scripts/review-judge.sh
# 实现双重判定规则
```

### 4.4 第四步：修改 Skills 内部逻辑

在每个 skill 的开头增加：
```markdown
## 阶段契约检查

1. 调用 stage-gate.sh 检查上游产物
2. 调用 task-manager.sh 读取 task.yaml
3. 决策 level（如未指定）
4. 验证输入完整性
```

在每个 skill 的结尾增加：
```markdown
## 状态更新

1. 生成阶段产物到 .claude/tasks/<task-id>/
2. 调用 task-manager.sh 更新 task.yaml
3. 推进 current_stage
```

## 6. 关键改造点

### 6.1 Prompt 内容保留

**原则**：Every 的 prompt 正文完全保留，不重写

**改造位置**：
- 仅在 prompt 前后增加阶段契约检查
- 仅修改输出路径配置
- 仅增加 task.yaml 读写调用

### 6.2 依赖管理

**Skills 依赖 Agents**：
- 确保所有被依赖的 agents 都已复制
- 保持 agents 调用路径正确
- 不修改 agents 内部实现

**Skills 依赖 Skills**：
- 通过文件系统传递（读取上游产物文件）
- 通过 task.yaml 传递状态
- 不使用内存或数据库

### 6.3 最小改动原则

**只改必须改的**：
- 命名空间（ce: → spec:）
- 输出路径（适配 .claude/tasks/）
- 增加契约检查
- 增加状态管理

**不改的内容**：
- Prompt 正文
- Agents 实现
- 核心逻辑
- 工作流程
