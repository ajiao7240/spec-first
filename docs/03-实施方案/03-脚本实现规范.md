## 3. task-manager.sh 历史草案（已退役）

### 3.1 功能定义

**用途**：管理 task.yaml 的创建、读取、更新

**命令接口**：
```bash
# 创建任务
task-manager.sh create --task-id <id> --title <title> --role <role> --level <level>

# 读取任务
task-manager.sh read --task-id <id>

# 更新任务
task-manager.sh update --task-id <id> --stage <stage> --status <status>
```

### 3.2 实现要点

**创建任务**：
```bash
create_task() {
    local task_id=$1
    local title=$2
    local role=${3:-generic}
    local level=${4:-L2}

    # 创建目录
    mkdir -p .claude/tasks/$task_id

    # 生成 task.yaml
    cat > .claude/tasks/$task_id/task.yaml <<EOF
task_id: $task_id
title: $title
role: $role
level: $level
current_stage: brainstorm
status: pending
created_at: $(date -u +"%Y-%m-%dT%H:%M:%S%z")
updated_at: $(date -u +"%Y-%m-%dT%H:%M:%S%z")
EOF
}
```

**读取任务**：
```bash
read_task() {
    local task_id=$1
    local yaml_file=".claude/tasks/$task_id/task.yaml"

    if [ ! -f "$yaml_file" ]; then
        echo "Error: task.yaml not found" >&2
        return 1
    fi

    cat "$yaml_file"
}
```

**更新任务**：
```bash
update_task() {
    local task_id=$1
    local stage=$2
    local status=$3
    local yaml_file=".claude/tasks/$task_id/task.yaml"

    # 更新 current_stage
    if [ -n "$stage" ]; then
        sed -i '' "s/current_stage: .*/current_stage: $stage/" "$yaml_file"
    fi

    # 更新 status
    if [ -n "$status" ]; then
        sed -i '' "s/status: .*/status: $status/" "$yaml_file"
    fi

    # 更新 updated_at
    sed -i '' "s/updated_at: .*/updated_at: $(date -u +"%Y-%m-%dT%H:%M:%S%z")/" "$yaml_file"
}
```

### 3.3 错误处理

- 验证 task_id 格式
- 检查目录是否存在
- 检查 YAML 文件是否存在
- 验证字段值合法性

## 4. stage-gate.sh 历史草案（已退役）

### 4.1 功能定义

**用途**：检查阶段契约，验证上游产物

**命令接口**：
```bash
# 检查上游产物
stage-gate.sh check --task-id <id> --stage <stage>
```

### 4.2 实现要点

**检查逻辑**：
```bash
check_stage_gate() {
    local task_id=$1
    local stage=$2
    local task_dir=".claude/tasks/$task_id"

    case $stage in
        brainstorm)
            # brainstorm 无上游依赖
            return 0
            ;;
        plan)
            check_file_exists "$task_dir/01-brainstorm.md"
            check_stage_completed "$task_id" "brainstorm"
            ;;
        work)
            check_file_exists "$task_dir/02-plan.md"
            check_stage_completed "$task_id" "plan"
            ;;
        review)
            check_file_exists "$task_dir/03-work.md"
            check_stage_completed "$task_id" "work"
            ;;
        compound)
            check_file_exists "$task_dir/04-review.md"
            check_review_passed "$task_dir/04-review.md"
            check_stage_completed "$task_id" "review"
            ;;
    esac
}
```

**辅助函数**：
```bash
check_file_exists() {
    local file=$1
    if [ ! -f "$file" ]; then
        echo "Error: $file not found" >&2
        return 1
    fi
    if [ ! -s "$file" ]; then
        echo "Error: $file is empty" >&2
        return 1
    fi
}

check_stage_completed() {
    local task_id=$1
    local stage=$2
    local yaml_file=".claude/tasks/$task_id/task.yaml"

    # 简化检查：验证 current_stage 已推进
    return 0
}

check_review_passed() {
    local review_file=$1
    local pass=$(grep "^pass:" "$review_file" | cut -d' ' -f2)

    if [ "$pass" != "true" ]; then
        echo "Error: review not passed" >&2
        return 1
    fi
}
```

### 4.3 错误处理

- 文件不存在时返回错误
- 文件为空时返回错误
- review 未通过时返回错误

## 5. review-judge.sh 历史草案（已退役）

### 5.1 功能定义

**用途**：实现 Review 双重判定规则

**命令接口**：
```bash
# 执行判定
review-judge.sh judge --correctness <n> --completeness <n> --executability <n> --reusability <n> --blocking <n>
```

### 5.2 实现要点

**判定逻辑**：
```bash
judge_review() {
    local correctness=$1
    local completeness=$2
    local executability=$3
    local reusability=$4
    local blocking_issues=$5

    # 规则1：关键维度底线
    if [ $correctness -lt 3 ] || [ $completeness -lt 3 ]; then
        echo "pass: false"
        echo "reason: 关键维度低于底线"
        return 1
    fi

    # 规则2：阻断问题优先
    if [ $blocking_issues -gt 0 ]; then
        echo "pass: false"
        echo "reason: 存在阻断问题"
        return 1
    fi

    # 规则3：总分参考
    local total=$((correctness + completeness + executability + reusability))

    if [ $total -lt 12 ]; then
        echo "pass: false"
        echo "reason: 总分过低"
        return 1
    elif [ $total -ge 16 ]; then
        echo "pass: true"
        echo "compound_recommended: true"
        return 0
    else
        echo "pass: true"
        echo "compound_recommended: false"
        echo "reason: 总分在参考区间"
        return 0
    fi
}
```

### 5.3 输出格式

输出 YAML 格式的判定结果，可直接用于 04-review.md 的 frontmatter。

## 6. migrate-assets.sh 实现规范

### 6.1 功能定义

**用途**：将任务产物迁移到资产目录

**命令接口**：
```bash
# 迁移资产
migrate-assets.sh migrate --task-id <id> --type <type> --target <target>
```

### 6.2 实现要点

```bash
migrate_asset() {
    local task_id=$1
    local asset_type=$2  # prompts/workflows/domain/cases
    local source_file=$3
    local target_name=$4

    local target_dir=".claude/assets/$asset_type"
    local target_file="$target_dir/$target_name"

    # 创建目标目录
    mkdir -p "$target_dir"

    # 添加 frontmatter
    cat > "$target_file" <<EOF
---
source_task_id: $task_id
source_role: $(get_task_role "$task_id")
source_stage: compound
created_at: $(date -u +"%Y-%m-%dT%H:%M:%S%z")
tags: []
---

EOF

    # 追加原文件内容
    cat "$source_file" >> "$target_file"
}
```

## 7. init-task.sh 历史草案（已退役）

### 7.1 功能定义

**用途**：初始化任务目录和 task.yaml

**命令接口**：
```bash
init-task.sh --title <title> --role <role> --level <level>
```

### 7.2 实现要点

```bash
init_task() {
    local title=$1
    local role=${2:-generic}
    local level=${3:-L2}

    # 生成 task_id
    local date=$(date +"%Y-%m-%d")
    local slug=$(echo "$title" | tr '[:upper:]' '[:lower:]' | tr ' ' '-')
    local task_id="$date-$slug"

    # 历史草案：调用 legacy task manager
    ./scripts/task-manager.sh create --task-id "$task_id" --title "$title" --role "$role" --level "$level"

    echo "Task initialized: $task_id"
}
```

validate_task() {

## 9. 脚本开发规范

### 9.1 代码风格

- 使用 bash shebang：`#!/bin/bash`
- 使用 `set -e` 启用错误退出
- 使用 `set -u` 检测未定义变量
- 函数命名使用下划线分隔

### 9.2 错误处理

- 所有错误输出到 stderr
- 使用非零退出码表示错误
- 提供清晰的错误信息

### 9.3 参数解析

使用 `getopts` 或手动解析：
```bash
while [[ $# -gt 0 ]]; do
    case $1 in
        --task-id)
            task_id="$2"
            shift 2
            ;;
        --stage)
            stage="$2"
            shift 2
            ;;
        *)
            echo "Unknown option: $1" >&2
            exit 1
            ;;
    esac
done
```
