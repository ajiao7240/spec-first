# 迁移部署指南

## 1. 概述

本文档提供从 Every 基座迁移到 spec-first 的详细步骤和部署指南。

## 2. 迁移前准备

### 2.1 环境要求

- Node.js >= 16
- npm >= 8
- Claude Code CLI 已安装
- Git

### 2.2 基座项目确认

确认 Every 基座位置：
```bash
ls -la /Users/kuang/xiaobu/compound-engineering-plugin/plugins/compound-engineering/
```

### 2.3 目标项目初始化

```bash
# 创建项目目录
mkdir -p spec-first
cd spec-first

# 初始化 git
git init

# 初始化 npm
npm init -y
```

## 3. 文件迁移步骤

### 3.1 第一步：复制核心文件

```bash
# 复制 .claude-plugin 元数据
cp -r /Users/kuang/xiaobu/compound-engineering-plugin/plugins/compound-engineering/.claude-plugin .

# 复制 agents
cp -r /Users/kuang/xiaobu/compound-engineering-plugin/plugins/compound-engineering/agents .
```

### 3.2 第二步：复制并改造 Skills

```bash
# 复制 5 个核心 skills
cp -r /Users/kuang/xiaobu/compound-engineering-plugin/plugins/compound-engineering/skills/ce-brainstorm \
      skills/spec-brainstorm

cp -r /Users/kuang/xiaobu/compound-engineering-plugin/plugins/compound-engineering/skills/ce-plan \
      skills/spec-plan

cp -r /Users/kuang/xiaobu/compound-engineering-plugin/plugins/compound-engineering/skills/ce-work \
      skills/spec-work

cp -r /Users/kuang/xiaobu/compound-engineering-plugin/plugins/compound-engineering/skills/ce-review \
      skills/spec-review

cp -r /Users/kuang/xiaobu/compound-engineering-plugin/plugins/compound-engineering/skills/ce-compound \
      skills/spec-compound
```

### 3.3 第三步：批量重命名

```bash
# 批量替换 skill 名称
find skills -name "*.md" -type f -exec sed -i '' 's/ce-brainstorm/spec-brainstorm/g' {} \;
find skills -name "*.md" -type f -exec sed -i '' 's/ce-plan/spec-plan/g' {} \;
find skills -name "*.md" -type f -exec sed -i '' 's/ce-work/spec-work/g' {} \;
find skills -name "*.md" -type f -exec sed -i '' 's/ce-review/spec-review/g' {} \;
find skills -name "*.md" -type f -exec sed -i '' 's/ce-compound/spec-compound/g' {} \;
```

### 3.4 第四步：创建目录结构

```bash
# 创建运行态目录
mkdir -p .claude/tasks
mkdir -p .claude/assets/{prompts,workflows,domain,cases}
mkdir -p .claude/steering

# 创建项目目录
mkdir -p scripts
mkdir -p templates/{task,review,compound}
mkdir -p resources/schemas
mkdir -p tests/{skills,integration}
mkdir -p docs
```

## 10. 故障排查

### 10.1 常见问题

**问题1：Skills 无法调用**

原因：插件未正确注册

解决：
```bash
# 检查插件目录
ls ~/.claude/plugins/spec-first

# 重新安装
npm uninstall -g spec-first
npm install -g spec-first
```

**问题2：阶段契约检查失败**

原因：上游产物不存在或为空

解决：
```bash
# 检查产物文件
ls -la .claude/tasks/<task-id>/

# 验证文件内容
cat .claude/tasks/<task-id>/01-brainstorm.md
```

**问题3：task.yaml 更新失败**

原因：脚本权限或路径问题

解决：
```bash
# 检查脚本权限
chmod +x scripts/*.sh

# 检查路径
pwd
```

### 10.2 日志调试

在脚本中增加调试输出：
```bash
set -x  # 启用调试模式
```

## 11. 回滚方案

如果迁移失败，可以回滚：

```bash
# 删除安装
npm uninstall -g spec-first

# 清理插件目录
rm -rf ~/.claude/plugins/spec-first

# 恢复使用 Every
# (Every 基座不受影响)
```
