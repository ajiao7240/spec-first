      ↓

      ↓
  生成 design.md（完整设计文档）
      ↓
  writing-plans skill（拆分任务、生成实施计划）
      ↓
  executing-plans / subagent-driven-development
      ↓
  逐个执行任务（task 1 → task 2 → task 3...）

  关键证据

  1. Brainstorming = 项目级 PRD

  从 brainstorming/SKILL.md:

  # Brainstorming Ideas Into Designs

  Help turn ideas into fully formed designs and specs through
  natural collaborative dialogue.

  Checklist:
  1. Explore project context
  2. Ask clarifying questions
  3. Propose 2-3 approaches
  4. Present design (architecture, components, data flow, error handling, testing)
  5. Write design doc → docs/superpowers/specs/YYYY-MM-DD-<topic>-design.md
  6. Spec review loop
  7. User reviews written spec
  8. Transition to implementation → invoke writing-plans skill

  这就是项目级 PRD + 技术方案文档！

  2. Writing-Plans = 任务拆分

  从 writing-plans/SKILL.md:

  # Writing Plans

  Write comprehensive implementation plans assuming the engineer
  has zero context for our codebase.

  ## Bite-Sized Task Granularity
  Each step is one action (2-5 minutes):
  - "Write the failing test" - step
  - "Run it to make sure it fails" - step
  - "Implement the minimal code" - step
  - "Run the tests" - step
  - "Commit" - step

  Save plans to: docs/superpowers/plans/YYYY-MM-DD-<feature-name>.md

  这就是任务列表文档！

  3. Executing-Plans = 逐个执行

  从 executing-plans/SKILL.md:

  # Executing Plans

  Load plan, review critically, execute all tasks, report when complete.

  For each task:
  1. Mark as in_progress
  2. Follow each step exactly
  3. Run verifications
  4. Mark as completed

  这就是任务执行！

  关键区别

  Superpowers 的优势

  1. 自动化流程 - Skills 自动触发，无需手动调用
  2. 质量保证 - 内置 spec review loop 和 plan review loop
  3. TDD 强制 - 每个任务都是 test-first
  4. Subagent 支持 - 可以并行执行多个任务

  Spec-First 当前缺失

  1. ❌ 没有 brainstorming → design.md 流程
  2. ❌ 没有 design.md → plan.md 转换
  3. ❌ 没有 plan.md → task execution 机制
  4. ❌ 只有单任务 PRD，没有项目级设计文档

  ---
  两种系统的本质区别

  Spec-First（当前）

  单任务 → prd.md → 直接开发
  - 适合：小功能、bug 修复
  - 缺失：项目级设计、任务拆分

  Superpowers

  项目需求 → design.md → plan.md → 批量执行任务
  - 适合：大型功能、架构改造
  - 完整：设计评审、任务拆分、质量门禁
