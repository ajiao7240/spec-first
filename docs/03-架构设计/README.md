## 当前建议口径

- `workflow_type` 只负责流程拓扑
- `execution_profile` 只负责任务画像
- `skill selector` 负责前置确认 `selected_skills`
- `runtime hook` 只负责注入，不负责选择
- `phase agent` 只在既定 bundle 内执行
- `decision_hints` 负责结构化约束与 gate 输入
- `evidence` 负责完成证据与追溯

## 使用建议

- 讨论“当前代码到底实现了什么”时，优先引用 [系统架构与数据流](./系统架构与数据流.md)
- 讨论“当前要落地哪一版方案”时，优先引用 [待办技术方案](./待办技术方案.md) 和 [功能保留废弃分析](./功能保留废弃分析.md)
- 需要回看中间推导、争议点和被归档的过渡方案时，再进入 [bak/](./bak/)
