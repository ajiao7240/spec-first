## 概述

本文档列出 spec-first 项目的待办任务，**采用正确的平台无关架构**。

**核心架构**:
```
execution_profile → skill selector → selected_skills (ID列表) → 各平台路径映射 → 注入
```

**核心原则**:
- ✅ task.json 只存储平台无关的数据（skill IDs）
- ✅ 文件路径是平台相关的实现细节
- ✅ 各平台 configurator 负责 skill ID → 文件路径的映射

**总工作量**:

### 任务 #4: skill selector

**目标**: 实现前置 skill bundle 选择逻辑

**工作量**: 3-4 小时

**文件创建**:


**关键修正**: ✅ 返回 skill ID 列表，不是文件路径

**实现步骤**:

1. **实现 selector 核心** (2h)
```python
def select_skills(execution_profile: dict, registry_path: str) -> list[str]:
    """根据 execution_profile 选择 skill bundle
    
    返回: skill ID 列表（不是文件路径）
    
    这是平台无关的纯数据转换逻辑:
    - 输入: execution_profile (dict)
    - 输出: selected_skills (skill ID 列表)
    - 不涉及任何平台特定的路径
    """
    registry = _load_registry(registry_path)
    task_mode = execution_profile.get('task_mode', 'default')
    surface = execution_profile.get('surface')
    
    selected_ids = []
    for skill in registry['skills']:
        if _is_applicable(skill, task_mode, surface):
            selected_ids.append(skill['id'])  # ✅ 返回 ID，不是 path
    
    return selected_ids

def _is_applicable(skill, task_mode, surface):
    applicable = skill.get('applicable_to', {})
    modes = applicable.get('task_modes', [])
    surfaces = applicable.get('surfaces', [])
    
    if modes and task_mode not in modes:
        return False
    if surfaces and surface not in surfaces:
        return False
    
    return True
```

2. **集成到 task_store** (1-2h)
```python
# task_store.py
from .skill_selector import select_skills

def cmd_create(...):
    # ... existing code ...
    selected_skill_ids = select_skills(
        execution_profile=profile,
        registry_path='.spec-first/skills-registry.json'
    )
    task_data['selected_skills'] = selected_skill_ids  # ✅ 存储 ID 列表
```

## 架构设计

### 完整数据流

```
用户创建任务
  ↓
推导 execution_profile (任务 #1)
  {
    "surface": "backend",
    "language": "typescript",
    "task_mode": "tdd"
  }
  ↓
应用 task_mode 映射 (任务 #2)
  decision_hints.implement.mode = "tdd_required"
  ↓
调用 skill selector (任务 #4)
  selected_skills = ["test-driven-development"]  // ✅ skill ID
  ↓
写入 task.json
  {
    execution_profile,
    decision_hints,
    selected_skills  // ✅ ID 列表，不是路径
  }
  ↓
各平台读取 skill IDs 并映射到路径 (任务 #8-19)
  Claude: "test-driven-development" → ".claude/skills/tdd/SKILL.md"
  Codex: "test-driven-development" → ".agents/skills/tdd/SKILL.md"
  iFlow: "test-driven-development" → ".iflow/skills/tdd/SKILL.md"
  ↓
注入到 agent prompt
```

### 平台无关原则

**核心层（平台无关）**:
- ✅ execution_profile 推导
- ✅ task_mode 映射
- ✅ skill selector 选择逻辑
- ✅ selected_skills (skill ID 列表)

**适配层（平台相关）**:
- ✅ skill ID → 文件路径的映射
- ✅ 各平台可以有不同的目录结构
- ✅ 各平台可以有不同的注入机制

**关键区别**:
```
❌ v3 的错误:
  selected_skills = [".spec-first/skills/tdd.md"]  // 路径
  → 假设所有平台使用相同路径

✅ v4 的正确:
  selected_skills = ["test-driven-development"]  // ID
  → 各平台通过映射表转换为实际路径
```
