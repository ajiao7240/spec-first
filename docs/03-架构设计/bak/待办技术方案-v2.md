## 架构修正说明

### 原方案问题

原 v1 方案包含任务 #4-6（skills-registry + skill selector + hook注入），存在严重问题：

1. **单平台假设** - 只考虑 Claude，忽略 iFlow/OpenCode/Codex
2. **技能系统冲突** - 与 Codex `.agents/.codex` 技能层冲突
3. **平台不闭环** - 其他平台无法受益

详见: `待办技术方案-架构问题分析.md`

### v2 修正

**删除任务**:
- ❌ 任务 #4: skills-registry.json（平台相关）
- ❌ 任务 #5: skill selector（平台相关）
- ❌ 任务 #6: Hook 注入 skill bundle（平台相关）

**保留任务**:
- ✅ execution_profile 字段（平台无关数据）
- ✅ task_mode 映射（平台无关编译）
- ✅ workflow templates 重构（平台无关）
- ✅ 代码清理
- ✅ 集成测试

## 架构设计

### 数据流图

```
第一阶段完成后的数据流:

task creation
  ↓
推导 execution_profile (任务 #1)
  ↓
应用 task_mode 映射 (任务 #2)
  ↓
生成 task.json
  {
    workflow_type,
    execution_profile,  ← 新增
    decision_hints      ← 增强
  }
  ↓
各平台自行使用
  - Claude: 可在 hook 中读取 execution_profile
  - Codex: 可在 .agents/.codex 中使用
  - iFlow: 可在 PreToolUse 中使用
  - OpenCode: 可在插件中使用
```

### 平台无关原则

**核心思想**: task.json 是数据，不是实现

**各平台自由度**:
- ✅ 可以读取 execution_profile
- ✅ 可以忽略 execution_profile
- ✅ 可以自行扩展使用方式
- ❌ 不强制统一的技能分发机制
