# 待办技术方案 - 架构问题分析

> **日期**: 2026-03-26
> **问题**: 原方案存在严重的跨平台架构问题

---

## 问题识别

### 问题 #1: 单平台假设

**原方案假设**: 只需修改 Claude 的 inject-subagent-context.py

**实际情况**: 项目是多平台模板系统
- Claude: `.claude/hooks/inject-subagent-context.py`
- iFlow: `.iflow/hooks/inject-subagent-context.py` (PreToolUse hook)
- OpenCode: `.opencode/plugins/inject-subagent-context.js` (插件模型)
- Codex: 完全不同的模型，只在 SessionStart 注入，使用 `.agents/.codex` 技能分发

**影响**:
- ❌ 方案不是"核心架构层"，而是 Claude 局部补丁
- ❌ 其他平台无法受益
- ❌ 维护成本高（需要为每个平台单独实现）

---

### 问题 #2: 绕过现有技能系统

**原方案**: 直接把 skill 文件内容注入 prompt

**问题**:
- ❌ 绕过现有 skill 安装/调用语义
- ❌ 形成第二套技能分发系统
- ❌ 与 Codex 的 `.agents/.codex` 技能层冲突

**现有技能系统**:
- Codex: `.agents/.codex` 技能分发层
- 其他平台: 各自的 skill/plugin 机制

---


## 正确的架构方向

### 方案 A: 平台无关的数据层

**核心思想**: execution_profile 和 decision_hints 是数据，不是实现

**实施**:
1. ✅ execution_profile 字段 - 在 task.json 中（平台无关）
2. ✅ task_mode 映射 - 在 task_store.py 中（平台无关）
3. ❌ 删除 skills-registry.json + skill selector（平台相关）
4. ❌ 删除 hook 注入 skill bundle（平台相关）

**原因**:
- task.json 是所有平台共享的数据格式
- 各平台可以自行决定如何使用 execution_profile
- 不强制统一的技能分发机制

---

### 方案 B: 利用现有 workflow 配置

**发现**: 项目已有 workflow 配置系统

检查现有代码:
```bash
packages/cli/src/configurators/workflow.ts
```

**可能的扩展点**:
- workflow 配置中已有平台无关的抽象
- 可以在这里扩展 execution_profile 支持

## 关键建议

### 1. 立即停止原方案实施

**原因**:
- 任务 #4-6 会创建平台相关的技能系统
- 与现有多平台架构冲突
- 维护成本高，扩展性差

### 2. 聚焦平台无关的数据层

**保留任务**:
- ✅ execution_profile 字段（task.json）
- ✅ task_mode 映射（decision_hints）
- ✅ workflow templates 重构
- ✅ 代码清理

**删除任务**:
- ❌ skills-registry.json
- ❌ skill selector
- ❌ hook 注入 skill bundle

### 3. 技能系统留给各平台

**原则**: 不要在核心层强制统一技能分发机制

**各平台自行决定**:
- Claude: 可以在自己的 hook 中使用 execution_profile
- Codex: 继续使用 `.agents/.codex` 技能层
- iFlow: 使用自己的 PreToolUse hook
- OpenCode: 使用自己的插件系统

---


## 总结

### 核心问题

原 `待办技术方案.md` 的任务 #4-6（skills-registry + skill selector + hook注入）存在严重架构问题：

1. **单平台假设** - 只考虑 Claude，忽略 iFlow/OpenCode/Codex
2. **绕过现有系统** - 创建第二套技能分发，与 Codex 的 `.agents/.codex` 冲突
3. **平台不闭环** - 其他平台无法受益

### 正确方向

**只做平台无关的数据层**:
- execution_profile（task.json 字段）
- task_mode 映射（decision_hints 编译）
- workflow templates 简化

**技能系统留给各平台**:
- 各平台根据 execution_profile 自行决定如何使用
- 不强制统一的技能分发机制

### 修正后工作量

- 原方案: 14-19h（8个任务）
- 修正后: 8-10h（5个任务）
- 减少: 6-9h（删除3个平台相关任务）
