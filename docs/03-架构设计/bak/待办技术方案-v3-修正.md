---

## 问题识别

### v2 的致命错误

**删除了核心能力链**:
```
execution_profile → skill selector → selected_skills
```

**后果**:
- ❌ execution_profile 变成"死数据"
- ❌ task_mode 映射没有实际作用
- ❌ 整个架构失去价值

---

## 正确的架构分层

### 平台无关层（必须保留）

```
execution_profile (数据)
  ↓
skill selector (逻辑)
  ↓
selected_skills (数据)
```

**为什么是平台无关**:
- skill selector 的**选择逻辑**是纯数据转换
- 输入: execution_profile
- 输出: selected_skills (文件路径列表)
- 不涉及任何平台特定的 hook 或注入机制

### 平台相关层（各平台自行实现）

```
各平台注入实现:
  - Claude: inject-subagent-context.py 读取 selected_skills
  - iFlow: inject-subagent-context.py 读取 selected_skills
  - Codex: 在 .agents/.codex 中使用
  - OpenCode: 在插件中使用
```

**为什么是平台相关**:
- 各平台的 hook/plugin 机制不同
- 但都读取同一个 selected_skills 字段

## 关键区分

### ❌ 错误理解

"skill selector 是平台相关的，因为需要注入到 hook"

### ✅ 正确理解

**skill selector 是平台无关的**:
- 只做数据转换: execution_profile → selected_skills
- 输出是文件路径列表
- 不涉及任何 hook 或注入

**注入是平台相关的**:
- Claude: 修改 inject-subagent-context.py
- iFlow: 修改 inject-subagent-context.py
- Codex: 修改 .agents/.codex
- 但这只是"读取 selected_skills 并注入"的简单逻辑

## 平台适配策略

### 共享部分

所有平台共享：
- ✅ execution_profile 推导
- ✅ task_mode 映射
- ✅ skill selector 选择逻辑
- ✅ skills-registry.json

### 各平台单独部分

每个平台只需实现"注入"步骤：
```python
# 所有平台通用代码
selected_skills = task_data.get('selected_skills', [])
skill_content = [read_file(path) for path in selected_skills]
inject_to_prompt(skill_content)
```

**工作量**: 每个平台 30min-1h

## 总结

### 错误原因

我混淆了"核心逻辑"和"平台适配":
- 误以为 skill selector 是平台相关
- 实际上 selector 是纯数据转换，平台无关
- 只有最终的"注入"步骤才是平台相关

### 正确方案

- ✅ 保留 skill selector（核心逻辑）
- ✅ 各平台只做简单注入（读取 selected_skills）
- ✅ 工作量: 15-21h（核心13-17h + 适配2-4h）
