## 关键决策

### ✅ 保留的核心功能

1. **workflow_type** - 拓扑定义（3种）
2. **decision_hints** - 最小 policy contract
3. **execution_profile** - 环境画像（新增，未实现）
4. **Hook 消费 contract** - Runtime enforcement

### ❌ 废弃的功能

1. **Preset 系统** - 被 task_mode 映射替代
2. **复杂 workflow 模板** - 只保留 3 种基础拓扑
3. **TDD Policy 硬编码** - 通过 skill bundle 实现

### 🔄 正确的架构关系

**重要**: 以下是正确的架构层次和职责划分：

1. **task_mode** (execution_profile 的一部分)
   - 表达任务执行模式（tdd/debug/default）
   - 通过简单映射编译为 decision_hints

2. **skill selector** (核心架构层，P1 必须)
   - 前置确认 skill bundle
   - 在 task_mode 之前运行
   - 负责选择合适的 skill 组合

3. **decision_hints** (已实现)
   - 保留硬约束和 gate 输入
   - 由 task_mode 映射生成
   - 被 hook 消费

**错误理解** ❌:
- skill selector 是"可选增强" (P2)
- task_mode 直接映射到 skill bundle

**正确理解** ✅:
- skill selector 是核心架构层 (P1)
- 流程: skill selector → task_mode → decision_hints

### 🔄 转换的功能

1. **--preset with-tdd** → `execution_profile.task_mode = "tdd"`
2. **--preset debug** → `execution_profile.task_mode = "debug"`
3. **Preset policy** → Skill bundle 动态注入
