```python
# skill_selector.py
def select_skills(execution_profile: dict, registry_path: str) -> list[str]:
    """根据 execution_profile 选择 skill bundle"""
    registry = _load_registry(registry_path)
    task_mode = execution_profile.get('task_mode', 'default')
    surface = execution_profile.get('surface')
    
    selected = []
    for skill in registry['skills']:
        if _is_applicable(skill, task_mode, surface):
            selected.append(skill['path'])
    
    return selected

def _is_applicable(skill, task_mode, surface):
    applicable = skill.get('applicable_to', {})
    modes = applicable.get('task_modes', [])
    surfaces = applicable.get('surfaces', [])
    
    if modes and task_mode not in modes:
        return False
    if surfaces and surface not in surfaces:
        return False
    
    return True
```

2. **集成到 task_store** (1-2h)
```python
# task_store.py
from .skill_selector import select_skills

def cmd_create(...):
    # ... existing code ...
    selected_skills = select_skills(
        execution_profile=profile,
        registry_path='.spec-first/skills-registry.json'
    )
```

## 架构设计

### 数据流图

```
第一阶段完成后的数据流:

task creation
  ↓
推导 execution_profile (任务 #1)
  ↓
应用 task_mode 映射 (任务 #2)
  ↓
调用 skill selector (任务 #5)
  ↓
生成 task.json
  {
    workflow_type,
    execution_profile,
    decision_hints,
    selected_skills
  }
  ↓
runtime hook 读取 (任务 #6)
  ↓
注入到 agent prompt
  - decision_hints
  - selected_skills 内容
  ↓
phase agent 执行
```

### 关键设计决策

#### 1. execution_profile 推导策略

**问题**: 如何自动推导 execution_profile？

**方案**:
- 检测项目文件（package.json, pyproject.toml, go.mod）
- 默认 task_mode = "default"
- 允许用户手动覆盖

**优点**:
- 自动化，减少用户输入
- 可扩展，支持更多语言

---


#### 2. skill selector 实现策略

**问题**: 简单映射 vs 复杂规则引擎？

**方案**: 简单映射（任务 #5）
```python
# 基于 task_mode 和 surface 的简单匹配
if task_mode == "tdd" and surface in ["frontend", "backend"]:
    select("test-driven-development")
```

**优点**:
- 实现简单（3-4h）
- 易于调试
- 满足当前需求

**未来扩展**: 如需复杂规则，可在 Phase 2+ 升级

---

#### 3. Hook 注入时机

**问题**: 何时注入 skills？

**方案**: 在 inject-subagent-context.py 中注入

**原因**:
- 已有 hook 基础设施
- 与 decision_hints 注入统一
- 无需修改 agent 代码

## 风险分析

### 技术风险

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| execution_profile 推导不准确 | 中 | 允许用户手动覆盖；提供默认值 |
| skill selector 规则不够灵活 | 低 | 先实现简单版本，后续可扩展 |
| hook 注入性能问题 | 低 | skills 文件较小，影响可忽略 |
| 向后兼容性 | 中 | 保留 decision_hints，新增字段可选 |

### 实施风险

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| 工作量估算偏差 | 中 | 按任务拆分，逐步验证 |
| 依赖关系复杂 | 低 | 明确依赖顺序，按序实施 |
| 测试覆盖不足 | 中 | 任务 #8 强制集成测试 |
