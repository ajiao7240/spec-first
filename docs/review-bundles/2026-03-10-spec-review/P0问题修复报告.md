### F-04: validate format 校验闭环

**问题**：
1. 非法 ID 被 `continue` 跳过而非报错
2. upstream/downstream 未校验
3. 路径占位符（如 `src/**`）未被检测

**修复**：
1. `src/core/validators/format-validator.ts:88-135`
   - 非法 ID 报错而非跳过
   - 添加 upstream/downstream 校验
   - 检测路径占位符（`/` 或 `*`）

```typescript
// 非法 ID 报错
const validation = validateId(id);
if (!validation.valid) {
  errors.push(`非法 ID 格式：${id} - ${validation.error}`);
  continue;
}

// 校验 upstream/downstream
function validateRefList(refList: string, field: string, rowId: string, errors: string[]): void {
  if (!refList || refList.trim() === '') return;

  const refs = refList.split(',').map(r => r.trim()).filter(Boolean);
  for (const ref of refs) {
    // 检测路径占位符
    if (ref.includes('/') || ref.includes('*')) {
      errors.push(`${rowId} ${field} 包含非法引用：${ref}（应为合法 ID）`);
      continue;
    }

    // 校验 ID 格式
    const validation = validateId(ref);
    if (!validation.valid) {
      errors.push(`${rowId} ${field} 包含非法 ID：${ref} - ${validation.error}`);
    }
  }
}
```

**验证**：
```bash
# upstream/downstream 中的非法 ID 将被检测
spec-first validate format FSREQ-20260310-TEST-001
# 预期：检测到 upstream 中的非法 ID
# 预期：检测到 downstream 中的路径占位符

# 预期错误示例：
# TASK-AUTH-001 upstream 包含非法 ID：REQ-PRD-AUTH-001
# TASK-AUTH-001 downstream 包含非法引用：src/core/**/task-auth-001.ts
```

**影响**：
- ✅ 格式校验增强，数据完整性保障提升
- ✅ 向后兼容（只是增加校验，不改变现有逻辑）
- ⚠️ 现有包含非法引用的矩阵将校验失败
