**验证结论**：断链检测功能已完整实现，能够检测 FR 缺少 PRD/DS/TASK/TC 的所有情况。

```typescript
// src/core/trace-engine/matrix.ts:44-85
export function checkMatrix(featureId: string, projectRoot: string): MatrixCheckResult {
  const brokenChains: MatrixCheckResult['brokenChains'] = [];
  for (const fr of frRows) {
    const missing: string[] = [];
    const hasPrd = (fr.upstream ?? []).some(u => u.startsWith('REQ-PRD-'));
    const hasDs = rows.some(r => r.type === 'DS' && r.upstream?.includes(fr.id));
    const hasTask = rows.some((r) => {
      if (r.type !== 'TASK') return false;
      return trace.lineage.hasAnyAncestor(r.id, new Set([fr.id]));
    });
    const hasTc = rows.some(r => r.type === 'TC' && r.upstream?.includes(fr.id));

    if (!hasPrd) missing.push('REQ-PRD-*');
    if (!hasDs) missing.push('DS');
    if (!hasTask) missing.push('TASK');
    if (!hasTc) missing.push('TC');

    if (missing.length > 0) {
      brokenChains.push({ frId: fr.id, missing });
    }
  }
  return { total: rows.length, orphans, brokenChains, vModelPairs, warnings };
}
```

**功能验证**：✅ 功能存在且工作正常

**注意**：功能实现正确，但检测规则使用 `REQ-PRD-*`，与文档禁止 `REQ-PRD` 存在矛盾。这是**规则设计问题**，不是**功能实现问题**。详见 `ID链路问题深度汇总.md` F-01。

```typescript
// src/core/trace-engine/coverage.ts:84-91
function calcImplCoverage(taskRows: MatrixRow[]): number {
  if (taskRows.length === 0) return 1;
  const implemented = taskRows.filter(r =>
    r.status === 'Implemented' || r.status === 'Verified' || r.status === 'Accepted',
  );
  return pct(implemented.length, taskRows.length);
}

// src/core/trace-engine/coverage.ts:95-99
function calcPrCompliance(taskRows: MatrixRow[]): number {
  if (taskRows.length === 0) return 1;
  const linked = taskRows.filter(r => r.upstream && r.upstream.length > 0);
  return pct(linked.length, taskRows.length);
}
```

**功能验证**：✅ C6 使用正确状态枚举，C7 计算 upstream 比例

## 附录：API 签名说明

**当前签名**：
```typescript
export function parseMatrix(featureId: string, projectRoot: string): MatrixRow[]
export function checkMatrix(featureId: string, projectRoot: string): MatrixCheckResult
```

**为什么不改为 `parseMatrix(content: string)`**：
1. featureId/projectRoot 提供必要上下文（相对路径解析）
2. 当前签名已被广泛使用（CLI/Gate/Coverage）
3. 改为 content 参数会丢失上下文，需要调用方自行处理路径

**结论**：保持现有签名。
