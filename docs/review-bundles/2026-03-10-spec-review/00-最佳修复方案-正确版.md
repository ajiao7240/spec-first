# ID 链路最佳修复方案（正确版）

> 生成日期：2026-03-10
> 原则：领域分层、职责单一、符合 Spec-First 渐进式开发目标

---

## 一、核心洞察：完整性 vs 灵活性

### Spec-First 的真实目标

```
核心价值：规范即单一真理源
开发模式：渐进式（01_specify → 02_design → 03_plan → ...）
质量控制：阶段门禁（Gate）
```

### 关键矛盾

**错误理解**：checkMatrix 应该检查"FR 必须有 DS/TASK/TC"

**问题**：
- 01_specify 阶段：只有 FR，没有 DS ❌ 会失败
- 02_design 阶段：有 FR + DS，没有 TASK ❌ 会失败
- 阻塞正常开发流程

**正确理解**：
- checkMatrix：链路合法性（不管阶段）
- Gate：阶段完整性（根据阶段动态要求）

## 四、Infrastructure Layer

### validateChain（双向校验）

```typescript
// src/core/trace-engine/id-validator.ts
export function validateChain(
  id: string,
  upstream: string[],
  downstream: string[]
): string[] {
  const validation = validateId(id);
  if (!validation.valid) return ['Invalid ID format'];

  const rule = ID_RULES[validation.type!];
  const errors: string[] = [];

  // 校验 upstream
  for (const upId of upstream) {
    const upValidation = validateId(upId);
    if (!upValidation.valid) {
      errors.push(`Invalid upstream: ${upId}`);
      continue;
    }
    if (!rule.upstream.includes(upValidation.type!)) {
      errors.push(`${validation.type} cannot have ${upValidation.type} as upstream`);
    }
  }

  // 校验 downstream
  for (const downId of downstream) {
    const downValidation = validateId(downId);
    if (!downValidation.valid) {
      errors.push(`Invalid downstream: ${downId}`);
      continue;
    }
    if (!rule.downstream.includes(downValidation.type!)) {
      errors.push(`${validation.type} cannot have ${downValidation.type} as downstream`);
    }
  }

  return errors;
}
```

## 五、Application Layer

### checkMatrix（只做链路合法性）

```typescript
// src/core/trace-engine/matrix.ts
export function checkMatrix(
  featureId: string,
  projectRoot: string
): MatrixCheckResult {
  const rows = parseMatrix(featureId, projectRoot);
  const errors: string[] = [];

  // 职责：链路合法性
  for (const row of rows) {
    const chainErrors = validateChain(
      row.id,
      row.upstream ?? [],
      row.downstream ?? []
    );
    errors.push(...chainErrors.map(e => `${row.id}: ${e}`));
  }

  // 职责：孤儿检测
  const orphans = rows.filter(r =>
    r.type !== 'Feature' && r.type !== 'REQ' && r.type !== 'RFC' &&
    (!r.upstream || r.upstream.length === 0)
  );

  // 不做：完整性判断（FR 必须有 DS）
  // 理由：这是阶段要求，由 Gate 负责

  return {
    total: rows.length,
    errors,
    orphans,
    valid: errors.length === 0 && orphans.length === 0,
  };
}
```

**关键**：不检查"FR 必须有 DS/TASK/TC"，这是 Gate 的职责。

## 六、Presentation Layer

### evaluateGate（阶段门禁）

```typescript
// src/core/gate-engine/gate-evaluator.ts
export function evaluateGate(
  featureId: string,
  stage: Stage,
  projectRoot: string
): GateResult {
  const rows = parseMatrix(featureId, projectRoot);
  const coverage = getCoverage(featureId, projectRoot);
  const matrixCheck = checkMatrix(featureId, projectRoot);

  const results: GateConditionResult[] = [];

  // 前置：链路合法性
  if (!matrixCheck.valid) {
    results.push({
      id: 'G-MATRIX',
      description: 'Matrix integrity',
      pass: false,
      detail: matrixCheck.errors.join('; '),
    });
  }

  // 阶段门禁（动态要求）
  const rules = GATE_RULES[stage];
  for (const rule of rules.required) {
    const value = coverage[rule.metric];
    results.push({
      id: `G-${stage}-${rule.metric}`,
      description: rule.name,
      pass: value >= rule.threshold,
      detail: `${rule.metric}=${(value * 100).toFixed(1)}%`,
    });
  }

  return {
    pass: results.every(r => r.pass),
    results,
  };
}
```

**关键**：完整性要求在这里，根据阶段动态调整。

## 七、职责对比

| 层次 | 职责 | 不做什么 |
|------|------|---------|
| checkMatrix | 链路合法性 | ❌ 不做完整性判断 |
| getCoverage | 覆盖率计算 | ❌ 不做门禁判断 |
| evaluateGate | 阶段门禁 | ❌ 不做链路校验 |

### 为什么这样划分？

**场景1：01_specify 阶段**
```
矩阵：只有 FR，没有 DS
checkMatrix: ✅ 通过（链路合法）
Gate: ✅ 通过（不要求 C1）
```

**场景2：02_design 阶段**
```
矩阵：有 FR，没有 DS
checkMatrix: ✅ 通过（链路合法）
Gate: ❌ 失败（C1 < 100%）
```

**场景3：05_verify 阶段**
```
矩阵：有 FR + DS，没有 TC
checkMatrix: ✅ 通过（链路合法）
Gate: ❌ 失败（C4 < 80%）
```

**结论**：checkMatrix 不管阶段，Gate 根据阶段动态要求。

## 八、总结

### 核心修正

| 审查意见 | 我的判断 | 理由 |
|---------|---------|------|
| checkMatrix 缺少完整性 | ❌ 错误 | 完整性是 Gate 职责，不是 checkMatrix |
| 覆盖率规则不完整 | ✅ 正确 | 需要 C1-C9 全量定义 |
| ID 规则不覆盖现有体系 | ✅ 正确 | 需要补齐 SYS/ARCH/MOD 等 |
| Gate 需保持质量门禁 | ✅ 正确 | 这是核心职责 |
| validateChain 需双向 | ✅ 正确 | 需要完整约束 |

### 最佳实践

1. ✅ checkMatrix = 链路合法性（不管阶段）
2. ✅ Gate = 阶段完整性（动态要求）
3. ✅ 符合 Spec-First 渐进式开发目标
4. ✅ 职责清晰，边界明确

**这才是真正的最佳实践。**
