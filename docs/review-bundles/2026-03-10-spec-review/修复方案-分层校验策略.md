# 修复方案：分层校验策略（长期治理方案）

> 生成日期：2026-03-10
> 适用场景：生产环境、大型团队协作
> 开发阶段建议：采用强失败策略（见《ID链路问题深度汇总.md》P0修复清单）

---

## 一、问题：当前方案过于严格

### 原方案的风险

```typescript
// 一个非法 ID 导致整个矩阵解析失败
parseMatrix() → validateId 失败 → throw Error → 全盘崩溃
```

### 大项目的实际场景

| 阶段 | 实际情况 | 原方案反应 | 问题 |
|------|---------|-----------|------|
| 需求阶段 | 只有 FR，DS/TASK 未写 | checkMatrix 报断链 | ❌ 阻塞 |
| 设计阶段 | DS 写了一半 | 部分 FR 断链 | ❌ 阻塞 |
| 并行开发 | 团队 A 写 FR，团队 B 写 DS | 时间差导致断链 | ❌ 阻塞 |
| 渐进迭代 | 核心功能先上，非核心标记 Deferred | 仍要求完整链路 | ❌ 阻塞 |

### 核心矛盾

**Fail Fast（数据质量）vs 渐进式开发（流程灵活性）**

⚠️ **开发阶段建议**：优先采用 Fail Fast 策略，本方案作为生产环境长期治理方案

## 二、解决方案：三层校验 + 阶段感知

### 错误分类

| 错误类型 | 示例 | 处理策略 | 理由 |
|---------|------|---------|------|
| **格式错误** | `AC-AUTH-001`（不支持的类型） | ❌ 阻断 | 数据质量问题 |
| **格式错误** | `FR-AUTH-001-EXTRA`（格式不对） | ❌ 阻断 | 数据质量问题 |
| **完整性错误** | FR 缺少 DS | ⚠️ 警告 | 流程阶段问题 |
| **完整性错误** | TASK 缺少 TC | ⚠️ 警告 | 流程阶段问题 |
| **覆盖率不足** | C1 < 80% | 🚦 门禁 | 质量要求 |

### 三层校验架构

```
┌─────────────────────────────────────────────────────────┐
│ L1: 格式校验（Format Validation）                        │
│ - 职责：校验 ID 格式合法性                                │
│ - 策略：强制，阻断式                                      │
│ - 时机：parseMatrix 解析时                               │
│ - 结果：throw Error                                      │
└─────────────────────────────────────────────────────────┘
                         ↓
┌─────────────────────────────────────────────────────────┐
│ L2: 完整性校验（Completeness Validation）                │
│ - 职责：校验链路完整性                                    │
│ - 策略：阶段感知，早期宽松后期严格                         │
│ - 时机：checkMatrix 检查时                               │
│ - 结果：warnings[] 或 errors[]（根据阶段）                │
└─────────────────────────────────────────────────────────┘
                         ↓
┌─────────────────────────────────────────────────────────┐
│ L3: 覆盖率门禁（Coverage Gate）                          │
│ - 职责：评估质量指标                                      │
│ - 策略：阶段配置，分阶段要求                              │
│ - 时机：gate check 评估时                                │
│ - 结果：pass / fail                                      │
└─────────────────────────────────────────────────────────┘
```

```typescript
const ID_PATTERNS: ReadonlyArray<{ type: IdType; regex: RegExp }> = [
  { type: 'Feature', regex: /^FSREQ-\d{8}-[A-Z][A-Z0-9]{1,15}-\d{3}$/ },
  { type: 'FR',      regex: /^FR-[A-Z][A-Z0-9]{1,15}-\d{3}$/ },
  { type: 'DS',      regex: /^DS-[A-Z][A-Z0-9]{1,15}-\d{3}$/ },
  { type: 'TASK',    regex: /^TASK-[A-Z][A-Z0-9]{1,15}-\d{3}$/ },
  { type: 'REQ',     regex: /^REQ-[A-Z][A-Z0-9]{1,15}-\d{3}$/ },  // 统一为 REQ-*
  { type: 'AC',      regex: /^AC-[A-Z][A-Z0-9]{1,15}-\d{3}-\d{2}$/ },  // 新增
  { type: 'TC',      regex: /^TC-(UT|IT|E2E|ST)-[A-Z][A-Z0-9]{1,15}-\d{3}$/ },
  // ... 其他类型
];
```

### 错误提示优化

```typescript
export function validateId(id: string): IdValidationResult {
  if (!id || typeof id !== 'string') {
    return { valid: false, error: 'ID must be a non-empty string' };
  }

  for (const { type, regex } of ID_PATTERNS) {
    if (regex.test(id)) {
      return { valid: true, type };
    }
  }

  // 提供更友好的错误提示
  const suggestions = getSuggestions(id);
  return {
    valid: false,
    error: `Unknown ID format: "${id}"${suggestions ? `\nDid you mean: ${suggestions}` : ''}`
  };
}

function getSuggestions(id: string): string | null {
  if (id.startsWith('AC-') && !id.match(/-\d{2}$/)) {
    return 'AC-<ABBR>-<SEQ>-<NN> (e.g., AC-AUTH-001-01)';
  }
  if (id.startsWith('REQ-PRD-')) {
    return 'Invalid format. Use REQ-<ABBR>-<SEQ> instead';
  }
  return null;
}
```

## 四、L2：完整性校验（阶段感知）

### 原则

**完整性要求 = 流程阶段问题 = 根据阶段决定严重程度**

### 阶段要求矩阵

| 阶段 | REQ | DS | TASK | TC | 处理策略 |
|------|-----|----|----|----|----|
| 00_init | - | - | - | - | 无要求 |
| 01_specify | ⚠️ | ⚠️ | ⚠️ | ⚠️ | 允许全部缺失 |
| 02_design | ⚠️ | ⚠️ | ⚠️ | ⚠️ | 允许缺 TASK/TC |
| 03_plan | ⚠️ | ⚠️ | ⚠️ | ⚠️ | 允许缺 TC |
| 04_implement | ❌ | ❌ | ❌ | ⚠️ | 必须有 REQ/DS/TASK |
| 05_verify | ❌ | ❌ | ❌ | ❌ | 必须完整 |
| 06_wrap_up | ❌ | ❌ | ❌ | ❌ | 必须完整 |

**图例**：
- ⚠️ 警告（warning）：缺失时记录警告，不阻断
- ❌ 错误（error）：缺失时报错，阻断 stage advance

### 实现

```typescript
// src/core/trace-engine/matrix.ts
export function checkMatrix(
  featureId: string,
  projectRoot: string,
  stage?: Stage  // 新增：阶段参数
): MatrixCheckResult {
  const rows = parseMatrix(featureId, projectRoot);
  const errors: string[] = [];
  const warnings: string[] = [];
  const trace = createTraceContext(rows);

  // 孤儿检测（格式正确但无上游）
  const orphans = rows.filter(r =>
    r.type !== 'Feature' && r.type !== 'FR' && r.type !== 'REQ'
    && (!r.upstream || r.upstream.length === 0)
  );
  for (const o of orphans) {
    warnings.push(`Orphan: ${o.id} has no upstream reference`);
  }

  // 断链检测（阶段感知）
  const frRows = trace.frRows;
  const brokenChains: MatrixCheckResult['brokenChains'] = [];

  for (const fr of frRows) {
    const missing: string[] = [];
    const hasPrd = (fr.upstream ?? []).some(u => u.startsWith('REQ-'));  // 统一为 REQ-
    const hasDs = rows.some(r => r.type === 'DS' && r.upstream?.includes(fr.id));
    const hasTask = rows.some(r => r.type === 'TASK' && trace.lineage.hasAnyAncestor(r.id, new Set([fr.id])));
    const hasTc = rows.some(r => r.type === 'TC' && r.upstream?.includes(fr.id));

    if (!hasPrd) missing.push('REQ');
    if (!hasDs) missing.push('DS');
    if (!hasTask) missing.push('TASK');
    if (!hasTc) missing.push('TC');

    if (missing.length > 0) {
      brokenChains.push({ frId: fr.id, missing });

      // 根据阶段决定严重程度
      const severity = getChainSeverity(stage, missing);
      const message = `${severity === 'error' ? 'Error' : 'Warning'}: ${fr.id} missing ${missing.join(', ')} mapping`;

      if (severity === 'error') {
        errors.push(message);
      } else {
        warnings.push(message);
      }
    }
  }

  return {
    total: rows.length,
    orphans,
    brokenChains,
    vModelPairs: checkVModelPairs(rows),
    errors,   // 新增
    warnings
  };
}

// 阶段顺序映射（字符串枚举无法直接比较）
const STAGE_ORDER: Record<Stage, number> = {
  '00_init': 0,
  '01_specify': 1,
  '02_design': 2,
  '03_plan': 3,
  '04_implement': 4,
  '05_verify': 5,
  '06_wrap_up': 6,
  '07_release': 7,
  '08_done': 8,
  '09_cancelled': 9,
};

function getChainSeverity(stage: Stage | undefined, missing: string[]): 'error' | 'warning' {
  if (!stage) return 'warning';
  const order = STAGE_ORDER[stage];

  // 04_implement 及之后：REQ/DS/TASK 必须有
  if (order >= STAGE_ORDER['04_implement']) {
    if (missing.some(m => ['REQ', 'DS', 'TASK'].includes(m))) {
      return 'error';
    }
  }

  // 05_verify 及之后：TC 也必须有
  if (order >= STAGE_ORDER['05_verify']) {
    if (missing.includes('TC')) {
      return 'error';
    }
  }

  return 'warning';
}
```

```typescript
// src/core/gate-engine/gate-check.ts
export function checkGate(
  featureId: string,
  projectRoot: string,
  stage: Stage
): GateCheckResult {
  const coverage = getCoverage(featureId, projectRoot);
  const matrixCheck = checkMatrix(featureId, projectRoot, stage);
  const requirements = STAGE_GATE_REQUIREMENTS[stage];

  const passed: string[] = [];
  const failed: string[] = [];
  const warnings: string[] = [...matrixCheck.warnings];

  // 检查必需指标
  for (const req of requirements.required) {
    const value = coverage[req.metric as keyof CoverageMetrics];
    if (value >= req.threshold) {
      passed.push(`✓ ${req.name}: ${(value * 100).toFixed(1)}% >= ${(req.threshold * 100).toFixed(0)}%`);
    } else {
      failed.push(`✗ ${req.name}: ${(value * 100).toFixed(1)}% < ${(req.threshold * 100).toFixed(0)}%`);
    }
  }

  // 检查可选指标（不阻断）
  for (const req of requirements.optional) {
    const value = coverage[req.metric as keyof CoverageMetrics];
    if (value < req.threshold) {
      warnings.push(`⚠ ${req.name}: ${(value * 100).toFixed(1)}% < ${(req.threshold * 100).toFixed(0)}% (optional)`);
    }
  }

  // 检查完整性错误（阻断）
  if (matrixCheck.errors && matrixCheck.errors.length > 0) {
    failed.push(...matrixCheck.errors);
  }

  return {
    pass: failed.length === 0,
    passed,
    failed,
    warnings,
    coverage
  };
}
```

## 八、迁移指南

### 对现有项目的影响

| 场景 | 影响 | 迁移方案 |
|------|------|---------|
| 矩阵中有非法 ID | ❌ 解析失败 | 修正 ID 格式或添加类型支持 |

### 迁移步骤

1. **Phase 1：格式清理**
   ```bash
   # 检查所有非法 ID
   spec-first validate format <featureId>
   # 修正所有格式错误
   ```

2. **Phase 2：规则统一**
   ```bash
   # 将 REQ-PRD-* 改为 REQ-*
   sed -i 's/REQ-PRD-/REQ-/g' specs/*/traceability-matrix.md
   ```

3. **Phase 3：阶段对齐**
   ```bash
   # 检查当前阶段的完整性要求
   spec-first gate check
   # 根据报错补全缺失链路
   ```

## 九、总结

### 核心改进

1. **格式错误 Fail Fast**：保证数据质量
2. **完整性阶段感知**：允许渐进式开发
3. **门禁分阶段配置**：质量要求逐步提升

| 维度 | 原方案 | 新方案 |

| 大项目友好 | ❌ 阻塞 | ✅ 渐进式 |
