## 1. 阶段状态机

```
                    ┌──────────────────────────────────────────┐
                    │                                          │
                    ▼                                          │
┌─────────┐    ┌──────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐
│ 00_init │───▶│01_specify│───▶│02_design│───▶│03_plan  │───▶│04_impl  │
└─────────┘    └──────────┘    └─────────┘    └─────────┘    └─────────┘
     │              │               │              │              │
     │              │               │              │              │
     ▼              ▼               ▼              ▼              ▼
┌─────────────┐                                           ┌─────────┐
│09_cancelled │◀──────────────────────────────────────────│05_verify│
│  (终态)      │                                           └─────────┘
└─────────────┘                                                │
                                                               ▼
                    ┌─────────┐    ┌─────────┐    ┌─────────┐
                    │06_wrapup│───▶│07_release│───▶│08_done  │
                    └─────────┘    └─────────┘    │ (终态)   │
                                                  └─────────┘

规则:
- 每个阶段只能前进到下一阶段，或取消
- DONE 和 CANCELLED 是终态，不可逆转
- 阶段推进前必须通过 Gate 校验
```

## 3. Skill 三层路由

```
用户输入: "/spec-first:rfc approve RFC-001"
                │
                ▼
┌─────────────────────────────────────────────────────┐
│ Layer 1: Semantic Map                                │
│                                                      │
│ "rfc approve" → { command: "rfc",                    │
│                   argTemplate: "transition {0} approved" }
└──────────────────────────┬──────────────────────────┘
                           │
                           ▼
┌─────────────────────────────────────────────────────┐
│ Layer 2: Runtime Route                               │
│                                                      │
│ RUNTIME_COMMANDS = { "id", "matrix", "stage",       │
│                      "rfc", "defect", "metrics",    │
│                      "gate", "golive", "ai", ... }  │
│                                                      │
│ "rfc" ∈ RUNTIME_COMMANDS → 直接分发 CLI             │
└──────────────────────────┬──────────────────────────┘
                           │
                           ▼
┌─────────────────────────────────────────────────────┐
│ Layer 3: Skill File                                  │
│                                                      │
│ resolveSkillPath("code") →                          │
│   skills/spec-first/07-code/SKILL.md                │
│                                                      │
│ 加载 → 组装 Prompt → 注入 Runtime Notice → 返回     │
└─────────────────────────────────────────────────────┘
```

## 4. Gate 评估流程

```
                 spec-first gate check --feature FSREQ-XXX
                                │
                                ▼
               ┌────────────────────────────────┐
               │ 1. 读取 stage-state.json       │
               │    获取 currentStage           │
               └────────────────┬───────────────┘
                                │
                                ▼
               ┌────────────────────────────────┐
               │ 2. 加载 Gate 条件定义          │
               │    GATE_CONDITIONS[stage]      │
               │    + 项目配置阈值              │
               └────────────────┬───────────────┘
                                │
                                ▼
               ┌────────────────────────────────┐
               │ 3. 构建评估上下文              │
               │    - traceability-matrix.md    │
               │    - RFC 状态                  │
               │    - 覆盖率数据                │
               └────────────────┬───────────────┘
                                │
                                ▼
         ┌──────────────────────┴──────────────────────┐
         │                                             │
         ▼                                             ▼
┌─────────────────┐                         ┌─────────────────┐
│ Layer 1: 代码评估│                         │ Layer 2: 命令评估│
│                 │                         │                 │
│ - 文件存在检查   │                         │ - 执行 shell    │
│ - 覆盖率计算     │                         │ - 解析输出      │
│ - 格式校验       │                         │                 │
└────────┬────────┘                         └────────┬────────┘
         │                                             │
         └──────────────────────┬──────────────────────┘
                                │
                                ▼
               ┌────────────────────────────────┐
               │ 4. 检查豁免 (Exception)        │
               │    - FR 级别匹配               │
               │    - 有效期检查                │
               └────────────────┬───────────────┘
                                │
                                ▼
               ┌────────────────────────────────┐
               │ 5. 聚合结果                     │
               │    PASS | PASS_WITH_WAIVER     │
               │         | FAIL                 │
               └────────────────┬───────────────┘
                                │
                                ▼
               ┌────────────────────────────────┐
               │ 6. 写入 gate-history.jsonl     │
               └────────────────────────────────┘
```

## 5. first Skill 执行流

```
                    spec-first first
                          │
                          ▼
┌──────────────────────────────────────────────────────────┐
│ 主线程                                                    │
│                                                          │
│ 1. 检测已有产物 (.spec-first/runtime/first/index.json)   │
│ 2. 判断执行模式: bootstrap / refresh                     │
│ 3. 调度 Wave 1-4 (并发控制)                              │
│ 4. 收集结果，写入 runtime 真源                           │
│ 5. 渲染 docs/ 人类阅读产物                               │
└──────────────────────────────────────────────────────────┘
                          │
        ┌─────────────────┼─────────────────┐
        │                 │                 │
        ▼                 ▼                 ▼
┌──────────────┐  ┌──────────────┐  ┌──────────────┐
│   Wave 1     │  │   Wave 2     │  │   Wave 3-4   │
│              │  │              │  │              │
│ - 技术栈检测  │  │ - API 分析   │  │ - 领域模型   │
│ - 依赖分析   │  │ - 代码结构   │  │ - 数据库     │
│ - 文档索引   │  │ - 关键流程   │  │ - 规范环境   │
└──────────────┘  └──────────────┘  └──────────────┘
        │                 │                 │
        └─────────────────┼─────────────────┘
                          │
                          ▼
┌──────────────────────────────────────────────────────────┐
│ 输出产物                                                  │
│                                                          │
│ Runtime (真源):           Docs (人类阅读):              │
│ - index.json              - tech-stack.md               │
│ - tech-stack.json         - api-docs.md                 │
│ - api-contracts.json      - codebase-overview.md        │
```
