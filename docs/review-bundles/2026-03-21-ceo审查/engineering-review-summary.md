## NOT in Scope

以下工作在本次 v2.0 重构范围之外，明确延后：

| 工作项 | 延后原因 | 潜在时机 |
|--------|---------|---------|
| **可视化前端** | 当前定位是 CLI 工具，前端会增加复杂度 | v3.0 或独立项目 |
| **多用户协作** | 需要后端、权限、实时同步，是另一个产品 | 不在 Spec-First 范围内 |
| **插件生态市场** | 先做好内置能力，生态是加分项 | v2.1+ |
| **遥测系统** | 需要隐私和合规判断，沟通成本高 | v2.1+ |
| **性能极致优化** | 当前不是第一矛盾，容易偏离主线 | v2.1+（根据真实瓶颈） |
| **国际化 (i18n)** | 当前用户主要是中文团队 | v2.1+（如有国际用户需求） |
| **Windows 原生支持** | 当前测试集中在 macOS/Linux | v2.1+（如有需求） |
| **IDE 插件集成** | 需要大量平台适配工作 | v3.0 或独立项目 |

## Failure Modes

### Medium Risk (v2.0 实现时需注意)
1. **动态能力加载失败**

   - **Error handling**: 需要统一 SpecFirstError 处理

2. **Level 检测错误**
   - **Failure mode**: Level 推断错误（例如有 gate.yaml 但用户想要 Level 1）
   - **Test coverage**: 计划在 Phase 2-3 中编写
   - **Error handling**: 需要提供 --level 参数覆盖
   - **User experience**: 应警告歧义并提供手动指定方式

3. **Migration 失败**
   - **Failure mode**: v1.x 项目结构损坏或不兼容
   - **Test coverage**: 计划在 Phase 5 中编写
   - **Error handling**: 需要备份和回滚机制
   - **User experience**: 应拒绝迁移并指向手动修复指南
