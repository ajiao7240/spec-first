## 一、项目概述

### 1.1 GSD-2 (gsd-pi)

| 属性 | 值 |
|------|-----|
| **定位** | 独立编码代理应用 (Standalone CLI Agent) |
| **技术栈** | TypeScript + Pi SDK (原生代理框架) |
| **运行方式** | 独立 CLI 二进制 (`gsd` 命令) |
| **架构** | Monorepo (5 个 packages) |
| **核心代码量** | ~14,000+ 行 TypeScript |
| **依赖** | Pi SDK, Playwright, Sharp, @clack/prompts |

**核心理念**: 从 Prompt 框架进化为真正的代理应用，直接控制会话、上下文和执行流程。

### 1.2 Get-Shit-Done (get-shit-done-cc)

| 属性 | 值 |
|------|-----|
| **定位** | Meta-Prompting / Context Engineering 框架 |
| **技术栈** | Node.js + CommonJS 工具链 |
| **运行方式** | 注入 Claude Code / OpenCode / Gemini / Codex 的命令系统 |
| **架构** | Commands + Agents + Hooks 组合 |
| **核心代码量** | ~5,400+ 行 CommonJS |
| **依赖** | esbuild, c8 (无外部 AI 依赖) |

**核心理念**: 通过精心设计的 Prompt、Agent 和工作流注入，让宿主 AI 更可靠地执行任务。

## 三、功能对比

### 3.1 核心工作流对比

| 功能维度 | GSD-2 | Get-Shit-Done |
|----------|-------|---------------|
| **项目初始化** | `/gsd` 交互式讨论 | `/gsd:new-project` 4步流程 |
| **阶段讨论** | 内置 discuss 流程 | `/gsd:discuss-phase` |
| **规划阶段** | 自动 research + plan | `/gsd:plan-phase` + Agent |
| **执行阶段** | `/gsd auto` 自动循环 | `/gsd:execute-phase` |
| **验证工作** | 内置 must-haves 校验 | `/gsd:verify-work` |
| **里程碑完成** | 自动推进 | `/gsd:complete-milestone` |

### 3.2 相同功能 (功能重叠)

#### 3.2.1 工作流阶段

两者都实现了相同的 **4 阶段开发闭环**：

```
Research → Plan → Execute → Verify
    ↑                              │
    └──────────────────────────────┘
```

| 阶段 | GSD-2 实现 | GSD-v1 实现 |
|------|-----------|-------------|
| **Research** | `research` 单元 + 自动调研 | `/gsd:research-phase` + 4 个并行 Agent |
| **Plan** | `plan_slice` 单元 + XML 任务 | `/gsd:plan-phase` + gsd-planner Agent |
| **Execute** | `execute_task` 单元 + 原子提交 | `/gsd:execute-phase` + gsd-executor Agent |
| **Verify** | must-haves + 自动校验 | `/gsd:verify-work` + gsd-verifier Agent |

#### 3.2.2 状态管理

两者都使用 **磁盘文件作为状态源**：

| 状态文件 | GSD-2 | Get-Shit-Done |
|----------|-------|---------------|
| 项目描述 | `PROJECT.md` | `PROJECT.md` |
| 决策记录 | `DECISIONS.md` | `STATE.md` (含决策) |
| 路线图 | `{milestone}-ROADMAP.md` | `ROADMAP.md` |
| 阶段计划 | `S{NN}-PLAN.md` | `{phase}-{N}-PLAN.md` |
| 任务摘要 | `T{NN}-SUMMARY.md` | `{phase}-{N}-SUMMARY.md` |
| 配置 | `~/.gsd/preferences.md` | `.planning/config.json` |

#### 3.2.3 Git 策略

两者都支持 **原子提交 + 分支管理**：

| Git 功能 | GSD-2 | Get-Shit-Done |
|----------|-------|---------------|
| 分支策略 | `branch-per-slice` + squash merge | 可选 `phase` / `milestone` 分支 |
| 提交粒度 | 每任务一提交 | 每任务一提交 |
| 提交格式 | `feat(S01/T03): description` | `feat(08-02): description` |

#### 3.2.4 上下文工程

两者都解决 **上下文窗口污染问题**：

| 技术 | GSD-2 | Get-Shit-Done |
|------|-------|---------------|
| 上下文隔离 | 每单元新 Session | Subagent 隔离 |
| 上下文预加载 | dispatch 时内联相关文件 | `<files_to_read>` 块 |
| 状态快照 | `STATE.md` 派生 | `STATE.md` 维护 |

### 3.3 不同功能 (差异点)

#### 3.3.1 GSD-2 独有功能

| 功能 | 描述 | 技术实现 |
|------|------|----------|
| **独立运行** | 不依赖 Claude Code，作为独立 CLI 运行 | Pi SDK Runtime |

#### 3.3.2 Get-Shit-Done 独有功能

1. 更多原生工具集成 (数据库、云服务)
2. 更强的多模型编排能力
3. 团队协作功能增强
4. 企业级特性 (审计、权限)

**Get-Shit-Done 可能的发展**:
1. 更多运行时支持 (更多 AI CLI)
2. 更智能的 Agent 调度
3. 社区 Skill 生态
4. 与 GSD-2 功能融合
