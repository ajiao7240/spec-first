## 方案 1：viewer 接入实时 Gate 评估

确认方案：`修正后方案`

### 问题

- 当前 `/api/feature/:id/gate-status` 仍读取 `gate-history.jsonl` 最新快照
- viewer 看到的是“上次结果”，不是“当前真实结果”

### 采用方案

后端同时返回：

- `current`：实时 evaluator 结果
- `stages`：历史摘要结果

### 实施方式

- 在 viewer 服务端直接复用 Gate evaluator
- 请求 `/gate-status` 时实时计算当前阶段 Gate
- 实时查看场景使用 `persist: false`
- 保留历史聚合逻辑，用于阶段流转时间线和历史回看

### 预期收益

- viewer 与 CLI 使用同一 Gate 真源
- 避免“viewer 显示可推进，但 CLI 已阻断”或反向偏差
- 不丢历史轨迹能力

## 方案 2：删除 viewer 内部重复阶段 target 真源

确认方案：`C`

### 问题

- viewer 后端仍维护本地 `getStageTargets()`
- 这会形成第二套阶段阈值解释器

### 采用方案

viewer 后端直接复用统一真源 `getStageMetricTargets()`，并通过现有 `/api/feature/:id/metrics` 返回 `metricTargets`

### 实施方式

- 删除 `server.js` 中本地 `getStageTargets()`
- 在 viewer 后端直接复用统一真源 `getStageMetricTargets()`
- 保持沿用现有 `/api/feature/:id/metrics` 接口返回 `metricTargets`
- 前端继续消费当前接口，不再新增额外 JSON 出口

### 预期收益

- 真正消除 viewer 内部重复解释器
- target 变更时无需再同步修改 viewer
- 避免新增重复接口层，保持当前数据面简洁

## 方案 3：viewer 阶段表改为真实生效语义

确认方案：`C`

### 问题

- 当前阶段表仍直接读取 `mergedRules.gateConditions`
- 展示的是“配置清单”，不是“当前真实生效结果”

### 采用方案

后端直接返回结构化展示模型，前端只负责渲染，不负责解释

### 返回结构建议

```json
{
  "effectiveGates": [],
  "warnings": [],
  "deliverables": [],
  "statusSummary": {
    "status": "PASS|FAIL|PASS_WITH_WAIVER",
    "blockingCount": 0,
    "warningCount": 0
  }
}
```

### 实施方式

- `/gate-status` 或配套接口直接返回：
  - `effectiveGates`
  - `warnings`
  - `deliverables`
  - `statusSummary`
- 前端去掉直接读取 `mergedRules.gateConditions`
- 界面拆为三块：
  - `Effective Gates`
  - `Warnings`
  - `Deliverables`

### 预期收益

- viewer 不再展示与 evaluator 脱节的原始 Gate 表
- warning 与 blocking 的语义边界清晰
- 前端不再承担规则解释职责

## 方案 4：`metrics report` 输出优化

确认方案：`B`

### 问题

- 当前 `metrics report` 仍偏向静态总览
- 尚未拆分为方案要求的两层视图：
  - `Coverage Snapshot`
  - `Current Stage Targets`

### 采用方案

在现有 `report` 命令中直接分两段输出

### 实施方式

输出顺序调整为：

1. 健康分与说明
2. `Coverage Snapshot`
   - 展示核心 5 指标原始值
3. `Current Stage Targets`
   - 仅展示当前阶段纳入判定的指标
   - 显示 target / current / status
4. `Bottlenecks`

### 规则约束

- 当前阶段未纳入判定的指标不标失败
- 健康分继续保留，但明确“不等于当前阶段 Gate”

### 预期收益

- metrics 输出与方案文档语义一致
- 用户能区分“原始快照”和“当前阶段治理判断”
- 降低 `report` 与 Gate 认知冲突
