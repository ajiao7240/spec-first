## 🎯 核心发现

### 发现 1: warning 语义缺失是关键问题

**全链路方案的重要洞察**:
```
当前系统只有 PASS/WAIVER/FAIL
没有"失败但不阻断"的语义
```

**影响**:
- C-PRD/C10/C11 降级为"警告"
- 但系统无法表达"警告"
- CLI 仍会显示 [FAIL]

**解决方案**:
- 需要增加 WARNING 状态
- 修改 GateResult 类型
- 更新 CLI 输出逻辑

### 发现 2: 指标消费层需要同步调整

**全链路方案的重要洞察**:
```
即使删掉 Gate 条件
health-score/bottleneck/metrics 仍会强调这些问题
```

**影响**:
- 删除 C1/C2/C5/C7 后
- 健康度评分可能仍然计算这些指标
- 瓶颈分析可能仍然报告这些问题

**解决方案**:
- 同步更新 health-score.ts
- 同步更新 bottleneck.ts
- 同步更新 metrics-engine

### 发现 3: 配置化能力未充分利用

**全链路方案的重要洞察**:
```
Layer 0/1/2/3 规则合并存在
但没有真正变成可切换的治理 profile
```

**影响**:
- 当前只能"全局调整"
- 无法"按项目/团队定制"

**解决方案**:
- 定义 default/strict/minimal 三种 profile
- 通过配置文件切换
- 保持代码不变
