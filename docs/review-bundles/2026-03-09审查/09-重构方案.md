# Skill: spec

定义需求规格，通过 Phase 0（PRD）+ Step 0-8 流程生成 FR 功能需求与验收标准（AC）。

## 核心流程

### Phase 0: PRD 生成
详见 [references/phase0-prd-workflow.md](references/phase0-prd-workflow.md)

**快速概览**:
- 0.0: Feature 快速初始化（防止信息丢失）
- 0.1-0.2: 任务锚定 + 质量扫描 + 上下文收集
- 0.3-0.4: PRD 生成 + 自检
- 0.5-0.6: PRD 补全对话 + 用户确认

**门禁**: C-PRD >= 85%

### Step 0-7: FR/AC 定义
详见 [references/steps-fr-ac-workflow.md](references/steps-fr-ac-workflow.md)

**快速概览**:
- Step 0: 任务存在性检查
- Step 1: 复杂度判定（详见 [complexity-matrix.md](references/complexity-matrix.md)）
- Step 2-5: 提问门禁 + 调研 + 发散 + 收敛
- Step 6-7: ADR 决策 + 最终确认 + Gate Check

## 复杂度自适应跳过规则

| 复杂度 | 执行路径 |
|--------|---------|
| Trivial | Phase 0 + Step 0-1 + Step 7 |
| Simple | Phase 0 + Step 0-2 + Step 5 + Step 7 |
| Moderate | Phase 0 + Step 0-5 + Step 7 |
| Complex | Phase 0 + Step 0-7 全量 |

详见 [references/complexity-matrix.md](references/complexity-matrix.md)

## 关键约束

### 字面即精神原则
不得自行变通流程。详见 [references/anti-rationalization-guards.md](references/anti-rationalization-guards.md)

### 模板驱动约束
- 必须写：业务目标、边界、验收标准、NFR、风险与约束
- 禁止写：模块实现细节、类/函数级算法、具体库选型

### CLI 硬约束
- ID 类型：使用 FR/DS/TASK/TC/REQ
- Matrix 状态：使用 Planned/InProgress/Completed/Verified/Blocked
- 确认策略：matrix update 必须添加 --yes

详见 [references/cli-commands-reference.md](references/cli-commands-reference.md)

## 质量门禁

- Phase 0.4: PRD 自检（C-PRD >= 85%）
- Step 7: Gate Check（G-SPEC-00 + 格式校验）

详见 [references/quality-gates.md](references/quality-gates.md)
