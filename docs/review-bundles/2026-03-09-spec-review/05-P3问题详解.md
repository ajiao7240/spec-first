# P3 文档完善详解

## P3-1: ABBR 生成规则不够健壮

### 问题描述

**当前规则**:
```markdown
- 从用户描述中提取 2-4 个关键词
- 取首字母大写组合（如 "用户认证" → AUTH）
```

**问题**: 中文关键词如何提取首字母？

### 解决方案

```markdown
**ABBR 生成规则**:
1. 提取 2-4 个关键词（中文或英文）
2. 中文关键词 → 翻译为英文 → 取首字母大写
   - 示例: "用户认证" → "User Authentication" → AUTH
3. 英文关键词 → 直接取首字母大写
   - 示例: "Homepage Redesign" → HOMEPAGE
4. 无法提取或翻译失败 → 使用 FEAT 作为默认值
5. ABBR 长度限制: 2-8 个字符
```

## P3-2: 质量评分公式过于简单

### 问题描述

**当前公式**:
```
初始评分 = (已明确项 / 必需项总数) × 100%
必需项: 业务目标(25%) + 功能边界(25%) + 约束条件(25%) + 成功标准(25%)
```

### 解决方案

**引入加权评分**:
```markdown
**质量评分规则**:
```
初始评分 = 业务目标(35%) + 功能边界(30%) + 成功标准(25%) + 约束条件(10%)
```

**理由**:
- 业务目标最关键（35%）：决定做什么
- 功能边界次之（30%）：决定范围
- 成功标准重要（25%）：决定验收
- 约束条件辅助（10%）：决定限制
```

## P3-4: 背景输入契约不清晰

### 问题描述

**当前内容**:
```markdown
- 背景质量字段与枚举遵循 `../shared/background-quality-contract.md`
- 当背景不足时进入 `degraded` 降级模式
```

**问题**: 没有说明如何判断"背景不足"

### 解决方案

**内联关键契约内容**:
```markdown
## 背景输入契约

**背景质量判定**:
- `complete`: PRD 存在 + constitution 存在 + 相关文件已扫描
- `partial`: PRD 存在但 constitution 缺失，或相关文件未扫描
- `degraded`: PRD 不存在或质量评分 < 40%

**降级模式行为**:
- `degraded` 时，Phase 0.5 最多 3 轮 2 问（而非 5 轮 3 问）
- `degraded` 时，Step 5 强制执行（即使 Trivial 复杂度）
- `degraded` 时，在 findings.md 顶部标记 `[DEGRADED_MODE]`
```
