## P2-1: Description 缺少清晰触发器

### 问题描述

**当前 description**:
```yaml
description: "定位 Feature 并校验阶段为需求规格（01_specify）"
```

**skill-creator 原则**:
> "Include all 'when to use' information in description - Not in the body."

## P2-2: 门禁设计过度（三道门禁）

### 问题描述

**位置**: 第 287-299 行

**当前设计**:
```markdown
**三道门禁**:
1. 可推导性检查 - 检查上下文是否已包含答案
2. 影响性过滤 - 仅保留会改变 FR/AC/NFR 的问题
3. 优先级排序 - 按风险排序
```

**问题**:
- "可推导性检查"需要 AI 理解代码语义，容易误判
- AI 难以准确判断"是否可从代码/文档推导"

### 解决方案

**简化为两道门禁**:

```markdown
**两道门禁**:

**门禁 1: 影响性过滤**
- 仅保留"会改变 FR/AC/NFR 结果"的问题
- 不影响输出的问题 → 使用合理默认值，记录到 findings.md 的 [ASSUMED] 清单

**门禁 2: 优先级排序**
- 按风险排序：范围 > 安全 > 合规 > UX > 技术偏好
- 每轮最多 3 个问题
- 提供 2-3 个可执行候选答案（A/B/C）
```

**假设记录机制**:
```markdown
## findings.md 假设清单

### [ASSUMED] 清单
- [ASSUMED][UX] 默认支持 Chrome/Firefox/Safari 最新版本
- [ASSUMED][TECH] 默认使用 REST API 而非 GraphQL
- [ASSUMED][PERF] 默认响应时间 < 2s 可接受
```

## P2-3: 状态管理字段冗余

### 问题描述

**当前状态头**:
```yaml
current_step: "Step 3"
completed_steps: ["Phase 0.0", ..., "Step 2"]
next_step: "Step 4"
```

**冗余分析**:
- `next_step` 可从 `current_step` 推导
- `completed_steps` 可从 `current_step` 推导（假设顺序执行）

### 解决方案

**简化状态头**:
```yaml
---
current_step: "Step 3"
skipped_steps: ["Step 4", "Step 5"]
complexity: "Moderate"
scenario: "iteration"
last_updated: "2026-03-05T10:00:00Z"
quality_score: 92
---
```

## P2-4: 一问一答规则与效率矛盾

### 问题描述

**当前规则**: 每次只问 1 个问题

**问题**: 独立问题（如"支持哪些浏览器？"和"最大文件大小？"）串行提问效率低

### 解决方案

**智能批量提问**:
```markdown
**提问规则**:
1. **独立问题可批量**: 如果问题之间无依赖关系，可一次提问 2-3 个
2. **依赖问题必须串行**: 如果问题 B 的答案依赖问题 A，必须先问 A
3. **每轮最多 3 问**: 无论独立或依赖，每轮最多 3 个问题

**示例**:
- ✅ 批量: "支持哪些浏览器？" + "最大文件大小？" （独立）
- ❌ 批量: "是否需要登录？" + "登录后显示哪些字段？" （依赖）
```

## P2-5: 跳过规则记录格式不统一

### 问题描述

**当前格式**:
```markdown
Step 4: SKIPPED — 无技术选型需求
Step 5: SKIPPED — Trivial 路径无需发散扫描
Step 7: SKIPPED — 无需 ADR
```

### 解决方案

**统一格式**:
```markdown
## Step X: [步骤名称]
SKIPPED — [复杂度档位] [具体理由]

**示例**:
## Step 4: Research-first Mode
SKIPPED — Simple 复杂度，无技术选型需求

## Step 5: Expansion Sweep
SKIPPED — Trivial 复杂度，无需发散扫描

## Step 7: ADR-lite
SKIPPED — Moderate 复杂度，无多方案权衡
```
