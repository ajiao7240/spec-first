## 四、对上一版报告的关键修正

- “`required_mcps` 在 AI 调用完成后才校验”表述不准确。
  - 现状：校验发生在 `runPostWriteGuards()`，确实偏晚，但属于“执行后守卫阶段”，不是文档中表述的“输出已写入后才发现”。
- “Gate 与 Skill 阈值存在硬编码分裂”曾成立。
  - 现状：`G-IMPL-01` / `G-VERIFY-01` 已统一迁移到配置真理源，文档与代码不再各自写死数值。
- “batch-executor 无集成测试”不准确。
  - 现状：已有 `tests/unit/batch-executor/*.test.ts`，但覆盖的是占位执行器，不是“真实 Agent 执行链路”。
- “07_release 完全无门禁”不准确。
  - 现状：`condition-registry.ts` 与 `golive.ts` 都要求 `release-note.md` 和 `smoke-test-report.md`，问题在于阶段推进仍会自动跳过。
- “某些问题定位到不存在文件/错误文件”已修正。
  - 正则风险不仅在 `slop-checker.ts`，也在 `completion-detector.ts`
  - 扩展命令注入文件实际存在：`src/core/tool-integration/ai-runtime-hook.ts`

## 五、建议优先级

1. 先修复两处 shell 注入：`manifest-engine.ts`、`ai-runtime-hook.ts`
2. 对齐 Gate 与 Skill 规范：至少修正 `G-IMPL-01` / `G-VERIFY-01`
3. 保持 `07_release` 当前“正式发布阶段 + 自动收口实现 + release 证据要求”的单一语义持续一致
4. 为 `saveTodoState()` 引入文件锁
5. 为 batch executor 明确为实验特性，或接入真实执行器

## 六、结论

这份 bundle 的主结论应从“64 项全面结论”收敛为“少量已证实的高风险问题 + 一批需要继续验证的改进项”。当前仓库确实存在关键安全与规范一致性问题，但上一版报告混入了统计失真、文件定位错误和部分过度推断，已在本版修正。
