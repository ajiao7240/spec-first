# 测试审查（校正版）

## 核验结果

- `pnpm test -- --run`：本地通过
- `pnpm typecheck`：本地通过
- 测试资产规模：
  - `tests/` 下 160 个测试文件
  - 约 1435 个 `it()/test()` 声明

说明：这里采用仓库可复核的口径，不沿用上一版“1464 tests”的静态结论。

## 已验证问题

### TEST-001 [High] 缺少 CI 自动门禁

现状：仓库中不存在 `.github/workflows/`

影响：测试和类型检查无法在 PR/主干自动兜底。

### TEST-002 [Medium] 覆盖率阈值偏低

位置：`vitest.config.ts`

当前阈值：

```ts
lines: 75
functions: 75
branches: 65
statements: 75
```

尤其 `branches: 65` 偏保守。

### TEST-003 [Medium] batch executor 测试只覆盖占位实现

证据：

- `tests/unit/batch-executor/integration.test.ts`
- `tests/unit/batch-executor/concurrent-executor.test.ts`
- `src/core/batch-executor/concurrent-executor.ts`

现状：测试确实存在，但执行器内部仍是 `setTimeout(100)` 假执行，因此测试并未覆盖真实 Agent 集成风险。

## 已验证亮点

- `auto-loop` 有较完整的单测与场景测试
- `watchdog` 有 unit + e2e 场景
- release 证据也有专门测试：`tests/unit/release-evidence-governance.test.ts`

## 修正项

- “完全缺少 batch-executor 集成测试”不成立
- “07_release 阶段没有任何测试”不成立，至少 release 证据治理已有测试

## 建议

1. 先加 CI，再调高覆盖率门槛
2. 为 batch executor 引入可注入执行器，再补真实链路测试
