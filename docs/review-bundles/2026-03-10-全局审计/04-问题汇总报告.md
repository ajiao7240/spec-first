## 问题清单

### 🔴 P0-1: 单字符缩写格式不一致

**根因**: id-validator.ts 要求 2-16 位，但 id-generator.ts 和 init.ts 允许 1-16 位

**证据**:
- `id-validator.ts:10` - `[A-Z][A-Z0-9]{1,15}` (2-16位)
- `id-generator.ts:54` - `[A-Z][A-Z0-9]{0,15}` (1-16位)
- `init.ts:48` - `[A-Z][A-Z0-9]{0,15}` (1-16位)

**影响**: 用户使用单字符缩写时，nextId 抛出异常

**为何现有机制未发现**:
- validateAbbr 在 nextId 开始时执行，但校验规则与 validateId 不一致
- 测试用例未覆盖单字符缩写场景

---

### 🔴 P0-2: REQ-PRD ID 格式未定义

**根因**: 代码中使用 REQ-PRD-NNN 格式，但 ID_PATTERNS 未定义

**证据**:
- `matrix.ts:72` - `u.startsWith('REQ-PRD-')`
- `sca.ts:151,304` - 检查 REQ-PRD upstream
- `id-validator.ts:8-23` - 无 REQ-PRD 定义

**影响**: parseMatrixContent 将 REQ-PRD-001 错误识别为 Feature 类型

**为何现有机制未发现**:
- parseMatrixContent 使用 fallback: `validation.type ?? 'Feature'`
- 静默降级，不抛出错误

---

### 🔴 P0-3: nextId 并发竞态

**根因**: 读取矩阵、计算序号、写入矩阵三步无锁保护

**证据**: `id-generator.ts:29-49` 无锁机制

**场景**:
```
进程A: parseMatrixIds() → 最大序号 5
进程B: parseMatrixIds() → 最大序号 5
进程A: appendToMatrix(FR-AUTH-006)
进程B: appendToMatrix(FR-AUTH-006)  // 重复
```

**影响**: 并发生成重复 ID

**为何现有机制未发现**:
- 单进程测试无法触发
- 无并发测试用例

---

### 🔴 P0-4: appendToMatrix 非原子操作

**根因**: 读-修改-写 三步分离

**证据**: `id-generator.ts:113-126`

```typescript
const content = readMarkdown(matrixPath);  // 读
const newRow = `| ${row.id} | ... |\n`;
writeMarkdown(matrixPath, content + newRow);  // 写
```

**影响**: 并发写入可能丢失数据

---

### 🔴 P0-5: advance 并发竞态

**根因**: loadState → 修改 → saveState 无锁

**证据**: `advance.ts:113,100` 无锁机制

**影响**: 并发 advance 可能导致 history 丢失

---

### 🔴 P0-6: current 文件无锁保护

**根因**: writeCurrentFeature 直接写文件

**证据**: `init.ts:499-503` 无锁

**影响**: 并发 init 可能覆盖 current 文件

---

### 🔴 P0-7: 矩阵状态未同步

**根因**: advance 推进阶段时，未更新矩阵中 ID 状态

**证据**: `advance.ts:107-171` 未调用 updateMatrixRow

**影响**: Stage 与矩阵状态不一致

**为何现有机制未发现**:
- checkMatrix 只检查引用完整性，不检查状态一致性
- Gate 校验不包含状态同步检查

---

### 🟠 P1-1: 循环依赖静默处理

**根因**: getAncestors 检测到循环返回空集合

**证据**: `upstream-lineage.ts:23`

**影响**: 用户无法发现循环依赖

---

### 🟠 P1-2: 重试计数不准确

**根因**: totalRetryDurationMs 累加 backoffMs 而非实际耗时

**证据**: `retry-controller.ts:191`

**影响**: 预算计算不准确

## 风险评估

### 高风险场景

1. **并发 init** - 多个进程同时初始化不同 Feature
2. **并发 nextId** - Skill 并行生成 ID
3. **并发 advance** - 自动推进与手动推进冲突

### 数据一致性风险

1. **ID 重复** - nextId 并发竞态
2. **状态不一致** - advance 未同步矩阵
3. **引用断链** - REQ-PRD 类型错误
