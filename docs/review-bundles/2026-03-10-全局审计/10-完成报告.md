## 二、核心发现

### 2.1 问题统计

**总计**: 12 个真实问题
- 🔴 严重 (CRITICAL): 4 个
- 🟠 高 (HIGH): 5 个
- 🟡 中 (MEDIUM): 3 个

### 2.2 严重问题 (CRITICAL)

1. **C1: nextId 并发竞态 + 非幂等**
   - 无锁保护，TOCTOU 竞态
   - 重复执行生成不同 ID
   - 影响所有 Artifact ID 类型

2. **C2: todo-state.json 双 schema 冲突**
   - TodoRunnerState 与 CheckpointState 写入同一文件
   - 恢复链断裂，状态解析错误
   - 影响 orchestrate 与 batch executor

3. **C3: 单字符缩写格式不一致**
   - 生成允许 1-16 位，校验要求 2-16 位
   - 导致运行时错误
   - 影响所有 ID 生成

4. **C4: REQ-PRD 类型未定义**
   - 代码使用但未定义格式
   - parseMatrixContent 错误识别为 Feature
   - 影响矩阵解析

### 2.3 高优先级问题 (HIGH)

5. **H1: TASK 解析规则碎片化**
   - 7 处不同的 TASK 解析逻辑
   - 状态词支持不一致
   - 影响 commit/catchup/prompt/batch

6. **H2: Feature ID 解析不统一**
   - 目录名、stage-state.featureId、current 三者未校验一致性
   - 影响 Feature 解析与状态读取

7. **H3: retry-controller 未接入**
   - retry 逻辑定义但未调用
   - budget/backoff 保护未生效
   - 影响错误恢复

8. **H4: advance 并发竞态**
   - 无锁保护
   - 并发推进可能丢失 history
   - 影响阶段推进

9. **H5: 审计 hash 链并发不安全**
   - 无锁保护，可能链分叉
   - 损坏行静默跳过
   - 影响审计完整性


### 2.4 中优先级问题 (MEDIUM)

10. **M1: appendToMatrix 非原子**
    - 非原子文件操作
    - 写入中断可能损坏矩阵

11. **M2: current 文件无锁**
    - 并发写入可能覆盖
    - 影响 Feature 指针

12. **M3: 循环依赖静默处理**
    - 检测到循环返回空集合
    - 用户无法发现问题

## 七、审计总结

### 7.1 整体评价

**Feature ID 主链**: ⭐⭐⭐⭐ (中等可靠)
- init 有锁保护，恢复机制完善
- 但解析规则不统一，缺少三元组校验

**Artifact ID 主链**: ⭐⭐ (局部可靠)
- 格式定义清晰，校验规则完整
- 但生成无锁，非幂等，并发不安全

**恢复链**: ⭐⭐⭐ (存在断点)
- catchup 机制设计完善
- 但 schema 冲突、解析碎片化导致假恢复

**幂等性**: ⭐⭐ (较弱)
- init 较好，nextId 较弱
- 缺少幂等键和去重机制

### 7.2 关键洞察

1. **并发安全是最大短板**
   - Feature 初始化有锁，但后续操作无锁
   - 需要系统性补充锁机制

2. **Schema 管理缺失**
   - 多处状态文件无版本控制
   - 需要引入 schema 守卫

3. **解析规则碎片化**
   - 同一产物多处解析逻辑不一致
   - 需要统一 parser

4. **静默降级过多**
   - 错误被吞没，用户无感知
   - 需要显式报错

### 7.3 用户关心的"死循环"问题

**结论**: ✅ 未发现真正的无限循环

- auto-loop 受 `maxIterations` 保护
- watchdog 有 timeout 和 heartbeat 检查
- 不存在无上限的无限重试

**但发现**:
- retry-controller 未接入（H3）
- todo-state schema 冲突导致假恢复（C2）
- TASK 解析碎片化导致错误恢复（H1）
