## 一、严重问题 (CRITICAL)

### C1. nextId 并发竞态 + 非幂等

**状态**: ✅ 两份报告均确认

**Claude 报告**: P0-3 - nextId 并发竞态
**Codex 报告**: F3 - nextId 非原子且非幂等

**证据**:
- `id-generator.ts:29-49` - 读取、计算、写入三步无锁
- `id-generator.ts:113-126` - appendToMatrix 非原子操作

**根因**:
1. 无锁保护 - 并发读写导致 TOCTOU
2. 非幂等 - 重复执行生成不同 ID
3. 无去重 - matrix 可能包含重复 ID

**影响**: 并发或重试场景下生成重复 ID

---

### C2. todo-state.json 双 schema 冲突

**状态**: ✅ Codex 报告独家发现

**Codex 报告**: F1 - todo-state.json 被两套不兼容 schema 复用

**证据**:
- `todo-runner.ts:52` - 定义 TodoRunnerState
- `checkpoint.ts:7` - 定义 CheckpointState
- 两者写入同一文件 `specs/{featureId}/todo-state.json`

**根因**: 文件路径复用，schema 不兼容

**影响**: 恢复链断裂，状态解析错误

---

### C3. 单字符缩写格式不一致

**状态**: ✅ 两份报告均确认

**Claude 报告**: P0-1 - 单字符缩写导致 ID 生成失败
**Codex 报告**: 隐含在 ID 格式校验中

**证据**:
- `id-validator.ts:10` - `[A-Z][A-Z0-9]{1,15}` (2-16位)
- `id-generator.ts:54` - `[A-Z][A-Z0-9]{0,15}` (1-16位)

**根因**: 生成与校验规则不一致

**影响**: 单字符缩写导致运行时错误

---

### C4. REQ-PRD 类型未定义

**状态**: ✅ Claude 报告确认

**Claude 报告**: P0-2 - REQ-PRD ID 格式未定义

**证据**:
- `matrix.ts:72` - 使用 `startsWith('REQ-PRD-')`
- `id-validator.ts:8-23` - 无 REQ-PRD 定义

**根因**: 代码使用但未定义格式

**影响**: parseMatrixContent 错误识别为 Feature 类型

## 二、高优先级问题 (HIGH)

### H1. TASK 解析规则碎片化

**状态**: ✅ Codex 报告独家发现

**Codex 报告**: F2 - TASK 解析规则碎片化

## 六、修复策略

### 锁机制统一

建议抽象通用锁工具 `withFileLock(path, action)`，统一用于：
- nextId (matrix lock)
- advance (state lock)
- current (pointer lock)
- audit log (chain lock)

### Schema 守卫

为所有状态文件增加 `kind/version` 字段：
- `todo-state.json` → `TodoRunnerState`
- `batch-checkpoint.json` → `CheckpointState`
- 加载时严格校验 schema

### Parser 统一

抽出单一 `parseTaskPlan()` 函数：
- 返回结构化 AST
- 所有模块依赖同一实现
- 状态词统一走 `normalizeTodoStatus()`

### 测试补充

增加以下测试场景：
- 并发 ID 生成测试
- Schema 冲突测试
- Parser 一致性测试
- 循环依赖检测测试
