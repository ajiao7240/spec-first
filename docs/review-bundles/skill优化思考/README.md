# skill 优化思考

本目录汇总针对 `spec-first` 各类 skill 的职责边界、入口设计与优化方案思考文档。

## 审查方式

- 先维护完整 skill 清单与审查跟踪表
- 再按顺序逐个审查、逐个形成单独文档
- 每完成一个 skill，就回写本 README 的审查状态与结论

## 当前结论摘要

- `first` 更适合研发 / 架构视角的项目认知入口
- `init` 更适合作为研发需求开发起点
- 产品更应直接进入 `spec` 相关链路
- 测试设计应前置进入 `task`，TDD 执行进入 `code`，最终由 `verify` 统一放行
- 最优设计不是让所有人都跑 `first`，而是建立统一认知底座，再按角色输出不同入口
- 当前系统已有清晰骨架，但控制层边界、测试阶段定位、规则真理源和命令面仍需收敛
- 最值得优先收敛的不是主链阶段命令，而是 `feature`、`context`、`review` 这三组辅助命令
