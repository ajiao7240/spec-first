# Spec-First 安装与更新说明

> **模式**: npm 全局安装（推荐） / pnpm link（开发模式）

## 前置条件

| 依赖 | 最低版本 | 说明 |
|------|---------|------|
| Node.js | ≥ 20.0.0 | 必需 |
| Git | ≥ 2.x | 必需 |
| pnpm | ≥ 8 | 仅开发模式需要 |

## npx 零安装（推荐试用）

无需全局安装，进入目标项目后可直接执行：

```bash
cd /path/to/your-project

# 诊断环境
npx -y spec-first@latest doctor

# 初始化 Feature
npx -y spec-first@latest init --feat REPORT --mode N --size M --platforms backend,admin-frontend --title "个人报告"

# 启动可视化面板并输出地址（自动动态端口）
npx -y spec-first@latest viewer open --print-url
```

> 说明：`npx` 每次会拉取/解析包，适合试用与临时执行；长期建议使用全局安装。

## 安装

```bash
npm install -g spec-first@latest

# 验证
spec-first --version
```

全局安装自动完成：
- CLI 全局可用（`spec-first` 命令）
- Skill 命令注册到 Claude Code + Codex
- MCP 配置补齐（sequential-thinking、context7 等）
- Claude Code SessionStart Hook 注册（打开项目自动启动 Viewer）

> 若 postinstall 未自动执行（pnpm 全局安装等场景），手动运行 `spec-first update` 即可。

### 验证安装

```bash
# 确认 CLI 可用
spec-first --version

# 确认 Skill 命令已注册
ls ~/.claude/commands/spec-first/*.md
ls ~/.codex/skills/spec-first/*
```

应看到 22 个 `~/.claude/commands/spec-first/*.md` 文件，以及 22 个 `~/.codex/skills/spec-first/*` 目录副本。

### 可视化面板（Viewer）

全局安装后，打开 Claude Code 会自动启动 Viewer（通过 SessionStart Hook）。

手动操作：

```bash
# 手动启动并打印 URL
spec-first viewer open --print-url

# 查看当前项目的面板地址
cat .spec-first/runtime/viewer-state.json
```

可选环境变量：
- `SPEC_FIRST_VIEWER_PORT`：固定端口（不设则自动分配）
- `SPEC_FIRST_VIEWER_HOST`：监听地址（默认 `127.0.0.1`）
- `SPEC_FIRST_VIEWER_AUTO_OPEN`：是否自动打开浏览器（`1`/`0`）

> Codex 暂无 SessionStart Hook 机制，需手动执行 `spec-first viewer open --print-url`。


## 首次使用（目标项目）

在目标项目中初始化第一个功能项（Feature）：

```bash
cd /path/to/your-project

# 初始化功能项（交互式引导，或直接传参）
spec-first init --feat AUTH --mode N --size M --platforms h5,java-backend
```

注意事项：
- `--feat` 必须全大写字母开头，匹配 `^[A-Z][A-Z0-9]{0,15}$`（如 `AUTH`、`REPORT`）
- `--platforms` 指定的平台必须在项目 `.spec-first/layer2/<platform>.yaml` 中存在（平台名即文件名去掉 `.yaml`）
- `init` 成功后会自动在项目 `.claude/commands/spec-first/` 下注册技能命令入口
- `init` 在 Git 仓库中会自动安装/更新 4 个 hooks：`prepare-commit-msg`、`commit-msg`、`pre-push`、`pre-commit`
- `init` 会自动注册 AI Runtime Hooks（PreToolUse/PostToolUse/Stop）到 `.claude/settings.json`

## Git Hooks 管理

```bash
# 手动安装/更新 hooks
spec-first hooks install

# 查看 hooks 状态
spec-first hooks status

# 卸载 spec-first hooks
spec-first hooks uninstall
```

说明：
- 如果当前目录不是 Git 仓库（无 `.git`），`hooks install/uninstall` 会给出提示并退出。
- `doctor` 在无 `.git` 场景会显示“已跳过”，不再报 hooks 告警。

## 更新

```bash
# 升级到最新版本
npm install -g spec-first@latest

# 刷新 Skill/MCP/Hooks（建议在项目目录执行）
cd your-project
spec-first update
```

`update` 命令自动完成：
- 刷新 Skill 命令到 Claude Code + Codex
- 补齐 MCP 配置
- 刷新 Git hooks（当前项目）
- 刷新 AI Runtime Hooks（PreToolUse/PostToolUse/Stop）
- 刷新全局 SessionStart Hook

可选参数：
- `--dry-run`：仅输出将发生的变更，不写文件
- `--skip-mcp`：跳过 MCP 配置补齐
- `--skip-hooks`：跳过 Git hooks 刷新
- `--host <target>`：仅刷新指定宿主（claude|codex|generic|all，可多次或逗号分隔）

> CLI 执行时若检测到新版本可用，会输出升级提示（不阻塞执行）。

技能文件（`skills/spec-first/*/SKILL.md`、`AGENTS.md`）修改后无需构建，Claude Code 下次会话会自动加载。

## 开发模式（仅源码贡献者）

```bash
git clone <repo> && cd spec-first
pnpm install && pnpm build && pnpm link --global
spec-first update   # 替代原来的 setup --global
```

> `setup --global` 已废弃（deprecated），内部转发到 `update`。

## 卸载

### 自动卸载（推荐）

```bash
# npm 全局安装方式 — preuninstall 自动清理宿主配置
npm uninstall -g spec-first

# 或 pnpm link 开发模式
pnpm remove --global spec-first
```

`npm uninstall -g` 会自动触发 `preuninstall` 脚本，等效于执行 `spec-first uninstall --keep-mcp`，自动清理：
- `~/.spec-first/skills/spec-first/`（Skills 缓存）
- `~/.claude/commands/spec-first/`（Claude 命令入口）
- `~/.codex/skills/spec-first/`（Codex skills）
- `~/.claude/settings.json` 中的 SessionStart Hook 条目
- 项目 `.claude/settings.json` 中的 AI Runtime Hooks（PreToolUse/PostToolUse/Stop/SessionStart/SessionEnd）
- `.git/hooks/` 中的 spec-first hook 块

### 手动卸载

如果 preuninstall 未自动执行（如 pnpm link 模式），先手动清理再卸载：

```bash
# 在项目目录中执行（清理项目级 hooks）
cd your-project
spec-first uninstall

# 然后卸载
pnpm remove --global spec-first
```

`uninstall` 命令选项：
- `--dry-run`：仅输出将清理的内容，不执行删除
- `--keep-mcp`：保留 MCP 配置（默认也保留，MCP 服务为通用组件）

### 彻底清理（可选）

卸载后如需彻底清理所有残留：

```bash
# 清理 spec-first 用户级目录
rm -rf ~/.spec-first

# 清理 MCP 配置中的 spec-first 相关服务（按需）
# 编辑 ~/.config/claude-code/mcp.json 和 ~/.codex/config.toml
```

## 常见问题

**Q: Claude Code 中看不到 `/spec-first:<skill>` 命令？**

确认已执行 `spec-first update`。使用 `ls ~/.claude/commands/spec-first/*.md` 检查文件是否存在。

**Q: `spec-first init` 报 FEAT 格式错误？**

FEAT 缩写必须大写字母开头，仅含大写字母和数字（匹配 `^[A-Z][A-Z0-9]{0,15}$`），如 `AUTH`、`REPORT`、`V3PAY`。小写或混合大小写会被拒绝。

**Q: init 时 platforms 相关报错？**

`init` 会对 `--platforms` 做前置校验：每个平台都必须存在对应 Layer2 YAML（`.spec-first/layer2/<platform>.yaml`），否则会直接报错并中止。

可先执行 `ls .spec-first/layer2/*.yaml | xargs -n1 basename | sed 's/.yaml$//'` 查看可选平台。

YAML 中的 `threshold` 等数值字段必须是数字类型，不能写成纯文本描述。

**Q: doctor/init 提示 `Binary/serena: Executable check failed`？**

先验证 Serena MCP 启动命令：

```bash
uvx --from git+https://github.com/oraios/serena serena start-mcp-server --help
```

命令行工具会自动尝试兼容回退：
- `uvx --from git+https://github.com/oraios/serena serena-mcp-server --help`
- `npx -y mcp-server-serena --help`

**Q: doctor 提示 Git hooks 未安装（prepare-commit-msg/commit-msg/pre-push/pre-commit）？**

执行：

```bash
spec-first hooks install
```

如果你是刚初始化的新 Feature，可先确认是否在 Git 仓库中（`ls .git`），因为自动安装只在 Git 仓库内生效。


**Q: 新项目里为什么不能直接运行 `node scripts/stage-viewer/bootstrap.js`？**

该路径只在 spec-first 源码仓库内存在。目标项目请使用 CLI 命令：

```bash
spec-first viewer open --print-url
# 或零安装方式
npx -y spec-first@latest viewer open --print-url
```
