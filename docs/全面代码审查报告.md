### ⚠️ 2.2 发现的问题

#### 问题 1: lib/ 目录残留

**现状**: 存在 `lib/` 目录，但已改用 `src/`

**检查**:
```bash
$ ls lib/ 2>/dev/null
cli.js  doctor.js  init.js  spec-commands.js  templates.js
```

**影响**:
- 造成混淆
- 可能被误用

**建议**: 删除 `lib/` 目录

---

#### 问题 2: package.json files 字段包含过多内容

**当前配置**:
```json
"files": [
  "bin/",
  "src/",
  "templates/",
  "skills/",      // 41 个 skills
  "agents/",      // 6 个 agents
  "scripts/",     // 6 个 scripts
  "README.md"
]
```

**问题**:
- skills/ 包含 41 个 skills，只有 5 个核心
- agents/ 和 scripts/ 在 npm CLI 模式中不使用
- 导致包体积过大

**建议**:
```json
"files": [
  "bin/",
  "src/",
  "templates/",
  "README.md"
]
```

或者只包含核心 skills:
```json
"files": [
  "bin/",
  "src/",
  "templates/",
  "skills/spec-brainstorm/",
  "skills/spec-plan/",
  "skills/spec-work/",
  "skills/spec-review/",
  "skills/spec-compound/",
  "README.md"
]
```
