### 1.2 核心设计原则

| 原则 | 说明 |
|------|------|
| **上下文注入** | 规范由 Hook 动态注入，不依赖 AI 记忆 |
| **纯调度器** | Dispatch Agent 只调度，不分析需求 |
| **质量门禁** | Hook 层强制完成度检查 |
| **需求拒绝** | Plan Agent 有权拒绝不合理需求 |
| **最小变更** | 所有修改遵循最小变更原则 |

## 2. Hook 系统架构

### 2.1 Hook 流水线

```
┌─────────────────────────────────────────────────────────────┐
│                    Hook Pipeline                             │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  会话开始                                                    │
│      │                                                       │
│      ▼                                                       │
│  ┌──────────────────┐                                        │
│  │ SessionStart Hook│ ─── 注入全局上下文                     │
│  └────────┬─────────┘                                        │
│           │                                                  │
│           ▼                                                  │
│  AI 工作中...                                                │
│           │                                                  │
│           ▼                                                  │
│  ┌──────────────────┐                                        │
│  │  PreToolUse Hook │ ─── 注入任务特定上下文                  │
│  └────────┬─────────┘     (每次 Task 调用前)                 │
│           │                                                  │
│           ▼                                                  │
│  子代理执行                                                  │
│           │                                                  │
│           ▼                                                  │
│  ┌──────────────────┐                                        │
│  │ SubagentStop Hook│ ─── Ralph Loop 质量门禁                │
│  └────────┬─────────┘     (子代理停止前)                     │
│           │                                                  │
│           ▼                                                  │
│  任务完成                                                    │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 Hook 职责

| Hook | 触发时机 | 核心职责 |
|------|----------|----------|
| **SessionStart** | 会话开始 | 注入工作流、规范索引、恢复状态 |
| **PreToolUse** | Task 调用前 | 读取 JSONL，注入任务特定上下文 |
| **SubagentStop** | 子代理停止 | 检查完成标记，强制完成度 |

## 5. Ralph Loop 质量门禁

### 5.1 工作机制

```
┌─────────────────────────────────────────────────────────────┐
│                 Ralph Loop Quality Gate                      │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  Step 1: Check Agent 执行完成                                │
│          │                                                   │
│          ▼                                                   │
│  Step 2: 尝试停止                                            │
│          │                                                   │
│          ▼                                                   │
│  Step 3: SubagentStop Hook 拦截                              │
│          │                                                   │
│          ▼                                                   │
│  Step 4: 检查迭代次数 (≤5)                                   │
│          │                                                   │
│          ├──── 超过 ────▶ 强制停止                           │
│          │                                                   │
│          ▼                                                   │
│  Step 5: 检查完成标记                                        │
│          │                                                   │
│          ├──── 全部存在 ────▶ 允许停止                       │
│          │                                                   │
│          ├──── 缺少标记 ────▶ 阻止停止                       │
│          │                         │                         │
│          │                         ▼                         │
│          │                   生成继续消息                     │
│          │                         │                         │
│          │                         ▼                         │
│          │                   Agent 继续执行                   │
│          │                         │                         │
│          └─────────────────────────┘                         │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```


## 6. 需求拒绝机制

### 6.1 Plan Agent 拒绝门禁

```
┌─────────────────────────────────────────────────────────────┐
│                  Requirement Rejection                       │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  用户需求 ────▶ Plan Agent 评估                              │
│                      │                                       │
│                      ▼                                       │
│              ┌───────────────┐                               │
│              │   模糊需求?   │──── 是 ────▶ 要求具体化       │
│              └───────┬───────┘                               │
│                      │ 否                                    │
│                      ▼                                       │
│              ┌───────────────┐                               │
│              │  不完整需求?  │──── 是 ────▶ 要求补充信息     │
│              └───────┬───────┘                               │
│                      │ 否                                    │
│                      ▼                                       │
│              ┌───────────────┐                               │
│              │  过大需求?    │──── 是 ────▶ 要求拆分任务     │
│              └───────┬───────┘                               │
│                      │ 否                                    │
│                      ▼                                       │
│              ┌───────────────┐                               │
│              │  有害需求?    │──── 是 ────▶ 直接拒绝         │
│              └───────┬───────┘                               │
│                      │ 否                                    │
│                      ▼                                       │
│                  接受需求                                    │
│                      │                                       │
│                      ▼                                       │
│                  创建任务                                    │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 6.2 拒绝类型

| 类型 | 说明 | 处理方式 |
|------|------|----------|
| **模糊** | 需求不够具体 | 要求明确目标、指标 |
| **不完整** | 缺少关键信息 | 要求补充输入/输出/边界 |
| **过大** | 单任务涉及多模块 | 要求拆分为子任务 |
| **有害** | 存在安全/规范风险 | 直接拒绝，说明原因 |

## 7. 对 spec-first 的借鉴总结

### 7.1 核心借鉴点优先级

| 借鉴点 | 优先级 | 价值 | 实现难度 |
|--------|--------|------|----------|
| **JSONL 上下文定义** | P0 | 替代硬编码，灵活配置 | 中 |
| **Ralph Loop 质量门禁** | P0 | 强制完成度检查 | 中 |
| **Hook 注入机制** | P0 | 确保规范遵循 | 中 |
| **需求拒绝机制** | P1 | 及早识别问题需求 | 低 |
| **纯调度器模式** | P1 | 降低 Agent 复杂度 | 低 |
| **Workspace 日志** | P1 | 会话历史追溯 | 低 |
| **动态完成标记** | P1 | 灵活质量门禁 | 低 |

### 7.2 架构借鉴建议

```
┌─────────────────────────────────────────────────────────────┐
│              spec-first 借鉴架构建议                         │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  1. 上下文管理                                               │
│     ├─ 采用 JSONL 格式定义上下文                             │
│     ├─ 每个 Feature 可配置不同上下文                         │
│     └─ Hook 层动态注入                                       │
│                                                              │
│  2. 质量门禁                                                 │
│     ├─ 在 Skill 执行后添加质量门禁                           │
│     ├─ 使用完成标记验证输出                                  │
│     └─ 迭代限制避免无限循环                                  │
│                                                              │
│  3. Agent 编排                                               │
│     ├─ 分离调度与执行                                        │
│     ├─ Plan 阶段可拒绝需求                                   │
│     └─ Research Agent 辅助分析                               │
│                                                              │
│  4. 会话管理                                                 │
│     ├─ Workspace 日志持久化                                  │
│     ├─ 会话状态恢复                                          │
│     └─ 上下文自动注入                                        │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 7.3 实施路线图

```
Phase 1: 基础设施 (P0)
├─ JSONL 上下文定义格式
├─ PreToolUse Hook 实现
└─ Ralph Loop 质量门禁

Phase 2: Agent 增强 (P1)
├─ Plan Agent 拒绝机制
├─ Research Agent 集成
└─ Dispatch 纯调度器

Phase 3: 会话管理 (P1)
├─ Workspace 日志系统
├─ SessionStart Hook
└─ 会话状态恢复

Phase 4: 优化迭代 (P2)
├─ 动态完成标记
├─ 迭代限制优化
└─ 错误处理增强
```
