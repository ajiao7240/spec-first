## 五、主要问题

### 5.1 产物生成已扩展，节点接入未同步扩展

当前问题的本质不是 `first` 生成得不够，而是：

- `first` 已经进入 runtime-first 完整模式
- 下游 skill 仍主要依赖旧的最小入口集

这会导致：

- 新增真源价值无法传导到后续阶段
- 正式专题文档虽然存在，但不进入主要工作流
- 方案设计与节点消费仍有断层

### 5.2 专题文档接入明显不足

当前 5 个正式专题文档中：

- `architecture.md`、`call-graph.md` 已在 onboarding 中接入
- `external-deps.md`、`local-setup.md`、`development-guidelines.md` 尚未看到下游节点显式使用

这说明专题文档“已经生成”，但“还没有进入系统性消费”。

### 5.3 新增 runtime 真源尚未形成下游输入层

以下 runtime 真源当前尚未看到被下游 skill 节点显式接入：

- `steering.json`
- `conventions.json`
- `critical-flows.json`
- `change-map.json`
- `entry-guide.json`
- `reboot-guide.json`
- `api-contracts.json`
- `structure-overview.json`
- `domain-model.json`

这与此前方案中“后续 skill 以 runtime truth 为输入层”的目标仍有差距。

### 5.4 modules.json 仍是残留文件

当前 runtime 目录里仍存在 `modules.json`，但：

- 该文件不在现行正式 contract 中
- 下游 skill 未见显式引用
- 容易造成“目录实况”和“正式模型”不一致

建议在下一轮治理里清理。
