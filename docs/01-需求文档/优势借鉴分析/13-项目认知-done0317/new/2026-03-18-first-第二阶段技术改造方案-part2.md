# Background Quality Contract

## 背景质量三级状态

### full
- 所有 required 资产可用
- 至少 50% optional 资产可用
- 节点可以高质量执行

### degraded
- 部分 required 资产缺失
- 节点可以执行，但需要更多假设
- 应提示用户运行 `spec-first first`

### blind
- 所有 required 资产缺失
- 节点仍可执行，但质量无保障
- 必须明确告知用户风险

## 节点降级行为规范

### 检查时机
- 节点启动时检查 `backgroundInputStatus`
- 从 `ResolvedSkillContext` 读取状态

### 降级行为
- **full**: 正常执行，无需提示
- **degraded**: 执行前提示缺失资产，建议运行 `first`
- **blind**: 执行前明确告知风险，询问是否继续

### 禁止行为
- ❌ 因背景缺失而阻断流程
- ❌ 自动运行 `spec-first first`（除非用户明确授权）
- ❌ 忽略降级状态，不告知用户

## 示例

### spec 节点
- **required**: steering, conventions
- **degraded 行为**: 提示"缺少项目导向和规范，可能需要更多澄清"
- **blind 行为**: 提示"无项目背景，将基于通用最佳实践"

### code 节点
- **required**: conventions, entry-guide
- **degraded 行为**: 提示"缺少开发规范，可能不符合项目约定"
- **blind 行为**: 提示"无项目背景，将使用通用代码风格"

## 5. 风险与缓解

### 风险 1：token 预算难以精确控制

**描述**：不同 runtime 资产大小差异大，难以精确预测 token 使用量

**缓解**：
1. 实现"软限制"策略：超出 10% 以内允许
2. 提供 `--first-context-budget` CLI 参数，允许用户覆盖
3. 记录实际 token 使用量，用于后续优化

### 风险 2：降级提示可能被忽略

**描述**：用户可能忽略 degraded / blind 提示，导致质量问题

**缓解**：
1. 降级提示使用醒目格式（⚠️ 符号）
2. 在 skill 执行结束时再次提示
3. 在 `spec-first status` 中显示背景健康度

### 风险 3：输入矩阵维护成本

**描述**：新增 skill 时需要同步更新输入矩阵

**缓解**：
1. 在 `SKILL_INPUT_MATRIX` 中提供 `default` 配置
2. 新增 skill 时自动继承 `default`
3. 单元测试检查所有 skill 都有定义

### 风险 4：向后兼容性

**描述**：现有 skill 可能依赖旧的上下文获取方式

**缓解**：
1. 保留 `resolveSkillContext()` 的旧接口
2. 新增 `options` 参数为可选
3. 逐步迁移现有 skill

## 7. 后续优化方向

### 优化 1：智能裁剪

**目标**：根据 skill 实际使用情况动态调整裁剪策略

**方案**：
1. 记录每个 skill 实际使用的资产字段
2. 优先裁剪未使用的字段
3. 定期分析使用数据，优化默认配置

### 优化 2：缓存机制

**目标**：减少重复读取 runtime 资产的开销

**方案**：
1. 在 `context-resolver` 中实现 LRU 缓存
2. 缓存 key 为 `${projectRoot}:${skillName}:${runtimeHash}`
3. runtime 变更时自动失效

### 优化 3：按需加载

**目标**：支持"先注入摘要，按需加载详情"模式

**方案**：
1. 在 prompt 中只注入资产摘要（如 `steering.goals`）
2. 提供 `@first:steering:full` 指令，允许 AI 按需加载完整内容
3. 通过 tool call 实现动态加载

## 8. 一句话结论

第二阶段的核心是"让下游节点按需消费 first，提升质量但不阻断流程"，技术实现的关键是：

1. **输入矩阵代码化**：明确每个 skill 需要什么
2. **context-resolver 增强**：支持 token 控制和降级策略
3. **dispatcher 接入**：自动注入 first 上下文到 prompt
4. **背景质量契约**：统一降级行为规范

只有这样，`first` 才能从"可选增强"真正变成"全流程公共背景输入层"。
