# 第三批专家集成改造任务文档

> 日期：2026-03-15  
> 范围：第三批低优先 / 按需 skill  
> 路线：继续沿用前两批路线，只做轻量 `fragment` 吸收与 `command-local guidance`，不引入重专家语义或额外运行时  
> 本版目标：把 `00-first`、`00-onboarding`、`01-init`、`02-catchup`、`14-status`、`15-doctor`、`16-sync`、`17-feature` 收敛成“最小增强、强一致性、低侵入”的任务清单

## 一、改造目标

| 项目 | 内容 |
|------|------|
| 保持不变 | 不改阶段状态机、不改 Gate 真源、不改变命令主语义、不引入第二套恢复/诊断系统 |
| 前置依赖 | 第一批 canonical profile 与第二批 fragment 附着机制已建立 |
| 第三批 skill | `00-first`、`00-onboarding`、`01-init`、`02-catchup`、`14-status`、`15-doctor`、`16-sync`、`17-feature` |
| 第三批定位 | 只做“引导、恢复、诊断、同步、切换”的轻量专家增强，不把这些 skill 变成重审查入口 |
| 本版重点 | 补齐 `01-init`、`16-sync`、`17-feature` 缺失的独立 docs test；采用 `skillName/static fragment + command-local guidance` 模式，避免无上下文命令硬套 task/stage 映射 |

## 二、当前实现约束

| 约束 | 当前实现 | 对任务拆分的影响 |
|------|------|------|
| `00-first` 已有大量 runtime truth source / docs projection 契约 | `tests/unit/first-skill-docs.test.ts`、`tests/unit/first-command.test.ts`、`src/core/skill-runtime/first-*` | 只能补轻量认知/建议层，不应扰动 first runtime 真源 |
| `00-onboarding` 已与 first runtime 资产联动 | `tests/unit/onboarding-skill-docs.test.ts`、`src/cli/commands/onboarding.ts` | 适合增强角色引导和路径推荐，不适合加复杂专家判断 |
| `01-init` 是 Feature 工作区和 stage-state 建立入口 | `src/cli/commands/init.ts`、`src/core/process-engine/init.ts`、`tests/unit/init.test.ts` | init 只能增强前置检查与建议，不应破坏初始化幂等与骨架产物 |
| `02-catchup` 已有上下文恢复实现 | `src/core/ai-orchestrator/catchup.ts`、`tests/unit/catchup-skill-docs.test.ts`、`tests/unit/ai-orchestrator.test.ts` | catchup 增强应围绕恢复摘要、阻塞识别、下一步建议，避免重写恢复逻辑 |
| `14-status` 已有背景质量与状态视图契约 | `tests/unit/status-skill-docs.test.ts`、`src/shared/status-mapper.ts` | status 适合增强风险解释和行动建议，不应改变状态真源映射 |
| `15-doctor` 已有 CLI 诊断与宿主修复路径 | `src/cli/commands/doctor.ts`、`tests/unit/doctor-skill-docs.test.ts`、`tests/unit/cli-metrics-doctor.test.ts` | doctor 只能补诊断分类和修复建议分层，不能改变现有 baseline 诊断结论 |
| `16-sync` 是矩阵/上下文同步入口 | `src/core/change-mgr/sync.ts`、`src/core/tool-integration/context-sync.ts`、`tests/unit/context-sync.test.ts` | sync 增强要围绕同步策略和冲突解释，不能引入新同步真源 |
| `17-feature` 是 feature 切换与列举命令 | `src/cli/commands/feature.ts`、`src/core/process-engine/feature.ts`、`tests/unit/feature.test.ts` | feature 只适合增强导航提示和状态摘要，不应承接专家决策 |
| 第三批多为无稳定 `feature/stage/task` 上下文命令 | `first/init/doctor/feature/onboarding` 多直接走 CLI 参数和 cwd | 不应接入 `taskType -> profile` 动态选择；应采用 `skillName -> fragments` 或 command-local guidance |
