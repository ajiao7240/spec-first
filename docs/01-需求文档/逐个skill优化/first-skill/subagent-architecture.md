# Subagent-Driven 架构设计

> 本文档描述 00-first skill 采用 Subagent-Driven 模式的架构设计方案。
> **v2.0.0**: 新增 quick/deep 双模式，quick 模式 4-5 个 Agent，deep 模式 8 个 Agent 三波派发。

---

## 设计目标

- **并行执行**：多个子 agent 同时工作，减少总执行时间
- **上下文隔离**：每个 agent 专注单一任务，避免主 context 爆炸
- **可观测性**：每个任务进度独立追踪，调试友好
- **容错性**：单个 agent 失败不阻塞其他 agent
- **模式分离**：quick 模式聚焦核心，deep 模式提供完整分析

---

## 架构概览

### quick 模式（4-5 个 Agent）

```
┌─────────────────────────────────────────────────────────────┐
│ P0: 定位 + 幂等检测 + Greenfield/Brownfield 判断                │
└─────────────────────────────────────────────────────────────┘
                              │
┌─────────────────────────────────────────────────────────────┐
│ P1a: 技术栈识别 (主线程，<30s)                                 │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│ quick 模式 Agent 派发（4-5 个，并行，<5min）                     │
├─────────────────────────────────────────────────────────────┤
│ • Agent A:  tech-stack.md（主线程直接生成）                    │
│ • Agent B:  codebase-overview.md（简化版，无符号分析）          │
│ • Agent C:  domain-model.md（业务概念与流程）                  │
│ • Agent D:  api-docs.md（轻量级，仅端点列表）                  │
│ • Agent E:  database-er.md（条件派发，检测到 DB 时）            │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│ P2: 汇总输出                                                  │
└─────────────────────────────────────────────────────────────┘
```

### deep 模式（8 个 Agent，三波派发）

```
┌─────────────────────────────────────────────────────────────┐
│ P0: 定位 + 幂等检测 + 端类型检测                                 │
└─────────────────────────────────────────────────────────────┘
                              │
┌─────────────────────────────────────────────────────────────┐
│ P1a: 技术栈识别 (主线程，<30s)                                 │
└─────────────────────────────────────────────────────────────┘
                              │
              ┌───────────────┴───────────────┐
              ▼                               ▼
┌─────────────────────────┐     ┌─────────────────────────┐
│ 第一波 Sub-Agents        │     │ P1b: Context7 收集     │
│ (并行，单 Agent 60s)      │     │ (与第一波并行)           │
├─────────────────────────┤     ├─────────────────────────┤
│ • A1: codebase-overview  │     │ • 映射 ESLint 规则      │
│ • A2: architecture      │     │ • 映射 Prettier 规则    │
│ • A3: call-graph        │     │ • 映射框架最佳实践      │
│ • B:  api-docs          │     └─────────────────────────┘
│ • C1: external-deps     │
│ • D:  database-er       │
└─────────────────────────┘
             │
             ├─ 等待 A1 模块清单 → A2 已在第一波
             │
             ├─ 等待 P1b + C1 → C2
             │
             ▼
┌─────────────────────────────────────────────────────────────┐
│ 第二波 Sub-Agents                                            │
├─────────────────────────────────────────────────────────────┤
│ • C2: development-guidelines → local-setup                    │
└─────────────────────────────────────────────────────────────┘
             │
             ▼ 等待 A2 + B + D 完成
┌─────────────────────────────────────────────────────────────┐
│ 第三波 Sub-Agents                                            │
├─────────────────────────────────────────────────────────────┤
│ • A4: domain-model（等待 A2 + B + D）                        │
└─────────────────────────────────────────────────────────────┘
             │
             ▼
┌─────────────────────────────────────────────────────────────┐
│ P3: 汇总 + README 生成 + 交叉验证                               │
└─────────────────────────────────────────────────────────────┘
```

## Agent 拆分策略

### 评估矩阵

| Agent | 并行潜力 | 复杂度 | 推荐拆分 | 拆分方案 |
|-------|----------|--------|----------|----------|
| **A1: codebase-overview** | ⭐⭐⭐⭐⭐ | 低 | ✅ 是 | 按前后端拆分：A1-Backend + A1-Frontend |
| **A2: architecture** | ⭐⭐⭐ | 中 | ❌ 否 | 依赖 A1，逻辑简单，保持单 Agent |
| **A3: call-graph** | ⭐⭐⭐⭐⭐ | 高 | ✅ 是 | 按前后端拆分：A3-Backend + A3-Frontend |
| **B: api-docs** | ⭐⭐⭐⭐⭐ | 低 | ✅ 是 | 按前后端拆分：B-Backend + B-Frontend |
| **C1: external-deps** | ⭐⭐ | 低 | ❌ 否 | 配置扫描，并行收益小 |
| **C2: dev-guidelines** | ⭐⭐⭐⭐ | 中 | ✅ 是 | 按模块拆分：C2-1~6 并行分析 |
| **D: database-er** | ⭐⭐ | 低 | ❌ 否 | DB 连接单点，无法并行 |
| **A4: domain-model** | ⭐⭐⭐⭐ | 中 | ✅ 是 | 已实现，4 个子任务并行 |

### 拆分原则

| 原则 | 说明 |
|------|------|
| **按模块拆分** | 优先按前后端/语言拆分，而非按步骤 |
| **独立产出** | 每个 sub-agent 产出独立的 JSON 结果 |
| **轻量传递** | 使用 JSON 上下文传递，避免文件 I/O |
| **降级保证** | sub-agent 失败时主线程可降级执行 |

## 数据流设计

### 中间数据格式

#### A1 模块清单（A1-1/A1-2 → A2/A4）

```json
{
  "modules": [
    {
      "name": "auth",
      "path": "backend/app/auth/",
      "responsibility": "认证与授权",
      "type": "backend"
    },
    {
      "name": "admin",
      "path": "frontend/admin/",
      "responsibility": "管理后台",
      "type": "frontend"
    }
  ]
}
```

#### API 文档片段（B-1/B-2 → A4）

```json
{
  "apis": [
    {
      "method": "POST",
      "path": "/api/users",
      "description": "创建用户",
      "file": "backend/app/api/admin/users.py",
      "line": 76
    }
  ]
}
```

#### DB Schema（D → A2/A4）

```json
{
  "tables": [
    {
      "name": "users",
      "columns": [...],
      "foreign_keys": [...]
    }
  ]
}
```

### 上下文传递

```
主线程
  ├─ tech_stack (P1a)
  ├─ context7_mappings (P1b)
  ├─ module_list (A1-1 + A1-2 合并)
  ├─ api_docs (B-1 + B-2 合并)
  ├─ external_deps (C1)
  └─ db_schema (D)

↓ 传递给后续 sub-agents

A2: 需要 tech_stack + module_list（如有 db_schema 则补充）
A3: 需要 tech_stack + module_list
A4: 需要 tech_stack + module_list + api_docs + db_schema
C2: 需要 context7_mappings + external_deps
```

## 超时与错误处理

### 超时策略

| 模式 | 单 Agent 超时 | 总超时 |
|------|---------------|--------|
| **quick 模式** | 60s | 120s |
| **deep 模式** | 60s（单个 Agent） | 300s（整体） |

整体并行阶段最大超时：**300s**（deep 模式）。

### 错误处理

```
sub-agent 执行
  │
  ├─ 成功 → 收集结果
  │
  ├─ 超时 → 标记 [超时]，尝试降级或跳过
  │
  ├─ 失败 → 标记 [失败]，主线程降级执行
  │
  └─ 部分成功 → 使用已生成部分，标记 [部分完成]
```

### 降级示例

```
A3-Backend (LSP) 超时
  ↓ 降级
A3-Backend (静态 import 扫描)
  ↓ 输出
call-graph.md 标注 [依赖分析: 静态模式，LSP 超时]
```
