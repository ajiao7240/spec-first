# Spec-First System Architecture Refactor Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** 把当前 `spec-first` 的混合型实现收敛成稳定的分层架构：CLI 只负责入口，流程引擎只负责状态迁移，`skill-runtime` 只负责路由与上下文，`shared` 只保留公共原语，`ai-orchestrator` 只保留编排控制面。

**Architecture:** 这次重构按”先冻结真源，再拆混合模块，最后收敛边界”的顺序推进。先把 `process-engine` 的 canonical 入口锁住，避免迁移时状态语义漂移；再把 `shared` 与 `trace-engine` 的 ID 契约拆开，消除基础层反向依赖；然后拆分 `skill-runtime` 和 `ai-orchestrator` 的混合职责；最后把 CLI 注册表从入口文件抽离成独立定义，减少后续命令增长带来的维护成本。

**Tech Stack:** TypeScript, ESM, Vitest, 文件系统持久化, Markdown 文档

**Prerequisites:** 本项目使用 pnpm 作为包管理器，执行前确保已安装 pnpm。

---

## Migration Order

1. 冻结 `process-engine` 的 canonical 迁移入口，保证状态机只保留一个正式入口。
2. 把 ID taxonomy 从 `core` 下沉到 `shared`，解除 `shared -> core` 的反向依赖。
3. 拆分 `skill-runtime` 的路由、策略、上下文投影职责（分两步：3a 路由+策略，3b 上下文投影）。
4. 把 CLI 顶层注册表抽离成数据驱动的命令清单。
5. 拆分 `ai-orchestrator` 的调度、恢复、观测职责，保留单一编排控制面。

- 每个任务都按”先写测试，再最小实现，再回归测试”的顺序执行。
