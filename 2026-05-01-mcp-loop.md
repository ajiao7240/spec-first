## 修复结论

本轮没有把 `using-spec-first` 扩展为新 workflow、状态机、规则引擎或 artifact producer。修订集中在入口治理的语义边界：

- Guide mode 只推荐，不启动。
- Self-work 路由先对齐项目角色契约，再按 review-only、review-plus-fix、implementation/prose change 分流。
- Current-host public workflow 是输出标准；other-host workflow 只作为用户意图信号，需要归一到当前 host。
- Standalone skills 不被伪装成 `/spec:*` 或 `$spec-*`。
- Artifacts 与 validation evidence 归属下游 workflow，不归 entry governor。

## 修复效果

- Workflow 更清晰：减少 “what next?” 被误执行、review+fix 被误停在 review-only、跨宿主入口误写三类高频歧义。
- Codebase -> Graph -> Spec -> Plan -> Tasks -> Code -> Review -> Knowledge 链路更可靠：entry governor 只做入口分流，不抢占下游 workflow 的 artifact 和 judgment。
- 隐式上下文更少：self-work 明确要求读取角色契约，跨宿主入口明确归一规则。
- 架构边界更稳：source skill 是真相源，runtime mirror 只通过 init 生成。
- 工程落地更可验证：contract test 覆盖新增治理语义，changelog 记录用户可见行为变化。
