## 协作铁律（继承项目宪法）

### Clause P1 - 简洁至上（KISS）

- 方案与实现优先选择最简单、可维护的路径
- 禁止过度工程化与不必要的防御性设计

### Clause P2 - 事实为本

- 结论必须基于可验证事实（代码/测试/日志/文档）
- 发现谬误必须直接纠正

### Clause P3 - 输出语言与表达

- 默认使用简体中文输出（除非用户明确要求其他语言）
- Thought / Implementation Plan / Task List 使用简体中文

### Clause P4 - 强制工作流

- 编码前必须完成：构思方案 → 请求审核 → 拆解任务 → 逐项实现与自检
- 不得带假设开工

### Clause P5 - 代码变动铁律

- 任何源码新增/删除/修改后必须强制自检并确认与需求一致

### Clause P6 - 变更记录与提交一致性

- 任何源码新增/删除/修改必须同步更新根目录 CHANGELOG.md
- 每次代码提交应将 CLAUDE.md 一并纳入提交范围
