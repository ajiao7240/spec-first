# PRD — FSREQ-19700101-LEGACY-BASELINE

> 存量系统可分析基线

## 1. 业务目标

### 1.1 问题陈述

当前仓库已经包含 CLI、Core、Skill、Template、宿主集成等多层能力，但这些能力分散在代码、运行态资产、技能文档和历史分析材料中，缺少一份统一、可追溯、可继续向下游阶段传递的现状基线。

如果没有这个基线，后续的 `spec -> design -> task -> code` 流程容易基于不完整或过期的假设推进，导致范围漂移、重复实现和风险遗漏。

### 1.2 业务价值

**预期收益**：
- 建立当前系统的事实底座，为后续设计和任务拆解提供统一输入
- 显式暴露能力缺口、技术债务和风险，减少后续返工
- 将“当前系统到底有什么”与“未来要改什么”分开，降低认知混乱

**成功指标**：
- 所有核心模块、关键流程和宿主边界均被盘点并写入可读文档
- 主要缺口和风险均有等级、证据和下一步动作
- 规格、发现、任务计划和文档关联索引彼此一致，可进入下一阶段审查

### 1.3 现有能力盘点

- CLI 命令路由、初始化、阶段流转、门禁、技能渲染、追溯 ID 生成、上下文构建、度量评分、模板渲染、校验、任务计划解析、规则真源、宿主安装与同步能力均已存在
- 当前运行时资产来自 `.spec-first/runtime/first/`，并由 `skills/` 下的技能目录驱动
- 现有阶段模型为 8 个活跃阶段 + 2 个终态阶段，且 `spec-first` 的主链路已从旧 gate/trace 心智迁移到节点化状态机
- 宿主侧安装已扁平化到 `~/.spec-first/skills` 与 `~/.codex/skills`
- 宿主安装态、宿主命令注册、同步结果和宿主健康诊断结果属于本次基线正式范围
- 仓库可见的外部集成点、部署接口和运维边界属于本次基线正式范围；若仓库内没有实际部署产物，则以代码、文档和运行态约束作为证据来源

**技术栈**：
- Node.js >= 20
- TypeScript >= 5.4
- tsup / Vitest / Handlebars / js-yaml / semver / update-notifier

**集成点**：
- Git 工作区与 Git Hook
- Claude / Codex 宿主技能目录
- 本地运行态资产 `.spec-first/`
- 发布打包与安装检查脚本

## 2. 功能需求

### 2.1 纳入范围

- 现有 core 模块的责任边界与运行流程
- CLI 主链路与技能分发链路
- 宿主安装态和项目运行态资产
- 宿主命令注册、同步、卸载与诊断路径
- 仓库可见的外部集成点、部署接口与运维边界
- 能力缺口、技术债务、风险与开放问题
- 后续设计与任务拆解所需的基线证据

### 2.2 排除范围

- 不设计新功能实现方案
- 不讨论具体代码级重构实现
- 不做向下兼容承诺
- 不将历史分析材料当作当前机制真源

### 2.3 基线交付物

- `spec.md`：结构化 FR/AC 基线
- `prd.md`：业务目标与范围说明
- `findings.md`：发现、决策、风险与下一步
- `task_plan.md`：后续执行的粗粒度任务表
- `document-links.yaml`：文档关联索引
- 宿主安装态与外部边界证据记录：作为 findings 的补充证据纳入

## 3. 非功能需求

### 3.1 追溯性

- 所有关键结论必须能追溯到至少一个证据路径
- 所有核心模块与关键流程必须有清晰的来源说明

### 3.2 完整性

- 必须覆盖 runtime summary 中列出的核心模块
- 必须覆盖 critical flows 中列出的关键流程
- 必须覆盖当前 Feature 的主链路与宿主边界

### 3.3 可靠性与确定性

- 同一组证据反复生成，结论应保持稳定
- 不同时间点的文本措辞可以调整，但事实结论不得漂移
- 不得把推测写成既定事实

### 3.4 安全与合规

- 不得输出密钥、凭据、个人数据或宿主敏感配置
- 外部信息不足时应标记为待确认，而不是擅自补全
- 宿主安装态只作为必要证据，不扩展成不受控的环境描述

### 3.5 可观测性

- 规格中的每个重要结论都应有证据或状态标记
- 未确认项必须能在 findings 中快速定位
- 文档之间的关联关系应可由 document-links 解析
- 宿主安装态与外部边界证据应能从 findings.md 或补充记录中定位

## 4. 验收标准

- [ ] 所有核心模块和关键流程已写入基线文档
- [ ] 能力缺口、技术债务与风险已明确分级
- [ ] 文档关联索引可解析，交付物之间可互相追溯
- [ ] 基线文档可进入 `spec-review`
- [ ] 未使用实现细节冒充需求描述

## 6. 术语表

| 术语 | 定义 |
|------|------|
| 基线（Baseline） | 当前系统事实底座，用于支持后续设计、规划和验证 |
| 证据路径（Evidence Path） | 支撑某条结论的文件或运行态资产位置 |
| 缺口（Gap） | 当前能力与后续需要之间的差距 |
| 技术债务（Technical Debt） | 会增加后续变更成本的历史包袱或遗留实现 |
| 宿主资产（Host Artifact） | 安装在 Claude / Codex / 本地环境中的技能与配置文件 |
