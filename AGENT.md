# Agent Execution Rules

> This file mirrors the repository execution expectations for agent-driven work.

## Hard Rule

Every effective repository update must be summarized in `CHANGELOG.md` before the task is considered complete.

## What Counts as an Effective Update

- Code changes
- Documentation changes
- Configuration changes
- Skill or workflow changes
- Test changes
- Any combination of the above

## When It Can Be Skipped

- Pure analysis with no file changes
- Pure Q&A with no file changes
- Suggestions that are not written back to the repository

## Completion Requirement

If a task changes files, it is not complete until:

1. The relevant files are updated.
2. The changelog entry is added.
3. Verification is performed when applicable.
